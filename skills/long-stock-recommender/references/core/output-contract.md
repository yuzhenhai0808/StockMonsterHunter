# 输出合同（所有模式唯一输出格式来源）

本文件是 long-stock-recommender 所有模式的**唯一**输出格式来源。任何模式输出都必须按本文件规定的表结构、字段、概率写法、自检顺序产出。模式文件不再单独定义输出字段，只能引用本文件。

## 默认输出顺序

1. 数据日期 / 所用模式 / 报价基准（日线收盘 or 实时报价）/ 当前阶段 / 本次结论等级(`实时可交易 / 条件可交易 / 日线候选 / 报价冲突`)
2. 报价校验表（仅当用户问"今天/盘前/盘中/现在买"时必出）
3. **总表**
4. **单票短评**（仅展开前 1-2 只主导偏向最清晰的候选）
5. **价格目标概率表**（每只展开候选必带）
6. 放弃/降级备注（点名票不进 Top 3 时在此说明输给谁）

## 总表（强制 11 列）

```md
| 票 | 模式 | 风险 | 核心逻辑 | 入场区间 | 止损 | 第一目标 | 触达概率 | 第二目标 | 建议仓位上限 | 一句话 |
|---|---|---|---|---|---|---|---:|---|---|---|
| TICKER | 追涨/反弹/回调/价值/财报跳空/叠加 | 低/中/高 | 2-8 个字 | $X.XX-$Y.YY | $X.XX | $X.XX | XX%-XX% | $X.XX | $XXX (总资金 X%) | 可做 / 观察 / 放弃 |
```

字段要求：
- **模式**：必须是 SKILL.md 意图路由表里的标准名（追涨 / 超跌反弹 / 回调买入 / 价值挖坑 / 财报跳空 / 叠加信号），不允许自造
- **核心逻辑**：必须是单一主导偏向（"财报预期差" / "估值修复" / "回调后趋势恢复" / "板块传导" / "业绩拐点"），不要列三四个原因
- **入场区间**：默认 `close*0.985 - close*1.015`；若最新价相对日线收盘偏离 ≥5%，改用最新价重算
- **触达概率**：必须区间（如 `35%-45%`），不允许单点
- **建议仓位上限**：以美元金额 + 总资金占比双标，例如 `$300 (总资金 3%)`
- **结论**：从 `可做 / 观察 / 放弃` 三档中选，不允许其他措辞

## 单票短评（强制 10 字段，控制 8-14 行）

每只展开候选必须含以下 10 行，缺一行视为不合规：

1. **主导偏向**：`这只票不是因为什么都好，而是因为 ______`（一句话；写不出来就别推）
2. **为什么看它**：1-2 行
3. **风险点**：1-2 行（必含资本事件 / 财报窗口 / 板块二元事件三项至少一项的判断）
4. **价格计划**：入场、止损、止盈分行列出
5. **概率校准**：`第一目标触达 XX%-XX%；第二目标触达 XX%-XX%；跌破止损 XX%-XX%`
6. **概率依据**：至少引用 3 项 —— `ATR14% / 20日位置 / 量比 / 相对 SPY 强弱 / 财报窗口 / 催化硬度 / 基本面估值冲突 / 当前报价置信度`
7. **建议仓位上限**：`$XXX (总资金的 X%)`
8. **计划买入股数**：按入场区间上沿估算
9. **失效条件**：什么价格 / 什么时长收不回就放弃
10. **一句话结论**

## 价格目标概率表（每只展开候选必带）

```md
| 价格 | 需要涨跌幅 | 目标周期内触达概率 | 周期末站上/跌破概率 | 操作 |
|---|---:|---:|---:|---|
```

概率写法纪律：
- 必须用区间 `XX%-XX%`，不允许单点
- 默认档位：`>=60%` 高概率但通常空间较小；`40%-60%` 中性可交易；`20%-40%` 赔率型；`<20%` 尾部情景
- Swing 默认给"目标周期内触达"和"周期末站上"
- 用户已有仓位的票，必须先给"反抽触达概率 / 跌破止损概率"，再讨论上涨目标
- 缺少实时/延长交易报价时，必须明写"概率基于最近完整日线 + 可见 Web 信息，置信度降低"，不允许把概率写得过窄
- 用户对该票本金 >30% 时，必须先给"账户风险情景概率表"再讨论上涨目标，不允许用高目标概率替代减仓纪律

## 报价校验表（今日/盘前/盘中/现在买必出）

```md
| 票 | 日线收盘价 | 当前/延长交易价 | 报价时间 | 报价来源 | 相对收盘涨跌 | 可交易性结论 |
|---|---:|---:|---|---|---:|---|
```

触发条件：用户问"今天 / 今日 / 现在买 / 盘前 / 夜盘 / 开盘怎么配 / 高收益当天机会"任一关键词出现。

规则：
- 来源优先级：用户券商报价 > Nasdaq / MarketWatch / ADVFN / StockAnalysis / Stocknear 等
- 最新价相对日线收盘偏离 ≥5% → 原 MCP/日线生成的 entry、buy_zone、止损、止盈、股数、仓位、概率和 R:R 全部作废，必须改用最新价重算，并标注"用户券商报价，需以盘口为准"
- 越过原买区 → 参与方式改为`回踩做`或`二次突破做`
- 拿不到当前/延长交易报价 → 输出必须写"本次不是当前可交易推荐，只是日线候选"，不允许给"现在买"、"主攻"、"可做"等当前交易措辞
- 公开源之间价差或涨跌幅冲突 `>=3%`,或用户报价与网页报价冲突但无法解释时间戳 → 只能标`报价冲突/只观察`,不得给当前入场、仓位或概率

## 证据等级与当日措辞

用户问`今天 / 今日 / 现在买 / 盘前 / 夜盘 / 开盘 / 盘中 / 高收益当天机会`时,每只候选必须先标注:

| 等级 | 允许措辞 |
|---|---|
| A/B | 可给当前条件计划、入场区间、仓位上限和正常概率 |
| C | 只能写`日线候选/待盘前报价`;若给概率,必须写"基于最近完整日线,置信度降低" |
| D | 只能写`报价冲突/数据不完整`;不给入场、仓位或概率 |

普通 swing 请求不要求实时价才能列候选,但必须明确`报价基准=日线收盘`,且不得暗示这是当前可交易价格。

## 用户纠错后的修正版状态表

用户指出价格错误时,必须先撤销旧结论,再输出:

```md
| 票 | 原基准价 | 用户/最新价 | 偏离 | 原计划状态 | 新证据等级 | 修正结论 |
|---|---:|---:|---:|---|---|---|
| TICKER | $X.XX | $X.XX | -X.X% | 作废/需重算 | A/B/C/D | 重算后可做/日线候选/报价冲突 |
```

若无法用用户价重算,不得继续保留原`可做`结论、建议仓位或概率。

## 必出元数据（页眉）

每次输出顶部必须含：
- 数据日期（YYYY-MM-DD，本地库最新交易日）
- 所用模式（同总表"模式"列同名）
- 报价基准（日线收盘 / 实时 / 延长交易 / 用户券商）
- 当前阶段（普通 swing 可写`非当日交易请求`）
- 本次结论等级（实时可交易 / 条件可交易 / 日线候选 / 报价冲突）
- 财报日（若候选中任一只 ≤30 天内有财报）

## 输出前自检清单（每次必跑）

发出回复前对照本表，缺一项就补齐：

1. 总表 11 列齐？模式名是否在标准集合内？
2. 单票短评是否含 10 字段？主导偏向是否一句话？
3. 概率是否全部区间表达？没有单点？
4. 报价校验表（若触发条件命中）是否出？
5. 资本事件复核结论是否在输出中显式写出（"已核查，无红旗" 或 "触发 X 红旗"）？
6. 财报 ≤30 天的候选是否跑过 [screening-workflow.md](../workflow/causal-first.md) "首推前三连校验"？
7. 来源是否区分（MCP筛选 / Web催化 / 财报窗口 / 用户点名复核）？用户点名但全市场排序不领先的票是否标"复核：观察/不推荐"？
8. 价值模式是否给出"持仓周期与退出信号"而不是硬止损？
9. 今日/盘前/盘中请求中,C/D 证据等级是否被阻断为日线候选/报价冲突,且没有当前入场、仓位或正常概率？
10. 若用户纠正报价,是否撤销旧计划并用用户券商价重算;无法重算时是否降为`报价冲突/只观察`？
