# 退出框架（建仓 / 减仓 / 持有 / 被套处理）

本文件统一所有模式的退出叙事。具体止损止盈数值见 [risk-rules.md](../core/risk-rules.md)；本文件聚焦"何时减、怎么减、被套怎么办"的操作动作。

## 一、参与方式（建仓三种）

每只候选必须在输出"价格计划"里明确写出参与方式之一：

1. **回踩做**：当前价已越过原买区，等回踩 MA10 / MA20 / 入场区间下沿再入
2. **二次突破做**：等股价跌破短期支撑后再次站回 + 量能确认后追
3. **放弃**：拒绝市价追入；列为观察清单

## 二、减仓 / 锁利节点

### 通用规则

- 第一目标到达 → **减仓 50%**，止损同步抬到入场价
- 第二目标到达 → 再减 30%，剩余 20% 用追踪止损（按 ATR14 或 MA10）

### 各模式退出节点

| 模式 | 第一止盈节点 | 第二止盈节点 | 持有底线 |
|---|---|---|---|
| 追涨 | `MAX(high_20d*1.08, close+1.8*risk, close+1.35*ATR14)` | 趋势线 / MA10 跌破 | R:R ≥1.6 |
| 超跌反弹 | 反弹到 MA20 即减 50% | 突破 MA50 再减 30% | 跌破 20 日低 *0.97 全清 |
| 回调买入 | 20 日前高 *1.05 | 突破历史阻力 +10% | 破 MA20 即清 |
| 价值挖坑 | 估值修复到 sector P/E 中位数 | 业绩出坑 / 行业复苏后突破历史阻力 | 不给硬止损；持仓 ≥3 月 |
| 财报跳空 | 财报后 +30% 锁 2/3 | 财报后 +50% 全清 | 财报后次日开盘 -5% 全清 |
| 叠加 A | 财报后 +30% 减 2/3 | 触发追涨第二止盈 | 财报后 -10% 全清 |
| 叠加 B | 财报后跳空 +15% 锁 2/3 | 20 日前高 *1.05 | 破 MA20 全清 |
| 叠加 C | 估值到 sector P/E 中位数 | 业绩拐点确认后 | 持仓 3-6 月，按价值退出 |

## 三、被套处理（用户已有仓位 + 浮亏）

### 步骤 1：先查三方认证

按 [web-cross-check.md](../core/web-cross-check.md) 三方认证模板填表，确认是"催化剂仍有效，只是市场短期波动" 还是 "thesis 已经破了"。

### 步骤 2：判断状态

| 状态 | 处理 |
|---|---|
| Thesis 仍有效 + 关键支撑未破 | 持有，按原计划止盈/止损 |
| Thesis 破了（催化剂证伪 / 财报失败 / 红旗触发） | 立即清仓，不等反抽 |
| 用户本金 >30% 集中在该票 + 浮亏 | 先给"账户风险情景概率表"（详见 [output-contract.md](../core/output-contract.md) 价格目标概率表）；劝降仓到 ≤20% 再讨论上涨 |

### 步骤 3：补仓门槛（极严格）

只有同时满足以下条件才允许讨论补仓：

- 主导偏向证伪检查（Step 3 三连校验）全部通过
- 无新增定增 / ATM / 反向拆股
- 价格守住关键支撑（前低 / MA50 / 关键回调位）
- 站回 VWAP / 当日开盘 / 昨收
- 量能回归（≥1.5x 20 日均量）
- 用户当前总仓位 ≤70%

任一不满足 → **不允许补仓**，仅讨论是否减仓 / 等下一波。

## 四、不要犯的错

- ❌ 第一止盈到达不减仓 → 利润回吐
- ❌ 跌破止损不止 → 等反抽
- ❌ 看到 +30% 就 hold 过财报（模式 5 严禁 hold 过 2 次财报）
- ❌ 用户被套时强推"补仓摊薄"
- ❌ 追涨模式里追阶段 4-5 的票（参见 [catalyst-stage-framework.md](catalyst-stage.md)）
- ❌ 价值模式给硬止损（破坏持仓周期）
- ❌ 财报跳空模式情绪化加仓（单票硬上限 2.5%）
