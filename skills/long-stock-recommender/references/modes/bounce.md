# 模式 2:超跌反弹

**场景**: 短期超卖,抛压衰竭,等反弹 | **持仓周期**: 3–10 天 | **预期收益**: +8-15%

## 触发条件

用户关键词:"超跌" / "反弹" / "抄底" / "跌多了" / "RSI 超卖"

## 流程

1. 调 `oversold_bounce_candidates()`,默认 3 档;如果用户只想看大盘股可以传 `price_buckets="100-500"`
2. 每个候选复核:
   - 跌的是不是有"接飞刀"风险(WebSearch 查 30 天内有没有重大利空 / 临床失败 / 业绩暴雷)
   - 财报窗口仍然要用 `earnings_dates` 查;财报前反弹不可靠
3. **持仓周期短**:强调反弹到 MA20 就减仓,不要当趋势股拿
4. 失效条件要明确:跌破 20 日低点 *0.97 立刻止损
5. **强制执行** [红旗复核](../workflow/red-flags.md)

## 输出必须包含

与追涨模式相同(见 [momentum.md](momentum.md))。
额外强调:**短持仓,反弹到 MA20 减仓**。

## 需要降级或排除

- 基本面恶化的"价值陷阱"(营收/利润连续下滑)
- 有重大利空未出尽(SEC 调查、集体诉讼、供应链断裂)
- 生科股临近关键临床读数
- 财报日 ≤ 7 天

## 输出要求

最终总表、单票短评、价格目标概率表、报价校验表、输出顺序与自检清单统一按 [../output-contract.md](../core/output-contract.md) 执行;本模式额外强调"短持仓,反弹到 MA20 减仓"。
