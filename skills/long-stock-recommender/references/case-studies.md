# 案例库（成功 / 失败复盘）

本文件汇总 long-stock-recommender 沉淀下来的真实案例，用于校准催化剂硬度、阶段判断、首推前三连校验、被套处理等规则。新增案例时按"事件 → 决策 → 结果 → 教训 → 已沉淀到哪条规则"五段写。

## 案例 1：TSEM —— 回调模式血泪教训（首推前三连校验的源头）

### 事件

2026/4/27 会话中，TSEM 处在回调模式候选 Top 1：
- 技术面：股价从 $52 回踩到 MA20 附近 $42，结构完美
- 量价：缩量回调，符合"洗盘"特征
- 表面无红旗

### 推荐时漏查的三项

| 项 | 实际 | 当时漏查的后果 |
|---|---|---|
| A. 历史财报反应 | 过去 4 次财报次日全跌（4/4） | "利好出尽型"未识别 |
| B. 估值 sanity | trailing PE 103，FCF 翻负 | "故事股定价"未标 |
| C. 管理层指引 | Q1 指引 $412M < Q4 实际 $440M（管理层亲口下调 6%） | "指引负面"未标 |

### 结果

财报后跳水。事后用户问"你为什么没看历史财报反应 / 没看 PE / 没看指引"。

### 沉淀到的规则

- [screening-workflow.md](workflow/causal-first.md) Step 3 "首推前三连校验" —— 强制对**最终首推**跑 A / B / C 三项校验
- 财报 ≤30 天 → 历史正反应 ≤1/4 直接排除（不限模式）
- PE > sector 中位数 3 倍 → 标"故事股定价"
- 管理层指引下调 → 标"指引负面，财报前必须空仓"

## 案例 6：CREG 2026-05-15 —— 当日可操作请求不能用日线价替代实时价

本案例用于区分 swing 日线候选和当日可交易推荐。long-stock 可以基于日线筛选 1 周-6 月候选,但用户问`今天 / 现在买 / 盘前 / 盘中`时,必须走当前报价校验。

事件:

- 本地日线显示 CREG `2026-05-14` 收盘约 `$0.7143`,盘中最高约 `$0.9099`
- 原回答错误地把昨日强势日线用于今日可执行计划
- 用户纠正: 券商当前看到 CREG 约 `$0.52`
- 用户价相对本地收盘价偏离约 `-27%`,原日线 entry 和风险收益全部失效

正确处理:

1. 如果请求是 swing 观察,CREG 最多只能写成`高风险日线候选`,并明确报价基准是最近完整日线。
2. 如果请求是今天/现在买,必须先查当前/延长交易报价;缺报价只能写`本次不是当前可交易推荐,只是日线候选`。
3. 用户提供 `$0.52` 后,必须用用户券商价重算入场、退出位、目标、仓位、股数和 R/R。
4. 若无法解释用户价与 Web/本地价差,标`报价冲突/只观察`,不得写`可做`。

沉淀规则:

- 普通 swing 日线计划和当日可操作计划必须显式分开。
- 日线收盘价相对当前/用户价偏离 `>=5%` 时,原 MCP/日线 entry、buy_zone、止损、止盈、股数、仓位和概率全部作废。
- 缺少当前报价时,措辞必须是`日线候选`,不能写`现在买`或`可做`。
