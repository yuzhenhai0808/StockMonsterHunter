# 日内 Web 交叉认证与逐步交易决策

本文件用于所有日内推荐、加仓、减仓、止损、复盘场景。目标是避免只听用户描述、只看本地日线、或只看单一网页源就给交易动作。报价要求严于 swing,因为日内决策必须建立在**当前可交易报价**之上,不允许沿用昨收。

## 三方认证硬门

每次给具体操作前,必须整理三类证据:

| 类别 | 必填字段 | 用途 |
|---|---|---|
| 用户信息 | 成本、仓位占比、已成交价、券商截图/盘口时间 | 作为最高优先级的实盘输入 |
| Web/公开源 | 最新价、时间戳、当前成交量/成交额、bid-ask 或 spread、最近成交连续性、日内高低点、新闻/SEC/IR、红旗 | 防止用户信息过期或误读,并判断能否实际成交 |
| 本地数据 | 前收、ATR、量比、20 日位置、财报窗口、基本面/short float | 判断结构、波动和风险边界 |

## 当前可交易性硬门

本地 `latest_prices` / `symbol_snapshot` 只代表最近一个完整日线,**不能当作盘中实时可交易报价**。凡是用户问"当下可以入 / 盘中 / 现在买 / 最新价",最终候选必须先通过当前可交易性检查。

### 当前流动性硬门

日内推荐必须先过**当前可交易流动性**硬门。`volume_ratio`、20 日均成交额、昨日爆量和本地 SQL 成交额只能证明历史活跃,不能替代当前盘前/盘中能否成交。

| 交易阶段 | 主攻/备攻最低门槛 | 只允许极小仓触发 | 观察/拒绝 |
|---|---|---|---|
| 盘前/夜盘/盘后 | 当前延长交易成交量 `>=300k` 股 **或** 成交额 `>= $1M`; 最近成交 `<=5` 分钟; bid-ask spread `<=2%`; 成交不是单笔孤岛 | 当前成交量 `100k-300k` 股 **或** 成交额 `$300k-$1M`; spread `<=3%`; 只能账户风险 `0.25%-0.5%` | 低于 `100k` 股且低于 `$300k`,或几分钟无成交,或 spread `>3%` |
| 09:30-09:45 ET | 默认观察;除非成交额已 `>= $3M` 且站稳开盘价/VWAP | 可极小仓二突,但必须成交连续 | 开盘 5 分钟内市价追单一律拒绝 |
| 09:45 ET 后 | 当前日成交额 `>= $5M` 且最近 5 分钟有连续成交; spread `<=2%` | 成交额 `$1M-$5M` 只能小仓战术 | 低于 `$1M` 或成交断续只能观察 |

关键规则:

- **相对量不能替代绝对量**: 20 日均量很低导致 `volume_ratio >100x`,但盘前实际成交只有几万股时,不得升主攻/备攻。
- **单源报价不能自动升 B**: 若 quote 页面只有价格和涨跌幅,没有当前成交量/成交额或 bid-ask,最多 C 级观察。
- **低流通不是流动性**: float 很小只说明弹性大,也说明滑点和停牌风险更高;必须用当前成交额和点差证明能进能出。

### 证据等级闸门

每只候选必须先标注报价证据等级,再决定能否推荐:

| 等级 | 条件 | 允许输出 |
|---|---|---|
| A 实时/夜盘可交易 | 用户券商实时报价,或两个公开源给出当前/延长交易报价,且有价格、时间戳、当前成交量/成交额和 bid/ask;同时通过当前流动性硬门 | 主攻/备攻、入场、仓位、正常概率表 |
| B 单源延长交易 | 一个公开源给出 after-hours / overnight / premarket 报价,且有明确时间戳、当前成交量/成交额和 bid/ask;同时通过当前流动性硬门的主攻/备攻最低门槛 | 可给条件主攻/备攻;必须标注单源报价,开盘前复核 |
| C 最近完整日线 + 新闻 | 只有本地最近完整日线和新闻催化 | 只能列为`观察/待盘前报价`;若给情景概率必须写成"若盘前站上...才触发" |
| D 报价冲突/过期 | 公开源日期过旧、源之间冲突、只有昨收、无成交、无时间戳或无法解释价差 | 不给入场、仓位或概率;只能写`数据不完整/报价冲突` |

只有 A/B 且通过当前流动性硬门才允许 `主攻/备攻`、具体入场计划和正常概率表;C 只能列为`观察/待盘前报价`,不得给"当前可买"结论;D 只能列入`数据不完整/报价冲突`。

### 报价获取顺序

固定不可调换:

1. 先查当前/延长交易 movers 榜单与候选 ticker 的 quote 页面(Nasdaq / MarketWatch / ADVFN / StockAnalysis / Stocknear / 券商可见报价)
2. 再查新闻催化(IR / SEC 8-K / 主流财经媒体)
3. 最后才用本地日线做技术计划

不能先基于日线价给入场表再补实时价。
拿到 movers / top gainers 榜单后,必须立刻对最终候选逐票打开 quote 页面或使用用户券商盘口,获取**当前时间点最新可交易价**;榜单价格只允许写作`候选发现价`,不得写作`最新报价基准`。

### Movers 快照不得当作最新成交价

对盘前/盘中涨幅榜、极端彩票、低价股和用户已持仓票,`movers / top gainers / scanner / Public榜单 / Reddit转贴`只能用于发现候选和涨幅背景。最终交易基准必须来自以下之一:

1. 用户券商实时价或截图时间戳。
2. 可打开的 quote 页面,同时有价格、更新时间、成交量或 bid/ask。
3. 两个公开 quote 源在同一交易阶段内价格差 `<3%`。

若只能拿到榜单快照、15 分钟延迟价、盘前旧时间点或无成交量/bid-ask 的 quote,证据等级最多 C,必须写`当前 quote 未确认;仅有 movers 发现价`,并降为`观察/待实时复核`;不得给主攻/备攻、仓位或正常概率。

### 延迟榜单触发价硬阻断

若报价来源标注 `15 分钟延迟 / delayed / scanner snapshot / movers snapshot`,该价格只能用于`发现候选`和`事后复盘`,不得直接生成入场触发价、止损、止盈、仓位或概率。即使该票后来实际大涨,只要输出时没有拿到当前 quote 页、用户券商盘口或双公开源互证,原推荐在流程复盘中必须标为`条件未成立 / 报价证据违规`,不能算作完整命中。

处理:

- 允许写: `若用户券商实时价确认仍在 $X 上方且成交连续,再按二突/回踩重算`。
- 不允许写: `站上 $X 就做 / 硬止损 $Y / 第一止盈 $Z` 这类基于延迟榜单价的完整交易计划。
- 若因延迟榜单给出了主攻/备攻或具体计划,复盘时必须记录为 skill 违规样本,并沉淀到 [../case-studies.md](../case-studies.md)。

### 当前夜盘价优先

凡是用户问`今天 / 今日 / 盘前 / 夜盘 / 开盘`这类需要当天执行的结论,必须先尝试获取**当前 overnight / premarket / after-hours 可见报价**:

- 若处于官方盘前前、overnight 或盘前早段,必须先尝试抓当前 overnight/盘前可见价
- 拿不到当前夜盘价时才退回最近 after-hours,并标注"不是当前盘前/夜盘"
- 旧 after-hours 或昨收只能用于`二日延续观察`,不得直接作为入场、止损、止盈和概率基准
- 若用户提供券商夜盘价,立刻用该价重算所有计划

### 最新报价驱动决策状态机

拿到 Web/用户券商最新价后,必须把最新价转成新的交易状态;不能只把报价放进校验表。用户问`现在入不入 / 还追不追 / 盘前怎么做`时,输出必须先给`最新价状态判定`。

判定顺序:

1. `最新价 <= 旧硬止损 / 结构失效价`:旧计划作废;未持仓写`不入,等 reclaim`;已持仓切换止损/被套处置。
2. `最新价在旧入场区内,且仍守住 VWAP/盘前承接区`:可保留或微调计划,但必须用最新价重算止损、止盈、R/R、仓位和计划股数。
3. `最新价低于旧入场区但高于旧止损`:先判定健康回踩 vs 承接失败。只有守住 VWAP/盘前低点/关键平台并出现成交回流,才可改为`回踩做`;否则只观察。
4. `最新价高于旧入场上限`:旧入场错过;改成`等回踩`或`二次突破`,重新给触发价和失效价。
5. `最新价触及旧第一止盈`:不得再按首轮入场计划追;只能给二轮计划,且仓位降一档。

示例:`ONDS 最新 $10.85 = 旧止损位 → 旧计划作废;现在不入,只等 $11.25 reclaim 或 $11.75 二突后重算。`

### 集合竞价/盘前限价单

夜盘 / 盘前做 T 补充规则:

- 只用限价单,不使用市价单
- 必须检查点差是否过宽;若买卖价差超过 `2%-3%`,默认降低仓位或放弃
- 必须检查盘口成交是否连续;若几分钟无成交,不要用日线成交额假装流动性充足
- 夜盘/盘前第一目标可以低于常规盘,通常 `+6%-12%` 就应兑现一半以上
- 若价格跌破盘前 VWAP 或关键整数位后反抽不过,停止同票继续做 T
- 夜盘盈利不能自动推导常规盘还能继续做;开盘后必须重新判断量能、VWAP 和前高 reclaim

### 盘后异动池

凡是用户问`极端 / 快速盈利 / 高收益 / 盘前怎么配 / 今天能不能冲`,除普通涨幅榜外必须单独抓`盘后公告/财报异动池`: 最近一次收盘后至当前盘前因 `earnings / 8-K / guidance / contract / FDA / partnership` 跳涨的 Top 1-5。

即使本地最近完整日线显示该票当日收跌、收在低点、或不在日线涨幅榜,只要满足以下任一条件,都必须进入审计池并标注`盘后点火/次日延续`:

- 盘后涨幅 `>= 20%`
- 盘后成交额 `>= $5M`
- 公开榜单列入盘后 Top 3

盘后点火票若最终未推荐,必须在备注说明具体原因(`报价不可复核 / 点差过宽 / 成交不连续 / 催化不足 / R/R 不够 / 开盘承接失败`),不得因为日线涨跌幅为负而静默删除。

## 来源优先级与冲突解决

| 优先级 | 来源 |
|---|---|
| 1(最高) | 用户提供的券商盘口报价(含时间戳) |
| 2 | 公司 IR / 官方 PR / SEC 8-K |
| 3 | Nasdaq / MarketWatch / Yahoo / Stocknear / StockAnalysis |
| 4 | 本地日线收盘价 |

冲突时上层覆盖下层。具体规则:

- 用户券商盘口优先用于交易价格,但必须标注`用户券商信息,公开源不一致`
- Web/公开源优先用于新闻、SEC、融资/ATM/反拆红旗
- 本地日线只能做背景,不能替代盘中报价或新闻
- 若公开源之间价差或涨跌幅冲突 `>=3%`,该票只能标`报价冲突/只观察`,不得给入场、加仓或主攻
- 用户报价与网页报价冲突但无法解释时间戳时,只能给两套情景,并明确哪套以用户券商盘口为准

## 用户报价纠错触发规则

用户说`我看到价格是 X`、`你价格不对`、`现在不是这个价`、`券商显示 X`、`已经跌到/涨到 X`时,必须立即停止沿用原推荐:

1. 承认原报价基准失效,把用户券商价作为最高优先级交易基准。
2. 用用户价重算入场、硬止损、第一/第二止盈、建议仓位上限、最大亏损、计划股数和 R/R。
3. 若实时/延长报价或用户价相对本地日线收盘价偏离 `>=5%`,原 MCP/日线生成的 entry、buy zone、止损、止盈、仓位和概率全部作废,不能只在备注中说明。
4. 若缺少成交量、bid/ask、时间戳或 Web 复核,只能输出`修正版状态表`,不得继续保留原主攻/备攻结论。
5. 若用户价与 Web/本地偏离 `>=5%` 且无法解释为不同交易时段,该票降为`报价冲突/只观察`。

### 当前价触发旧止损的硬门

若用户券商价或可复核实时/延长报价已经 `<=` 上一版计划的硬止损、降仓触发价或结构失效价,旧入场计划必须立即作废:

- 用户未持仓:结论先写`不入/旧计划已失效`;取消旧限价或旧触发单。
- 用户已持仓:切换到 [../workflow/exit-framework.md](../workflow/exit-framework.md) 的被套/止损处置,不得继续按旧入场计划分析。
- 不得把旧止损价改口成新低吸价。只有价格重新站回 VWAP / 开盘价 / 盘前承接区 / 前高 reclaim 之一,且基于当前价重算 R/R 后,才允许给新的入场触发。
- 若缺少成交量、bid/ask 或分时结构,只能写`观察,等 reclaim`,不得给主攻/备攻。

## 报价校验表

最终候选总表后必须加报价校验:

```md
| 票 | 昨收 | 最新价 | 报价时间 | 来源 | 成交量/Bid-Ask | 相对昨收 | 可交易性 |
|---|---:|---:|---|---|---|---:|---|
| TICKER | $X.XX | $X.XX | HH:MM ET | 用户券商/StockAnalysis/Nasdaq/... | X股 / $X-$Y | +X.X% | A/B/C/D |
```

若实时/延长报价与日线收盘价偏离 `>=5%`,必须基于实时/延长报价重算交易计划。若入场上限被跳空越过,参与方式必须改成`回踩做`或`二次突破做`,不能保留旧"入场上限"。

## 候选池报价矩阵

对进入最终 Top 3 的每只票,在输出前内部整理:

```
日线收盘价 / 当前或延长交易价 / 报价时间 / 报价来源 / 相对收盘涨跌 / 可交易性结论
```

缺少当前或延长交易价的票只能列为`观察/数据不完整`。

## 时间段显式化

输出必须写明当前所处阶段和判断基准:

- `官方盘前前 / 盘前 / 盘中 / 盘后 / overnight`
- 当前纽约时间或报价源给出的 ET 时间

若官方盘前尚未开始,不得把上一日 after-hours 报价说成"当前盘前";只能说"最近可见延长交易报价"。若 Web 市场时钟与本地环境时钟冲突,以 Web 市场时钟和交易所交易状态为准,并在输出中说明。

```
"{ticker} stock news today"
"{ticker} catalyst today"
"{ticker} 8-K OR offering OR ATM OR reverse split last 7 days"
```

### 涨幅榜 / 异动池

```
"top gainers stocks today {date}"
"premarket gainers {date}"
"stocks moving today {date}"
"unusual volume movers"
"after hours top gainers {date}"
```

### 财报相关

```
"{ticker} earnings date investor relations"
"{ticker} reports results {current_month} {current_year}"
"{ticker} q1 q2 results date site:globenewswire.com OR site:businesswire.com OR site:prnewswire.com"
```

```

```

```

```

## 防用户信息误导

- 用户说"跌很多 / 涨很多 / 能到某价"时,不要直接采用结论;先核对价格、时间段和成交量
- 用户截图必须读出截图时间和交易阶段:盘中、盘后、夜盘、昨日收盘
- 用户成本、仓位、卖出价不完整时,先复述假设并给可替换公式
- 如果用户信息与公开源冲突,给两套情景,并明确哪套以用户券商盘口为准

## 当 Web 无法访问

- 必须明确告知用户:"本次未查社会面 / 报价,建议自行复核"
- 不允许"K 线漂亮就推荐"
- 概率必须打宽,不允许写得过窄
