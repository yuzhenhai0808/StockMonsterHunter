# 日内输出合约

本文件是 `day-trading-recommender` 的全局输出合同。凡是给出日内做多候选、当日可操作推荐、盘前 gap、财报跳空首日、轧空、龙头追击、微盘爆拉,必须先满足这里的格式和自检要求。模式文件只补充筛选逻辑,不得覆盖本文件的最终输出字段。

## 默认输出顺序

1. `当前时段与报价基准`: 写明交易阶段(盘前 / 开盘观察期 / 盘中 / 尾盘)、纽约时间或报价源 ET 时间、本地日线日期、是否为实时/延长交易报价;若价格来自 movers / top gainers / scanner 榜单,必须标注`仅候选发现价,非当前 quote`
2. `候选总表`: 推荐 2-3 只时必给;先于单票展开
3. `单票展开`: 默认只展开前 1-2 只主攻,其余只保留表格行
4. `报价校验`: 每只最终候选写最新可见报价、时间、来源;若没有 quote 页面、两个公开 quote 源互证或用户券商价,只能标`观察/待实时复核`
5. `放弃/降级备注`: 热门票/盘后异动池中未推荐的票必须写一句原因

若回答缺少 `候选总表`(2 只及以上时)、`单票展开` 必填字段、`报价校验` 任一项,必须补齐后再输出。若数据不足无法补齐,该票只能标为`观察/数据不完整`,不得给主攻、入场、仓位或概率。

## 证据等级与概率

| 等级 | 定义 | 允许输出 |
|---|---|---|
| A 实时/夜盘可交易 | 用户券商实时报价,或两个公开源给出当前/延长交易报价,且有价格、时间戳、成交量或 bid/ask | 主攻/备攻、入场、仓位、正常概率表 |
| B 单源延长交易 | 一个公开源给出 after-hours / overnight / premarket 报价,且有明确时间戳和成交量或 bid/ask | 可给条件主攻/备攻;必须标注单源报价,开盘前复核 |
| C 最近完整日线 + 新闻 | 只有本地最近完整日线和新闻催化,没有当前/延长交易报价 | 只能列为`观察/待盘前报价`;若给情景概率必须写成"若盘前站上...才触发" |
| D 报价冲突/过期 | 公开源日期过旧、源之间冲突、只有昨收、无成交、无时间戳或价差无法解释 | 不给入场、仓位或概率,只能写`数据不完整/报价冲突` |

概率是交易路径校准,不是收益保证。必须用区间表达,例如 `35%-45%`。A/B 级必须给概率区间;C 级只能给`若站上/回踩确认后的粗略情景概率`;D 级写`报价不足,不估概率`。

概率必须以`计划入场价`为起点,衡量价格先触及`第一止盈`还是先触及`硬止损`,不要用"当天可能触达目标"替代触发后的路径概率。估算依据至少引用 3 项: `证据等级 / 当前或延长报价来源 / ATR14% / 当日或盘前量比 / 距 VWAP 或关键高低点 / 催化硬度 / 是否第 2-3 天 / 大盘与板块方向`。

默认概率分档: `>=60%` 高概率但通常肉薄, `40%-60%` 中性可交易, `20%-40%` 赔率型, `<20%` 彩票/尾部情景。

## 候选总表(强制 12 列)

```md
| 票 | 机会类型 | 阶段判断 | 参与方式 | 入场上限 | 止损 | 第一止盈 | 先到止盈 | 先到止损 | 建议仓位上限 | 催化 | 一句话 |
|---|---|---|---|---:|---:|---:|---:|---:|---:|---|---|
| TICKER | 主攻/备攻/防守备选/极端彩票 | 初启动/扩张中/尾声/不可做 | 回踩做/突破做/只观察/放弃 | $X.XX | $X.XX | $X.XX | XX%-XX% | XX%-XX% | $XXX (总资金 X%) | 财报/回购/订单/叙事 | <一句结论> |
```

字段要求:

- `机会类型`: 必须是 `主攻 / 备攻 / 防守备选 / 极端彩票` 之一,不允许自造
- `阶段判断`: 必须是 `初启动 / 扩张中 / 尾声 / 不可做` 之一
- `参与方式`: 必须是 `回踩做 / 突破做 / 只观察 / 放弃` 之一
- `入场上限 / 止损 / 第一止盈 / 先到止盈 / 先到止损 / 建议仓位上限`: 全部必填
- `催化`: 只写 2-6 字,不能写完整句子
- `一句话`: 只写执行结论,例 `主攻可做` / `榜首但不能做` / `高弹性但已尾声` / `观察等回踩`
- 没有 A/B 证据等级的票不允许出现在主攻/备攻位置;只能列 `观察/数据不完整`

## 单票展开(强制字段,12-18 行)

每只展开候选必须含以下字段,缺一行视为不合规:

```md
候选 N: {TICKER} ({公司名简称})
机会类型: {主攻 / 备攻 / 防守备选 / 极端彩票}
阶段判断: {初启动 / 扩张中 / 尾声 / 不可做} {若满足则补 ⚡轧空可能}

当前时段
- 交易阶段: 盘前 / 开盘观察期 / 盘中 / 尾盘
- 现在该做什么: 挂条件单 / 等回踩确认 / 等二次突破 / 只减仓不加仓 / 不开新仓

交易计划
- 当前价 / 前收盘 / 当日变动: $X.XX / $X.XX / +X.X%
- 参与方式: 回踩做 / 突破做 / 只观察 / 放弃
- 入场上限价: $X.XX
- 硬止损价: $X.XX (-X.X%)
- 第一止盈: $X.XX (+X.X%)
- 第二止盈: $X.XX (+X.X%)
- 概率校准: 入场后先到第一止盈 XX%-XX%; 入场后先到止损 XX%-XX%; 第二止盈延展 XX%-XX%
- 最晚平仓: 今日 14:45 ET
- 建议仓位上限: $XXX (总资金的 X%)
- 计划买入股数: X 股 (按入场上限价估算)

为什么看它
- 它为什么可能走出 15%-50% 波动: <一句话>
- 主导偏向: <一句话,只能写一个主驱动>
- 现在参与还来不来得及: <一句话>
- 如果错了为什么不至于深套: <一句话>
- 主催化: <一句话,带 WebSearch URL>
- 数据时间: 日线 {latest_date}; 最新报价 {quote_time}

风险检查

- 资本事件复核: 已查, 过去 7 天无红旗 / 触发 X 红旗
- 概率依据: 证据等级 X; ATR14% / 量比 / 关键价位 / 催化硬度

一句话结论
- <主攻可做 / 只适合低仓进攻 / 榜首但不能做 / 高弹性但已尾声>
```

## 报价校验

候选总表下方必须加报价校验行/表:

```md
| 票 | 昨收 | 最新价 | 报价时间 | 来源 | 成交量/Bid-Ask | 相对昨收 | 可交易性 |
|---|---:|---:|---|---|---|---:|---|
| TICKER | $X.XX | $X.XX | HH:MM ET | 用户券商/StockAnalysis/Nasdaq/... | X股 / $X-$Y | +X.X% | A/B/C/D |
```

规则:

- 报价获取顺序固定:先查当前/延长交易 movers 榜单与候选 ticker 的 quote 页面 → 再查新闻催化 → 最后才用本地日线做技术计划
- 用户提供的券商报价优先级最高;给出后必须用该价重算入场、止损、止盈、仓位和 R/R,并标注"用户券商报价,需以盘口为准"
- 若实时/延长报价与日线收盘价偏离 `>=5%`,原 MCP/日线生成的 entry、buy zone、入场上限、止损、止盈、仓位、计划股数、概率和 R/R 全部作废;必须基于实时/延长报价重算交易计划,不能只在备注中说明价格变化
- 若入场上限被跳空越过,参与方式必须改成`回踩做`或`二次突破做`,不得保留旧入场上限
- 每次给出当前操作建议前,必须追加`最新价状态判定`,按 [web-cross-check.md §最新报价驱动决策状态机](web-cross-check.md) 把最新价映射为`旧计划有效 / 旧计划作废 / 等回踩 / 等二突 / 只观察 / 持仓处置`;不得只展示最新价而不调整策略
- 若公开源之间价差或涨跌幅冲突 `>=3%`,或用户报价与网页报价冲突但无法解释时间戳,该票只能标`报价冲突/只观察`
- 拿不到当前或延长交易报价 → 该票只能列为`观察/待盘前复核`
- 若最新报价来自 `15 分钟延迟 movers / scanner / top gainers` 且没有 quote 页面、用户券商盘口或双公开源互证,该票只能列为`观察/待实时复核`;不得填写入场上限、止损、止盈、仓位和正常概率。可以写条件触发语句,但必须以`用户券商实时价确认后重算`为前提。

用户指出价格错误时,必须先撤销旧结论并输出修正版状态: 原基准价、用户/最新价、偏离幅度、原计划是否作废、新证据等级、修正结论。若无法用用户价重算,不得继续保留原主攻/备攻、建议仓位或概率。

## 单票数量控制

- 推荐 1 只时:可以只给单票展开,不必强行表格
- 推荐 2-3 只时:必须先给候选总表 + 报价校验,再展开前 1-2 只
- 推荐 >3 只时:总表 + 只展开前 2 只,其余只保留表格行;一定写每只为何在 Top 3 之外

## 时间上下文自适应

输出时必须先判断**当前美东时间**处于哪个交易阶段,再决定怎么写操作建议。不要把盘前分析模板套用到盘中或尾盘。

- 盘前(< 09:30 ET):可以写盘前涨跌幅、盘前挂条件单、盘前高低点;给"开盘后 09:45 ET 再确认"的计划
- 开盘观察期(09:30-09:45 ET):不写"立即追入";优先写"等 VWAP / 首个 5 分钟区间确认"
- 盘中(09:45-14:00 ET):默认按盘中逻辑写,改成"回踩确认 / 突破确认 / 现价附近是否还能做";盘前数据只作背景
- 尾盘(14:00-14:45 ET):默认不开新仓;重点写减仓、锁盈、按时平仓
- 14:45 ET 之后:不再给日内新开仓建议;只允许写"今天放弃 / 等明天"

必须写清楚数据日期:日线 `{latest_date}`、最新报价 `{quote_time}`。

## 放弃/降级备注

热门票、涨幅榜 Top 1-3、本地防漏池 Top 10、用户点名票、盘后异动池若未进入最终主攻/备攻,必须用一句话说明原因。原因只能从这些类型中选择或组合:

`报价源冲突 / 已停牌 / 红旗过重 / 已尾声 / R/R 不够 / 催化不足 / 量能未放 / 报价不可复核 / 入场位置变差 / 流动性不够 / 开盘承接失败`

## 最终自检清单

输出前逐项检查:

1. 是否写明当前交易阶段、报价基准与日线日期
2. 推荐 2 只及以上时是否先给候选总表
3. 候选总表是否包含强制 12 列,字段值是否在标准集合内
4. 每只主攻/备攻是否完成单票展开,且 12-18 行内包含全部必填字段
5. A/B/C/D 证据等级是否决定了能否给概率和仓位
6. 是否同时写了建议仓位上限和最大亏损
7. 是否有报价校验表
8. 是否解释了热门但放弃/降级的票
9. 主导偏向是否只写一个驱动,而不是堆叠多个利多
10. 概率是否全部区间表达,无单点
11. 若用户纠正报价,是否撤销旧计划并用用户券商价重算;无法重算时是否降为`报价冲突/只观察`
12. 若报价源是延迟 movers/scanner,是否已阻断主攻/备攻、入场价、止损、止盈、仓位和正常概率
