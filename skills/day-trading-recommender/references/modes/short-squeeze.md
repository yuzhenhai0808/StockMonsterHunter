# 模式 5:轧空猎手

**场景**: 寻找高空头 + 低流通 + 技术启动中的股,博空头回补引发的加速上涨 | **持仓**: 开盘 30 分钟 - 收盘 | **目标**: +10~20% | **胜率**: ~40%

## 触发条件

用户关键词:"轧空" / "squeeze" / "空头挤压" / "逼空" / "短头回补"

**另外**,默认 premarket-gap 模式在输出任何候选时,会**自动检查轧空条件**并用 ⚡ 标记,不需要用户单独触发本模式。

## 流程

### Step 1 - 本地数据库初筛

Read [scripts/squeeze_candidates.sql](../../scripts/squeeze_candidates.sql),通过 `query_market_data` 执行。SQL 会筛:
- 空头占流通 ≥ 15%
- 流通股 ≤ 2000 万
- 近 5 日涨幅 > 10%(已启动)
- 市值 $1 亿 - $50 亿(避开超微盘和大盘)

### Step 2 - 短期挤压动能验证

对每只调 `symbol_snapshot`:
- `volume_ratio` ≥ 2.0 → 🟢 成交量已放大
- RSI14 在 60-75 → 🟢 强但未超买
- `close > ma20 > ma50` → 🟢 三均线多头排列
- `short_ratio`(days-to-cover)≥ 3 → 🟢 回补需要多日,更容易轧

### Step 3 - 催化剂真实性

WebSearch:
```
"{ticker} short squeeze news today catalyst"
"{ticker} reddit wallstreetbets mentions"
"{ticker} analyst upgrade last 7 days"
```

- 有具体利好(升级/订单/业绩) → 🟢 真实轧空
- 只有社交媒体炒作 → 🟡 meme 轧空,仓位减半
- 无任何催化 → 🔴 技术性反弹,非轧空,归其他模式

### Step 4 - 空头覆盖成本分析

WebSearch:
```
"{ticker} borrow fee cost to short"
```

借券费率(cost-to-borrow):
- 费率 > 50% 年化 → 🟢 空头已痛苦,回补压力大
- 费率 < 10% → 🔴 空头舒服,不急于回补,轧空动能弱

### Step 5 - R/R 计算

轧空股 ATR 通常 5-10%,用 -5% 止损:

- 入场上限价 = 现价(若盘前已大涨,等开盘后 15 分钟再决定)
- 硬止损 = 入场 × 0.95(-5%)
- 第一止盈 = 入场 × 1.10(+10%)
- 第二止盈 = 入场 × 1.18(+18%)
- R/R = 10/5 = **2.0** ✅

**盘中风控**:开盘 30 分钟若未破第一止盈,减仓 50%;未到中午未破第二止盈,全清。

### Step 6 - 输出

按 [../output-contract.md](../core/output-contract.md) 输出模板。

## 输出必须包含

(在 [../output-contract.md](../core/output-contract.md) 输出模板基础上,补充本模式特有字段)

**本模式特有字段**:
- 空头占流通 %
- 流通股(千万股)
- Days-to-cover
- 借券费率(若查到)
- 近 5 日涨幅 %
- 催化真实性等级

## 需要降级或排除

- 已经 3 天涨 > 80%:轧空主力已走,追高风险高,**排除**
- 空头 < 10%:不是轧空结构,归其他模式
- 流通 > 5000万:大流通,轧空效应被稀释,降级
- 财报 ≤ 5 天:财报可能直接决定方向,轧空信号被覆盖,**排除**
- 生科轧空:临床数据是真实催化,归财报/公告模式

## 轧空 vs 微盘 pump 的区别

容易混淆,但**本质不同**:

| | 轧空(本模式) | 微盘 pump(模式 2) |
|---|---|---|
| 市值区间 | $1 亿 - $50 亿 | < $1 亿 |
| 催化 | 真实利好 + 机构空头被轧 | 叙事 + 社交媒体拉盘 |
| 空头比例 | ≥ 15% | 通常 < 10%(pump 股没人空) |
| 胜率 | ~40%(有基本面) | ~35%(纯博弈) |
| 仓位 | 可达资金 3-5% | ≤ 1% |

**判错的代价**: 把 pump 股当轧空做,会用标准仓位而非彩票金,爆仓风险放大。
