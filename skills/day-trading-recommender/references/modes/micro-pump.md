# 模式 2:微盘爆拉跟风(HTCO/YAAS 型)

**场景**: 用户明确说"类似 HTCO/暴拉/仙股/pump"时的专用模式 | **持仓**: 5-60 分钟 | **目标**: +8~15%,少数极端情况更高 | **胜率**: ~35%(赔率主导,但很多时候根本不该做)

## 重要警告(写在最前面)

本模式覆盖的是**极端投机 / 纯 pump** 场景。它保留下来,但只负责最激进的尾端机会,**不再吞掉所有高波动票**。

也就是说:
- 高波动但有真实催化、有继续扩张结构的票,应先归默认模式 / squeeze / gap / breakout
- 只有当它更像 pump-and-dump 博弈时,才进入本模式

本模式不会用"推荐"措辞,只会用"如果你一定要参与,这样做更安全"。但对于 WOK/YAAS/HTCO 这类当天涨幅榜极端彩票,**不能因为红旗静默过滤**;必须单独展示为`极端彩票榜首池`,再说明不适合重仓。

**强制硬线**:
- 单笔仓位 ≤ **总资金的 1%**(通常 $50-$200 之间)
- 不过夜,**14:45 ET 必清仓**,无论盈亏
- 连亏 2 次本模式的交易 → 当日禁用本模式,冷静期 24 小时

## 触发条件

用户关键词:"微盘" / "暴拉" / "HTCO 那种" / "翻倍" / "仙股" / "pump" / "赚快钱" / "3 天涨 200%"

## 流程

### Step 0 - 情绪评估

扫描最近 10 条用户消息,如果包含:
- "我亏了" / "翻本" / "捡回" / "亏麻了"
- 一天内问 >3 次日内股票
- 上次建议被拒后情绪化追问

→ **拒绝推荐本模式**。改推其他高赔率模式或劝用户今日空仓。

但仍需输出`极端彩票榜首池`的审计表,让用户知道当天最猛的是哪些票,并把参与结论写成`观察/禁止参与/极小仓触发`,不得正常推荐。

### Step 1 - 本地数据库初筛

Read [scripts/micro_momentum.sql](../../scripts/micro_momentum.sql),通过 `query_market_data` 执行。SQL 会筛:
- 市值 $1000 万 - $5 亿
- 3 日涨幅 > 50% 或 5 日涨幅 > 80%
- 20 日均成交量 ≥ 10 万股
- 当日收盘 ≥ $1

### Step 1B - 极端彩票防漏扫描

额外扫描或从 Web movers 中记录:

- 当日/当前涨幅 `>=80%`
- 价格 `$0.1-$10`
- 当日成交额 `>= $10M`
- 相对量 `>=50x`
- 日内振幅 `>=80%`

这类候选即使市值太小、反拆、融资或无硬催化,也必须进入`极端彩票榜首池`展示;只是不进入正常推荐池。

### Step 2 - 对每只候选查资本事件红旗

WebSearch:
```text
"{ticker} offering OR ATM OR reverse split OR dilution OR 8-K"
```

触发任一 → 归入"超高危"级,仓位硬砍到 ≤ $50。

### Step 3 - 对每只候选查 pump 识别表

见 [../workflow/red-flags.md](../workflow/red-flags.md),按下列维度打分:
- 52W 高 / 现价 > 10 → 有合股历史(+2 分)
- 流通股 < 500 万 → 极低 float,易拉易砸(+2 分)
- 空头 < 5% → 无空头子弹,纯散户博弈(+1 分)
- 营收 < $500 万 / 年 → 无基本面支撑(+2 分)
- 股东大会 / 投票公告 7 天内 → 有资本重组诱因(+2 分)

总分越高,越偏纯 pump。

### Step 4 - 盘中 Level-1 数据查询

WebSearch:
```text
"{ticker} stock price today intraday chart"
```

拿到现价和今日涨跌幅,计算:
- 距今日高点的回撤 %
- 距启动日收盘价的涨幅 %
- 是否已经第 3 天尾声

### Step 5 - R/R 计算(严格 -5% 硬止损)

- 入场上限价 = 现价或合理回踩位
- 硬止损 = 入场 × 0.95 (-5%)
- 第一止盈 = 入场 × 1.08 (+8%)
- 第二止盈 = 入场 × 1.15 (+15%)
- R/R = 1.6 起跳

如果现价已经是今日高点附近(距高 < 2%)、无回踩结构,改成观察模式不入场。

### Step 6 - 输出

按 [../output-contract.md](../core/output-contract.md) 的输出模板 + 本模式的额外强制字段。

## 输出必须包含

- **极端彩票榜首池**: Top 1-3 最猛票,即使全部不推荐
- **Pump 风险分**
- **仓位上限**
- **最坏情况**:"如果 pump 崩盘,你最多损失 $X"
- **退出纪律**:盘中不设第二次加仓,跌破止损立即全出

## 需要降级或排除

- 当日已经是第 3 天 +50%:泡沫尾声,**排除**
- 市值 < $1000 万:过于 microcap,券商可能拒单,排除
- 最近一周有过 > 50% 单日跌幅:pump 已崩过,二次 pump 风险更高,排除
- 生科微盘:二元临床风险 × pump 风险 = 双重爆炸,**一律排除**

## 职责边界

- 本模式只负责**极端投机/纯 pump**
- 不要把所有高波动票都塞进本模式
- 如果某只票有真实催化、量能扩张、仍有继续扩张空间,优先交给默认模式、gap、squeeze 或 breakout

## 输出要求

最终候选总表、单票展开、报价校验、概率与最终自检统一按 [../output-contract.md](../core/output-contract.md) 执行;本模式额外要求显式写出微盘 Pump 总分(见 [../red-flags.md](../workflow/red-flags.md))、仓位上限和"如果 pump 崩盘最多损失 $X"的最坏情况。
