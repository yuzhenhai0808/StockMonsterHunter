# 模式 0:高赔率进攻雷达(默认)

**场景**: 用户要"今天盘中买卖/今日有什么/短线机会",但未指定单一模式。  
**持仓**: 开盘后 5 分钟 - 14:45 ET 前清仓。  
**目标**: 优先找**单次能打出大肉**的高赔率机会,而不是最稳的票。

## 核心原则

默认同时扫描多种能让股价当天走出高弹性的催化:

- 事前催化雷达:尚未大涨但未来 `1-10` 个交易日有明确事件窗口、同业传导或预期差
- 财报前预判埋伏: 提前 `3-10` 天布局潜在超预期和指引上修
- 财报后首日延续:业绩、指引、回购、margin 改善
- 分析师上调/新覆盖:容易触发中小盘单日扩张
- 订单/合作/并购/回购/监管批准:真实公告优先
- 控制权交易/救命融资:具名买方、固定每股价格、交易规模和 8-K/公司公告确认时,按事件锚价重估处理
- 行业/政策资金:AI、核能、国防、关税、降息、板块轮动
- 强势突破:20 日/52 周新高附近,放量加速
- 轧空:高 short interest + 低 float + 已启动 + 有真实催化
- 涨幅榜龙头追击:只在结构对时参与,不是见第一就追
- 微盘爆拉:只在不是纯 pump、仍有赔率时纳入,否则降到极端彩票

**默认不再把低波动稳健票放在主推位。**
如果高流动性大票只能给 +5% 到 +8% 的温和空间,而另一只中小盘有明确催化、量能扩张、能争取 +15% 到 +30%,默认优先后者。

## 流程

### Step 0 - 数据新鲜度

调 `db_health`:

- `yfinance.latest_date` 是今日或昨日:可用本地日线筛选
- 超过 2 个交易日:只用 WebSearch + 用户提供的实时报价,不要硬推本地候选
- `earnings_calendar.last_sync` 超 24 小时:财报相关候选必须 WebSearch 双验证

### Step 1 - WebSearch 拉全催化候选

至少覆盖以下查询方向,每类取有明确 ticker 和来源的候选:

```text
"stocks moving today {date} earnings upgrade buyback order partnership"
"after hours earnings movers {date} raised guidance beat"
"stocks up after earnings guidance yesterday after hours {date}"
"premarket movers {date} stocks news"
"analyst upgrades stocks today {date}"
"stocks up after earnings guidance buyback {date}"
"AI nuclear defense biotech stocks moving today {date}"
"top gainers stocks today {date}"
"after hours gainers {previous_trading_date} stocks"
"overnight trading gainers {date} US stocks"
"{ticker} after hours premarket price {date}"
```

收集字段:

- ticker
- 催化类型
- 新闻 URL
- 新闻时间
- 当前报价状态: `当前盘前/盘中`、`最近 after-hours`、`仅昨收`、`报价冲突`
- 盘前/盘中/after-hours 价格、涨幅、成交量/成交额;若没有,写明缺失

**每步交易决策认证**:

- 推荐前: Web 确认新闻/公告/SEC 和当前或延长交易报价
- 加仓/补仓前: Web 确认催化未证伪、无新增融资/ATM/反拆/停牌红旗、价格守住关键低点并重新站回 VWAP/开盘价/昨收之一
- 减仓/止损前: Web 或用户券商盘口确认是否跌破 VWAP/开盘价/关键平台,避免把延迟报价当成实时破位
- 复盘前: 用 Web/行情和本地最新完整日线确认最终开高低收、成交量和新闻变化
- 若用户信息与 Web 冲突,用户券商价只作为交易价格输入;新闻、红旗和状态判断必须继续用公开源复核

**延长交易强弱硬要求**:

- 对 Web movers、上一交易日涨幅榜、财报后首日、AI/数据中心/国防/生科催化股,进入最终 Top 12 复核前必须尝试抓最新 after-hours / premarket / overnight 报价
- 官方盘前前或 overnight 时段,优先抓当前 overnight/夜盘可见价;若只能拿到上一段 after-hours 或旧缓存,该票只能标`观察/待盘前复核`,不得把旧价用于正常入场、止损、止盈和概率表
- 若官方盘前尚未开始,使用最近可见 after-hours 作为"二日延续强弱"代理,但输出必须标注不是当前盘前
- 上一交易日大涨后,延长交易仍 `>= +5%` 或突破上一日高点/关键整数位的票,自动进入复核池,并在排序中加权;典型场景是 RXT 这类新闻点火后夜盘继续确认的二日延续票
- 若延长交易明显低于收盘或跌破上一日 VWAP/关键承接区,即使日线和新闻强,也从`主攻`降为`等修复`
- 若报价源互相矛盾或只有旧缓存,不能把该票列为`可入`;只能列为`观察/等待开盘确认`

**用户点名 ticker 的处理**:

- 点名 ticker 只能作为额外一行加入候选池,字段 `source=user_mentioned`
- 不允许跳过上面的 WebSearch 全市场扫描
- 不允许把点名 ticker 写进主攻位,除非它在合并候选池排序中进入 Top 3
- 如果点名 ticker 未进入 Top 3,输出时可在"点名票复核"里说明原因,但不要把它混进推荐仓位表

### Step 2 - 本地候选补充

并行补六类本地候选,避免 WebSearch 漏掉未上榜但已经启动的票:

- **财报预判埋伏池(默认必跑,未来 3-10 天财报窗口)**
- **事前催化雷达池(默认必跑,未来 1-10 个交易日明确事件窗口;详见 pre-catalyst-radar.md)**
- **财报跳空首日池(默认必跑)**
- `gap_candidates.sql`:放量接近高点的 gap 候选
- `squeeze_candidates.sql`:轧空候选
- `micro_momentum.sql`:高弹性微盘候选
- `long_momentum_candidates(price_buckets="10-100,100-500", limit_per_bucket=5)`:中大盘强势突破

合并候选后去重,最多进入复核 12 只。优先级先看:

1. 是否具备**高弹性扩张空间**
2. 是否有未来 `1-10` 个交易日的明确事件窗口、同业传导或高质量预期差,且价格还没有完全反应
3. 是否有当前盘前/盘中或最近 after-hours 的强确认,尤其是二日延续站上前高/整数位
4. 是否有 48 小时内真实催化,或未来 `3-10` 天内高质量财报预期差
5. 是否仍在初启动或扩张中,而不是尾声
6. 是否有可执行回踩/二次突破结构,或明确的财报前埋伏区间
7. R/R 是否能 ≥ 1.5

特殊优先级:

- `控制权交易 / 救命融资 / 固定每股价战略投资` 且有公司公告或 8-K + 主流媒体双确认时,优先级可高于普通二日延续票
- 这类票即使历史 20 日均成交额低,只要当前盘前/盘中成交额已经显著放大,应进入 Top 12 复核;不得因昨日完整日线低量或基本面亏损静默删除
- 若开盘先跌后重新站回 VWAP/开盘价/盘前承接区,按`洗盘后重估`结构重新评分,而不是直接按开盘下跌排除
- 若交易对价主要是票据、存在交割条件或大幅稀释,只能给日内事件交易,不能给隔夜/长期投资结论

排序纪律:

- 每只候选保留 `source`: `pre_catalyst` / `web_mover` / `extended_hours_mover` / `local_gap` / `local_squeeze` / `local_micro` / `earnings_pre` / `earnings_day1` / `user_mentioned`
- 每只候选保留 `quote_status` 与 `quote_time`;如果缺少当前或延长交易报价,必须在排序备注里说明
- 若用户点名票进入推荐表,必须写明"它不是因用户点名入选,而是因为在候选池中排名第 X"
- 若没有足够数据完成候选池排序,不得输出"资金规划";只能输出"数据不足,请提供实时盘前报价或稍后重扫"

### Step 3 - MCP 补全

对候选调:

- `latest_prices(symbols=...)`
- `symbol_snapshot(symbols=..., lookback_days=60)`
- `earnings_dates(symbols=..., within_days=30)`
- `company_fundamentals(symbol)` for Top 候选

补齐:

- 昨收、20 日均成交额、量比、ATR14%、RSI14、20 日/52 周位置
- 市值、float、short_percent_of_float、short_ratio、现金流、收入/利润质量
- 未来财报窗口,避免财报前被砸

### Step 4 - 硬排除

任一命中则排除,不要降级保留:

- 近 7 天有 offering / ATM / reverse split / bankruptcy / SEC investigation
- 新闻催化无法追溯到可靠 URL
- R/R < 1.5
- 纯尾盘拉抬
- 已第 3 天以上连续暴涨,且现价接近日内极端高点
- 生科二元事件未发生,只是赌 FDA/PDUFA/Phase 3 出结果

不要误排除:

- 控制权交易/救命融资票有稀释或票据结构时,不是自动排除;应降为`日内事件交易`,再看锚价折价、成交额和 VWAP reclaim
- 低价股历史日均成交额低,但当日成交额已显著放大且消息为 A-/A 级硬催化,不能只按历史流动性判死刑

### Step 5 - 打分

每只候选按五个维度打 0-10 分,重点不是稳,而是**高赔率扩张**:

- 爆发潜力 0-3:
  - 3:当日或近 3 日弹性大,量能明显扩张,启动速度快
  - 2:有弹性,但扩张速度一般
  - 1:温和上涨,爆发力有限
- 延续概率 0-2:
  - 2:48 小时内新鲜催化,且更像第 1 天或第 2 天早段
  - 1:催化仍在,但已部分发酵
  - 0:更像尾声或单纯回光返照
- 交易结构 0-2:
  - 2:有明确回踩位或二次突破位,止损后仍保留充足利润空间
  - 1:能做,但结构一般
  - 0:现价太高或结构太差
- 资金拥挤度 0-1:
  - 1:市场开始关注但尚未极端一致
  - 0:已经拥挤,只剩最后一棒
- 风险红旗 0-2:
  - 2:无红旗
  - 1:轻微疑点,但不影响进攻
  - 0:红旗偏多,只适合极低仓或放弃

**附加解释规则**:
- ATR 高、3 日弹性大、量比高,可以给爆发潜力加分,不再默认降级
- 真正需要降级的不是高波动,而是**高波动但赔率差 / 已尾声 / 无回踩空间 / 无真实催化**

### Step 6 - 输出最多 3 只

固定结构:

- `主攻` 1 只:单次大肉优先的最高赔率机会
- `备攻` 1 只:同样高弹性,但确认度略低
- `防守备选` 0-1 只:只有进攻机会不够干净时才出现

仓位层级:

- **高弹性主攻**: 可以给标准进攻仓,但仍受硬止损限制
- **高风险进攻**: 缩仓参与
- **极端 pump 彩票仓**: 只允许极小仓位

若盘前未开或缺少实时价:

- 不伪造当前价/盘前量
- 用昨收和触发价输出"开盘后条件单"
- 明确写"等 09:45 ET 后 VWAP/突破确认"

## 输出额外字段

除 [../output-contract.md](../core/output-contract.md) 固定模板外,每只还要写:

- 为什么它可能走出 15%-50% 波动
- 现在参与还来不来得及
- 如果错了为什么不至于深套
- 是否属于财报后右侧、轧空、盘前扩张、龙头追击或纯高弹性突破

## 回退

如果所有候选都不过硬门或都已尾声:

- 明确说"今天没有值得做的大肉机会"
- 建议 10:30 ET 后重新扫描,或让用户给出正在看的 ticker 做风控评估

## 输出要求

最终候选总表、单票展开、报价校验、概率与最终自检统一按 [../output-contract.md](../core/output-contract.md) 执行;默认模式额外要求每只写"为什么可能走出 15%-50% 波动 / 现在还来不来得及 / 错了为什么不至于深套"四问。
