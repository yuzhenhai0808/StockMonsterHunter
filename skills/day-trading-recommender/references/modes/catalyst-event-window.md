# 模式 6:临床事件窗口(catalyst-event-window)

**场景**: 生科微盘 + 已发布的真临床数据 + 未来 30 天内有具名会议/AdCom/PDUFA 日期 + 当日突破 52W 高 | **持仓**: 开盘 30 分钟 - 收盘(或事件日前一交易日清仓) | **目标**: +10~25% | **胜率**: ~40%(赔率驱动型)

## 与默认禁令的关系(必读)

day-trading-recommender 默认对"市值 < $1 亿生科"+"FDA/临床事件"一票否决(见 earnings-gap-day1.md 第 94 行;long-stock-recommender 的 news-review-checklist 同口径)。

**本模式是该默认禁令的窄白名单豁免**。Step 0 全部条件满足才豁免;任一项失败 → 退回默认禁令,排除。**不能用本模式给"赌 PDUFA"或"押 Phase 3 出数据"找借口** —— 那些是事件未发生的二元赌博,本模式只做"事件已发生 + 后续具名窗口"。

## 触发条件

用户关键词:"临床催化" / "ATS" / "ASCO" / "AACR" / "AdCom" / "PDUFA" / "Phase 3 数据" / "EDSA 那种" / "生科催化"

**自动触发**:用户提到具体生科 ticker 且 `company_fundamentals` 显示市值 < $500M。

## 流程

### Step 0 - 默认禁令豁免预检(本模式特有)

以下 4 项**全部成立**才进入下一步,任一失败 → **排除**:

1. 临床/数据公告**已发生**(可 WebSearch 验证日期),不是"即将公告"
2. 未来 30 天内有**具名**会议/PDUFA 日期(具体日期 + 具体会议名),不是"将公布"
3. 现金跑道 ≥ 12 个月(不会因催化延期被迫增发)
4. 过去 7 天无 ATM/直接发行/反向拆股/S-1 红旗

### Step 1 - 临床数据公告时间复核(WebSearch)

```
"{ticker} phase 3 OR phase 2 results announcement date"
"{ticker} clinicaltrials.gov NCT primary completion"
```

- 公告日 ≤ 3 个月前 = 🟢 数据新鲜,市场仍在定价
- 公告日 > 6 个月前 = 🟡 已被消化,**降级**(R/R 要求提到 2.0)
- 公告但未通过(failed primary endpoint) = 🔴 **排除**

### Step 2 - 未来事件窗口确认(WebSearch)

```
"{ticker} ATS 2026" / "ASCO 2026" / "AACR 2026" / "ASH 2026" / "AHA 2026"
"{ticker} upcoming presentation conference"
"FDA PDUFA calendar {drug name}"
"{ticker} AdCom advisory committee date"
```

判定:
- 找到具体日期 + 具体会议名 = 🟢 真催化窗口
- 只有"将公布"无具体日期 = 🔴 **排除**
- 事件日 ≤ 7 天 = ⚡ 高赔率窗口期,允许更激进入场上限
- 事件日 7-30 天 = 🟢 标准窗口
- 事件日 > 30 天 = 🟡 太远,降级

### Step 3 - 现金跑道 + 资本事件复核

调 `company_fundamentals(ticker)`,计算:

```
跑道(月) = total_cash / (|operating_cashflow_ttm| / 12)
```

- ≥ 12 个月 → 🟢
- 6-12 个月 → 🟡 降级,仓位减半
- < 6 个月 → 🔴 **排除**(催化未实现前必增发)

WebSearch:

```
"{ticker} 8-K offering OR ATM OR S-1 OR direct offering last 30 days"
"{ticker} reverse split last 90 days"
```

任一红旗 → **排除**(同 [../red-flags.md](../workflow/red-flags.md) 默认 B 门规则,本模式不放宽)。

### Step 4 - 量能 + 突破真实性(技术面)

调 `symbol_snapshot(ticker, lookback_days=60)`,要求:

- `volume_ratio_20d` ≥ **5x**(生科突破日量比要求比常规模式更高)
- `close_to_20d_high_pct` ≥ 99% 或已破 52 周高
- 当日涨幅 ≥ 30%(没到 30% 说明催化未点燃,只观察)
- 累积 5 日涨幅 < 单日涨幅 × 1.5(确认是单日点火,不是连续 pump 残骸)

不满足 → 不入场,只观察。

### Step 5 - R/R 计算(本模式标准:-10% 止损)

生科微盘 ATR 通常 12-20%,4% 止损会被噪音打掉。本模式:

- **入场上限价** = 突破日收盘价 × 1.03(给 3% 缓冲,超过别追)
- **硬止损** = 入场价 × 0.90(-10%)
- **第一止盈** = 入场价 × 1.15(+15%, 卖 50%)
- **第二止盈** = 入场价 × 1.25(+25%, 剩余 50%, 事件日前一交易日必清)
- R/R = 15/10 = **1.5**(达标);加权(50/50)= **2.0** ✅

### Step 6 - 仓位上限(硬线)

不论催化多真,**仓位上限 = 总资金 2%**。

理由:
- ATR 12-20% 意味着一天可能 ±10%
- 临床股有"会议口风变盘前突然跳水"风险
- 必须确保最大单笔亏损 ≤ 总资金 0.2%(2% × 10%)

### Step 7 - 时机决策树(本模式特有)

**A. 盘后挂条件单(明日开盘成交)**
- 盘前 8:00 ET 看一次预报价,> 入场上限就取消
- 用 stop-limit buy:触发价 = 入场上限,限价 = 触发价 + 0.5%
- 同步挂硬止损卖单

**B. 开盘后立即(9:30 ET)**
- 等开盘前 5 分钟成交 ≥ 30% 日均量再入,否则取消
- 进单后 9:35 量能不延续 → 立刻全清

**C. 开盘后回踩(10:30 ET 后)**
- 等回踩 VWAP 或 MA10
- 进单点必须有"买盘吸收"信号(K 线下影线 + 量能小幅放大)

## 输出必须包含

(在 [../output-contract.md](../core/output-contract.md) 输出模板基础上,补充本模式特有字段)

**本模式特有字段**:
- 临床数据公告日期 + 试验名称 + 主要终点 + URL
- 未来事件名 + 日期 + URL
- 现金跑道月数
- 默认禁令豁免审核(逐项 ✅/❌)
- 事件日前清仓日期(硬性提醒)

## 需要降级或排除

- 现金 < 6 个月跑道 → 排除
- 临床数据 > 6 个月前 → 降级(R/R ≥ 2.0)
- 未来事件无具名 → 排除
- 同时叠加财报窗口 ≤ 3 天 → 排除(双重二元风险)
- 当日涨幅 < 30% 或量比 < 5x → 不入场(只观察)
- 失败的临床试验 → 排除
- 反向拆股 / S-1 / ATM 7 天内 → 排除

## EDSA 4/27 样板(基准案例)

| 检查项 | EDSA 实况 | 判断 |
|---|---|---|
| 临床数据公告 | 2/24 公布 Phase 3 阳性(Paridiprubart ARDS) | 🟢 2 个月新鲜 |
| 未来事件 | 5/15 ATS Innovation Summit + 5/20 ATS 国际会议主试者展示 | 🟢 双 catalyst,17-22 天窗口 |
| 现金跑道 | $12M cash / $7.9M 年烧 = 18 个月 | 🟢 |
| 资本事件 7 天 | 已查无红旗 | 🟢 |
| 默认禁令豁免 | 4/4 全过 ✅ | 🟢 启用 |
| 量比 | 13x | 🟢 |
| 突破 | +54% 单日,刚破 52W 高 $11.87 | 🟢 |
| 5 日累积 | +65% (≈ 1.2x 单日,非连续 pump) | 🟢 |
| ATR | 14.4% | 用 -10% 止损 |

**本模式输出**:
- 入场上限 $11.50,硬止损 $10.30(-10.4%)
- 第一止盈 $13.20(+14.8%, 50% 仓位)
- 第二止盈 $14.50(+26.2%, 剩余仓位)
- 仓位上限 1-2% 总资金
- **5/14 收盘前必清**(ATS 峰会前一交易日,不博事件落地)

| 失败的 Phase 3 反弹 | 临床公告已发生但 failed primary | Step 1 排除条件 | 排除 |

## 纪律提醒

- **事件日前一交易日必清**,不持有跨事件 —— 哪怕看着要继续涨。落地反应是赌博,本模式不博
- **现金跑道 < 12 个月降级,< 6 个月排除** —— 因为催化日历有时会"被推迟"(生科常态),没钱的公司一推迟就增发
- **本模式不替代盘前 gap 默认模式**。用户没说生科关键词时不主动切换
- **不要把这个模式当万能** —— 40% 胜率意味着 6 成会亏,只是单笔赔率高把期望值拉正

## 输出要求

最终候选总表、单票展开、报价校验、概率与最终自检统一按 [../output-contract.md](../core/output-contract.md) 执行;本模式只补充上文"本模式特有字段"和"事件日前一交易日必清"硬规。
