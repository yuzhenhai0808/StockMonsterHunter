# 模式 9:事前催化雷达

**场景**: 用户想在股票还没明显波动前,提前找可能吃到"第一口"上涨的候选。  
**持仓**: 事件前 `1-10` 个交易日小仓预埋,或只设价格/新闻提醒;事件兑现后按日内规则处理。  
**目标**: 找**赔率尚未被市场完全定价**的候选,不是追已经上榜的 movers。

## 定位

这是 `day-trading-recommender` 的**事前模式**。它和默认高赔率雷达不同:

- 默认高赔率雷达偏"已经有量价或新闻启动后的当天作战"
- 本模式偏"事件窗口之前建立候选池和条件单"
- 它不承诺命中,只提高在催化发生前盯到正确票的概率

必须先说明:

```text
这是事前预判池,不是已验证启动信号;仓位要比盘中确认票更小。
```

## 触发条件

用户关键词:

- 事前预判
- 第一口
- 还没涨
- 爆发前
- 提前埋伏
- 未启动
- pre-catalyst
- before it moves

若用户同时问"今天能不能买",先输出本模式的`可提前埋伏 / 只设提醒 / 等确认`分类,再给日内条件单。

## 候选来源

### A. 确定日期事件

优先:

- 未来 `0-10` 天财报,但价格未过热
- FDA / PDUFA / AdCom / 临床会议 / ASCO / AACR / ATS / ESMO
- 公司已公告的投资者日、产品发布、重大会议发言
- 宏观/政策/行业日历能直接影响的小盘高 beta 板块,如 BTC 矿股、核能、量子、AI 电力、国防

### B. 同业传导

WebSearch:

```text
"{sector} earnings beat guidance this week"
"{peer1} earnings beat raised guidance {current_year}"
"{theme} stocks moving after earnings guidance"
```

优先找:

- 同行业已有 1-2 家刚爆,但目标票还没公布
- 目标票过去对同业消息有跟随弹性
- 股价仍在 20 日高点下方 `5%-20%`,还没完全抢跑

### C. 压缩待爆形态

用 `symbol_snapshot` / SQL 找:

- 20 日振幅足够大,但最近 3-5 日收窄
- 成交量温和放大,但还没爆量
- 价格靠近 20 日高点下方 `3%-12%`
- RSI14 在 `45-68`,避免已经过热
- 20 日均成交额 `>= $10M`;高弹性小票可放宽到 `$3M`,但仓位降级

## 筛选流程

### Step 0 - 先排除"已经不是第一口"

任一命中则从事前池移出,改为日内/二日延续模式:

- 过去 1 日已涨 `>20%`
- 过去 5 日已涨 `>60%`
- 延长交易已涨 `>15%`
- 已经登上主流涨幅榜 Top 3

例外:若用户明确要二日延续,再切回 `top-gainer-momentum` 或 `monster-stock-hunter`。

### Step 1 - 建立事件池

并行取:

- `earnings_dates(within_days=10)`
- WebSearch 未来一周 FDA / 临床 / 投资者日 / 行业会议
- WebSearch 同业财报传导和主题轮动
- 本地日线压缩候选

每只候选必须记录:

- `event_type`
- `event_date`
- `event_source_url`
- `confidence`: `confirmed / likely / rumor`
- `price_reaction_so_far`: 1d / 5d / 20d

### Step 2 - 事前评分

每只 0-10 分:

- 事件明确度 0-2:
  - 2:官方确认日期或公司公告
  - 1:高可信日历/行业会议,但公司未单独公告
  - 0:传闻或社媒线索
- 赔率未定价 0-2:
  - 2:5 日涨幅 < 15%,距 20 日高 `5%-20%`
  - 1:已有小幅抢跑
  - 0:已经大涨,不是第一口
- 历史反应 0-2:
  - 2:过去相似事件有 `+15%` 以上反应
  - 1:有温和正反应
  - 0:历史反应差或经常高开低走
- 板块/同业传导 0-2:
  - 2:同业刚爆且逻辑直接传导
  - 1:主题热,但传导间接
  - 0:无明显传导
- 风险红旗 0-2:
  - 2:无融资/反拆/SEC/做空红旗
  - 1:轻微红旗,只能小仓
  - 0:硬红旗,放弃

### Step 3 - 仓位分类

- `可提前埋伏`: 评分 `>=7`,事件日期明确,价格未过热
- `只设提醒`: 评分 `5-6`,但还缺触发信号或日期不够硬
- `等公告后确认`: 事件可能大,但不确定性太高,只等新闻/盘口确认
- `放弃`: 已经涨完、无事件源、红旗过重

### Step 4 - 触发条件

事前票不直接追高,必须给触发器:

- 价格触发: 突破近 5 日高点 / 站上 20 日高点 / 回踩不破 MA10
- 新闻触发: 官方 PR / IR 日期确认 / 同业二次验证
- 量能触发: 盘前或盘中相对量 `>=1.5x`
- 失效触发: 跌破预埋区下沿、事件日期被推迟、同业财报传导失败

## 仓位纪律

对 `$5,000` 账户:

- A 级事前埋伏: 本金 `$400-$700`,最大亏损 `$25-$50`
- B 级观察转入: 本金 `$250-$450`,最大亏损 `$20-$35`
- 传闻/未确认事件: 不建仓,只设提醒
- 总事前埋伏本金不超过 `$1,200`

事前票若在事件前已经涨到第一目标:

- 默认兑现 `50%-70%`
- 剩余仓位止损抬到成本或关键支撑
- 不把已经兑现的利润重新加到同票里

## 输出格式

先给总表,最多 `3-5` 只。

```md
| 票 | 分类 | 事件窗口 | 事件源 | 预判分 | 当前阶段 | 预埋区 | 触发加仓 | 失效位 | 第一目标 | 建议本金 | 一句话 |
|---|---|---|---|---:|---|---:|---:|---:|---:|---:|---|
| TICKER | 可提前埋伏/只设提醒/等公告后确认 | MM-DD | 财报/FDA/同业/政策 | 8/10 | 未启动/轻微抢跑/过热 | $X-$Y | $Z | $X | $Y | $XXX | 为什么可能吃第一口 |
```

单票展开必须写:

- 这单的"第一口"来自哪里
- 事件日期和来源 URL
- 为什么市场可能还没完全定价
- 如果错了,哪里证明预判失败
- 是否需要在事件前减仓

## 明确不能做的事

- 把传闻当确定催化
- 因为想吃第一口就重仓单票
- 忽略事件延期风险
- 用已经大涨的票伪装成事前预判
- 没有日期、没有来源、没有触发器却给买入建议

## 输出要求

最终候选总表(可在上文 `## 输出格式` 节用本模式独有列扩展)、单票展开、报价校验、概率与最终自检统一按 [../output-contract.md](../core/output-contract.md) 执行;事前催化雷达额外要求把候选分成`可提前埋伏 / 只设提醒 / 等公告后确认`三类,且没有日期/来源/触发器的不得给买入建议。
