# 日内退出框架

本文件统一日内的入场方式、失效信号、被套处置和补仓门槛。具体止损止盈数值见 [risk-rules.md](../core/risk-rules.md);本文件聚焦"何时减、怎么减、被套怎么办"的操作动作。

## 允许的参与方式

### 1. 回踩做

适用于:

- 盘前高开
- 开盘后第一次回踩
- 不破关键区间或 VWAP
- 高波动龙头首波冲太快,但第一次真正像样的回踩出现

要求:

- 回踩后要有承接
- 不能一路掉回涨停前起点
- 对 monster 票,回踩幅度可以更深,关键是**回踩后能不能迅速被接住**

### 2. 二次突破做

适用于:

- 开盘第一波冲高后整理
- 再次放量突破小平台
- 高波动票首波过猛,无法舒服低吸时

要求:

- 整理时间不能太久
- 量能要跟得上
- 若波动极大,允许平台更宽,不要用过窄的稳健票标准去要求它

### 3. 放弃 / 只观察

出现以下任一情况直接放弃:

- 开盘直接一字冲太高
- 当前价贴着日内高点,没有结构
- 只剩追高这一个动作
- 波动很大但完全没有承接,只有上蹿下跳
- R/R 计算下来 < 1.5,且无法通过放宽止盈到 +6% 修复

### 4. 做 T(快进快出)

适用于:

- 反拆首日 / 低流通 squeeze / 榜首惯性票
- 当日振幅极大,但价格能反复回到 VWAP 或关键平台上方
- 量能爆炸,买卖盘足够承接小仓快速进出
- 用户目标是吃 `8%-20%` 波段,不是持有到收盘
- 用户已经用同类票做出过盈利,需要把利润保护放在继续扩张之前

要求:

- 只在确认承接后做低吸或二突,不在垂直拉升中追
- 每轮先设硬止损,第一目标及时减仓
- 一旦跌破 VWAP 且反抽不过,停止同票继续做 T
- 做 T 成功后不要把利润重新暴露给尾盘回撤,尾盘前降低仓位
- 每次输出必须写清这是第几轮 T:`首轮试错 / 盈利后第二轮 / 只允许利润仓`
- 若上一轮已盈利到账户 `1.5%-2%`,下一轮默认只允许用原风险的 `50%-70%`
- 若上一轮已盈利到账户 `5%+`,下一轮默认应建议停止同票交易;若用户坚持,只能用已实现利润 `10%-20%` 做最大亏损预算

### 5. 被套反抽处置

适用于:

- 用户已经在高波动票中持有 `30%+` 本金仓位
- 当前浮亏明显,但标的仍有真实催化、爆量、short squeeze 或二次修复结构
- 价格没有持续跌破催化前平台,并且出现 VWAP / 开盘价 / 昨收 / 用户成本附近的 reclaim
- 已按 [web-cross-check.md](../core/web-cross-check.md) 复核当前价、成交额、新闻和红旗

目标:

- 先解除账户级灾难风险
- 再保留足够仓位参与二次修复
- 不把风险控制做成低位清仓、反抽卖飞

分层动作:

- 第一层:若仍是满仓或 70%+ 本金,先降到 `40%-60%`(风险刹车,不是看空标的)
- 第二层:若价格站回 VWAP/开盘价/昨收,停止继续砍仓,把剩余仓位改成`反抽仓`
- 第三层:若二次修复评分达标且 Web 认证无红旗,可在关键支撑或 reclaim 后小幅补`修复仓`,但必须重算均价和最大亏损
- 第四层:在用户成本下方 `5%-10%` 的逃生区分批减仓,不要一次性卖光
- 第五层:若反抽越过用户成本或前高,剩余仓位按止盈处理

失效:

- reclaim 后再次跌破 VWAP 且 5-15 分钟收不回
- 跌破当日低点或催化前关键平台
- 反抽到逃生区但量能明显衰减、连续冲不过
- 出现 offering / ATM / 反拆 / 交易所风险等新红旗

## 失效信号

入场后出现任一信号,立即按硬止损执行:

- 跌破 VWAP 且不回
- 跌破爆量起涨区
- 二次突破失败并快速回落
- 量能明显衰减
- 高波动票若连续两次尝试上攻都被砸回,说明空间可能已经用完
- 跌破入场上限价 + 5 分钟未收回(快速失败)

## 标准处理

- 入场后立刻带止损(盘前能挂就挂)
- 第一止盈触发后减仓 50%,剩余仓位的止损抬到入场价
- 第二止盈触发后全部清仓,不贪
- `14:45 ET` 前清仓
- 做 T 票如果已经完成 `10%+` 单轮收益,优先兑现一半以上,不要等"完美顶部"
- 做 T 票如果已经完成账户 `$100` 利润(对 `$5,000` 账户约 2%),应优先保护成果:可以停止交易,或只用利润仓做极小第二轮
- 做 T 票如果已经完成账户 `$250-$300` 利润(对 `$5,000` 账户约 5%-6%),默认当天目标已经达成;只有在 VWAP / 关键平台 / 前高 reclaim 三者至少满足其一时,才允许极小第二轮

## 重仓亏损标准处理

- 先确认用户真实成本、股数、当前仓位占账户比例,不得用猜测成本做处置
- 如果用户成本价与系统假设相差 `>=5%`,必须重算全部减仓/反抽价位
- 用户截图或口述价格必须标注时间段,并用 Web/公开源或本地最近完整日线解释差异;无法解释时只给情景,不给确定操作
- 若标的已经从低点重新站回 VWAP/开盘价,不得继续使用"跌破止损立即全出"的普通入场规则;应切换为`被套反抽处置`
- 输出必须包含:`立即风险刹车仓位 / 停止继续减仓价 / 反抽减仓价 / 失效价 / 最多还能亏多少`

## 补仓修复输出

只有满足 Web 认证和二次修复条件时才能给补仓。输出必须包含:

- `补仓触发`:价格站回哪个关键位才补
- `补仓金额`:补仓后总本金占账户比例
- `补仓后均价`:用用户成本和补仓价重算
- `失效价`:跌破后取消修复计划
- `最大账户亏损`:失效价触发时账户还会亏多少
- `反抽阶梯`:成本下方、成本附近、前高/目标价分别卖多少

不满足时,只能减风险或保留观察仓,不得建议补仓。

## 同日加仓禁令

- 跌到止损位前不要加仓(摊平是日内最大忌)
- 达到第一止盈减仓后,剩余仓位不要再加
- 日内**只能减仓,不能加仓**(盘前挂条件单除外)
- 同票止损后当日默认不再重买(除非有重大新信号 + 三连校验重新通过)

## 切换到 swing 的硬规则

如果在日内中找到不想平仓的票:

1. 平掉日内仓位(按本框架规则)
2. 重新用 long-stock-recommender 评估是否值得 swing 持有
3. 若决定 swing,用**新的资金**建立 swing 仓位,止损和目标按 swing 框架设

不要让日内仓位变成 swing。
