# 日内交易案例库

本文件汇总 day-trading-recommender 沉淀下来的真实日内案例,用于校准催化硬度、阶段判断、Pump 评分、止损执行。

新增案例时按"事件 → 决策 → 结果 → 教训 → 已沉淀到哪条规则"五段写。

## 案例 1:HTCO 4/28 微盘 Pump 评分 11(高度疑似)

### 事件

- 当日:HTCO 现价约 $38,3 日涨幅 +240%
- 市值约 $8300 万,流通股 287 万,空头 18%
- 催化:5/7 股东投票改 B 类股投票权(典型"拉升后稀释"路径)
- 催化源:stocktitan + moomoo(二线媒体 + 社交)

### Pump 评分

| 维度 | 值 | 分 |
|---|---|---|
| 市值 $8300 万 | 1 |
| 流通股 287 万 | 2 |
| 52W 高 / 现价 1.6 | 0 |
| 3 日涨幅 +240% | 2 |
| 5 日涨幅 > 300% | 2 |
| 年营收 $2.14 亿 | 0 |
| 盈利:亏损 -10%,FCF 正 | 1 |
| 空头 18%(轧空结构,不加分) | 0 |
| 公告:股东投票改投票权 | 2 |
| 催化源 二线媒体 | 1 |
| **总分 = 11** | | 高度疑似 pump |

### 决策与结果

总分 11 → 高度疑似 pump,建议仓位 ≤ $50。空头 18% 的轧空结构允许极小仓博一把,但因为资本事件(5/7 投票)是典型"拉升后稀释"路径,主导偏向只能写`pump + 软催化情绪冲刺`,不允许伪装成"基本面突破"。

### 沉淀到的规则

- [red-flags.md](workflow/red-flags.md) "微盘 Pump 评分表":总分 9-14 → 仓位上限 $50
- 空头 ≥ 15% 的轧空结构可与 pump 评分叠加判断,但仍走极小仓
- 资本事件(投票/合股/拆股)与 pump 评分高合并出现 → 直接拒绝重仓主推

## 案例 2:YAAS 4/27 典型 pump-and-dump(总分 19)

### 事件

- 当日盘前:YAAS @ $1.80,3 日涨幅 +94%(盘前)
- 市值约 $1200 万,流通股 332 万
- 52W 高 / 现价 = $560 / $1.80 = 311(极度合股过)
- 年营收 $54 万,亏损 + FCF 负
- 催化:海南房产收购 + 合股背景,无明确新闻源

### Pump 评分

| 维度 | 值 | 分 |
|---|---|---|
| 市值 $1200 万 | 2 |
| 流通股 332 万 | 2 |
| 52W 高/现价 311 | 2 |
| 3 日涨幅 +94% | 2 |
| 5 日涨幅 > 90% | 1 |
| 年营收 $54 万 | 2 |
| 亏损 FCF 负 | 2 |
| 空头估 < 5% | 2 |
| 7 日内公告:房产收购+合股 | 2 |
| 催化源:无 | 2 |
| **总分 = 19** | | 典型 pump-and-dump |

### 决策与结果

总分 19 → 典型 pump-and-dump,**不建议任何参与**,只做教育性分析。即使用户点名,也只能输出"已劝阻,只观察"。

### 沉淀到的规则

- Pump 总分 ≥ 15 → skill 拒绝推荐(任何仓位)
- 52W 高/现价 > 100 + 多次合股历史 = 自动归入"典型 pump-and-dump"
- 无明确新闻源的微盘暴拉默认假设为操纵,举证责任反转

## 案例 3:MXL 财报跳空(归 long-stock 不归日内)

### 事件

- 财报前 3 天:MXL @ $34 横盘 2 个月,RSI ~55
- 4/24 财报后跳空 +76% 至 $60
- Pump 评分 = 1(完全不属于 pump)

### 决策与结果

事前用日内模式无法捕捉(K 线无突破信号,日内雷达看不到)。事前用 long-stock-recommender 模式 5(财报跳空)在 4/16-4/22 有 5-7 天建仓窗口,可埋伏。

### 沉淀到的规则

- 日内模式不捕捉这种"财报前埋伏 + 财报后大跳空"机会;归 swing 的模式 5
- 但**财报后首日**已经跳空 +76% 时,本 skill 的`财报跳空首日`模式仍可处理:看是否还有第二段延续(成交额必须扩张、盘中守住开盘价)
- 不要把"事后看 MXL"当作日内雷达的失败 —— 它本来就不在日内的射程内

## 案例 7:CREG 2026-05-15 报价冲突导致日内计划作废

本案例用于所有日内推荐和用户纠错场景。它证明: 日内交易必须建立在当前可交易报价之上;本地日线、昨日高点或旧 after-hours 只能做候选背景。

事件:

- 本地日线显示 CREG `2026-05-14` 收盘约 `$0.7143`,盘中最高约 `$0.9099`,日涨约 `+257%`
- 原回答错误地用本地日线强势给出今日彩票仓计划
- 用户纠正: 券商当前看到 CREG 约 `$0.52`
- 用户价相对本地收盘价偏离约 `-27%`,远超 `>=5%` 旧计划作废阈值

正确处理:

1. 立即停止沿用原计划,输出`修正版状态表`。
2. 用户券商价作为最高优先级交易基准。
3. 用用户价重算入场、硬止损、止盈、建议仓位上限、最大亏损、计划股数和 R/R。
4. 若无法拿到报价时间、成交量/bid-ask 或 Web 复核,只能标`报价冲突/只观察`。
5. 不得继续保留原主攻/备攻、建议仓位或概率。

沉淀规则:

- `latest_prices` / `symbol_snapshot` / SQL close 不是实时价。
- 用户报价与日线收盘偏离 `>=5%` 时,原 entry/buy zone/止损/止盈/仓位/概率全部作废。
- 用户报价与 Web/本地无法解释地冲突时,证据等级为 D,不给入场、加仓或主攻。
- 今日/盘前/盘中请求必须先有报价校验表,再有交易计划。

## 案例 8:RGNT/GMM/FAC 2026-06-09 延迟 movers 误当交易基准

本案例用于修正`15 分钟延迟 movers + 当日强波动 + 条件触发价`场景。它证明: 候选方向可以对,但如果输出时没有当前 quote 页、用户券商盘口或双公开源互证,就不能把延迟榜单价写成可执行计划。

原输出问题:

- 使用 StockTitan movers 约 `09:44 ET` 的 15 分钟延迟快照作为报价基准。
- 对 `RGNT` 给出 `$2.40` 二突、`$2.20` 止损、`$2.65/$2.90/$3.20` 目标,但没有 quote 页/盘口/bid-ask 互证。
- 对 `GMM` 给出 `$0.22` 二突和 `$0.19` 失效,同样只有延迟榜单背景。
- `FAC` 作为新上市/De-SPAC 票被放进推荐表,但本地库无数据,缺少可交易 quote 互证,只能审计,不应进入备攻计划。

事后真实完整日线:

| 票 | 2026-06-09 开 | 高 | 低 | 收 | 量 | 相对前收 | 复盘分类 |
|---|---:|---:|---:|---:|---:|---:|---|
| RGNT | 2.52 | 6.55 | 2.23 | 2.41 | 103.57M | +88.28% | 方向命中,但报价流程违规 |
| GMM | 0.2132 | 0.2345 | 0.0920 | 0.1079 | 1.01B | +19.89% | 快打可赢,持有失败 |
| FAC | 缺本地日线 | 缺 | 缺 | 缺 | 缺 | Web-only | 新上市审计,不应给备攻 |

正确处理:

1. `RGNT` 可以写成`强观察 / 待实时 quote 确认`;只有用户券商或 quote 页证明仍在 `$2.40` 上方、成交连续且点差可控时,才允许重算二突计划。
2. `GMM` 因收盘远低于盘中高点,只能算`快打命中,持有失败`;输出应更强调第一目标快速兑现和不持有。
3. `FAC` 是新上市/De-SPAC Web-only 候选;本地无日线结构时,必须标 C/D 审计,不得进入备攻计划。
4. 复盘时若股票最终大涨,也不能掩盖报价证据违规;流程评分与方向评分分开。

新增规则:

- 15 分钟延迟 movers 只能发现候选,不能生成完整交易计划。
- Web-only 新上市票缺本地结构时,只能列`审计/待实时复核`,除非用户券商或两个公开 quote 源互证。
- 复盘分类必须拆成`方向是否对`和`执行流程是否合规`;方向命中但报价违规,不得算完整命中。
- 对盘中高弹性票,若给的是条件二突,必须写`实时 quote 确认后重算`,不能把延迟价直接当触发价。

## 沉淀规则总览

来自上面案例的可执行规则:

1. Pump 总分 ≥ 15 → skill 拒绝推荐
2. Pump 总分 9-14 → 仓位上限 $50,标 🔴
3. Pump 总分 5-8 → 仓位上限 $200,标 🟡
4. 52W 高/现价 > 100 + 多次合股历史 → 自动归入"典型 pump-and-dump"
5. 无明确新闻源的微盘暴拉 → 举证责任反转
6. 资本事件 + 高 Pump 分 → 直接拒绝重仓
7. 空头 ≥ 15% 的轧空结构可与 pump 评分叠加判断,但仍走极小仓
8. 财报跳空类 +76% 案例不在日内雷达射程,归 swing 模式 5
9. 今日/盘前/盘中推荐必须先通过当前报价校验;用户报价与日线偏离 `>=5%` 时旧计划作废
10. 15 分钟延迟 movers/scanner 只能发现候选;没有 A/B quote 互证时不得给入场、止损、止盈、仓位或正常概率

更多规则见 [red-flags.md](workflow/red-flags.md)、[catalyst-stage.md](workflow/catalyst-stage.md)、[risk-rules.md](core/risk-rules.md)。
