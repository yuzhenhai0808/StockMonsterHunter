# 妖股输出合约

本文件是 `monster-stock-hunter` 的全局输出合同。凡是给出妖股、彩票仓、榜首、翻倍票、当日可操作候选,必须先满足这里的格式和自检要求。模式文件只补充筛选逻辑和少量场景字段,不得覆盖本文件的最终输出字段、用户可见输出顺序或主表结构。

## Canonical 输出架构

**唯一权威**: 本文件是本 skill 唯一的用户可见输出格式来源。其它 workflow / mode 文件可以定义:

- 候选如何筛选、排序、降级。
- 某个字段如何打分或计算。
- 某个场景必须补充哪些证据或备注。

其它文件不得定义新的首屏结构、不得要求用独立表格替代本文件的`最终候选总表 / 单票详情表 / 报价校验 / 概率表`。如果 mode 文件写了"本模式输出必须包含 X 表",解释为"把 X 字段并入本文件对应主表或放入`场景审计附录`",而不是改变默认输出顺序。

用户问普通推荐时,首屏必须先解决"买什么、怎么做、什么时候放弃",防漏审计和研究日志放在后面。**全景优先例外**: 用户问`+40% / 高涨幅 / 高回报 / 翻倍 / 最猛 / 彩票仓 / 极端`时,首屏必须先展示`当前盘前/盘中 40%+ 全景审计池`,再给最终候选总表;这类问题的第一失败模式是漏掉 MWC/SLXN 式已飞极端票,不是噪音过多。

本文件只负责**输出格式、表格字段、证据等级、自检清单**。各表格字段的判定口径在 sibling 默认必读文件中,不在本文件:

- `原型编号 / 早信号命中数 / 当前仓位段 / 升仓触发价 / 降仓触发价` 字段口径 → [../workflow/ignition-forecast.md](../workflow/ignition-forecast.md)
- `三连校验 / 微盘资本事件硬门` 字段口径 → [pre-recommend-triplet.md](pre-recommend-triplet.md)
- `催化硬度 A/A-/B/C/D/E` 字段口径 → [../workflow/catalyst-stage.md](../workflow/catalyst-stage.md) §催化硬度分层
- `红旗 / 红旗分` 字段口径 → [../workflow/red-flags.md](../workflow/red-flags.md)
- `结论池(预判/确认/已过热放弃)` 与 `因果分` 口径 → [../workflow/causal-first.md](../workflow/causal-first.md)
- `妖性评分` 口径 → [../workflow/causal-first.md](../workflow/causal-first.md) §妖性硬门

## 默认输出顺序

1. `一句话结论`: 先写首选票、是否当前可做、最关键触发价/失效价。没有 A/B 级候选时直接写"没有当前可买主攻,只给条件观察"。
2. `当前时段与报价基准`: 写明交易阶段、纽约时间或报价源 ET 时间、本地日线日期、最新报价、来源、是否用户券商、结论等级(`实时可交易 / 条件可交易 / 观察 / 报价冲突`)。
3. `当前盘前/盘中/AH 40%+ 全景审计池`: 仅在`+40% / 高涨幅 / 高回报 / 翻倍 / 最猛 / 彩票仓 / 极端`请求中前置必出;覆盖 Web movers Top 10、premarket Top 10、intraday Top 10、after-hours/overnight Top 10、本地最近完整日线 40%+、IPO/new listing 异动票,先展示再分层降级。`most traded / most active / unusual volume` 不能替代涨幅榜覆盖。
4. `预测评分总表`: 开放式`预测 / 今日 / 妖股 / 40%+ / 高回报 / 翻倍 / 最猛`请求必出,放在最终候选前。按 [../workflow/ignition-forecast.md §多维度权重综合评分](../workflow/ignition-forecast.md) 给 5-10 只候选打分,候选必须同时覆盖`纸面雷达候选`与`Web movers/40%+审计候选`;按综合分排序,不得按榜单涨幅排序。
5. `稳定三篮复核表`: 开放式`今日 / 妖股 / 翻倍 / 彩票仓 / 高弹性`请求必出;用户质疑推荐来回变、要求结合涨跌 Top、或出现抄底/修复诉求时必须前置到最终候选前。
6. `最终候选总表`: 池子合并后只给 1-3 只,并写清来源池;若没有 A/B 级且流动性过硬的票,改写为`最终观察/放弃表`,不强行凑主攻。
7. `持仓/破位处置表`: 用户已持仓、给出成本/截图、询问跑路/补仓/减仓,或候选最新价低于原触发价/承接区/硬止损时必出;该票必须从最终候选表移出,不能继续当新推荐。
8. `实时价存活/失败复核表`: 每次`妖股 / 彩票仓 / 翻倍 / 极端 / 今日可交易`请求必出,防止旧快照和榜单价混进最终入场计划。
9. `单票详情表`: 默认只展开前 1-2 只;每只 A/B 主攻/备攻必须表格化展开。C/D 观察票可压缩,但要写缺失项和降级原因。
10. `催化时间链表`: 每次把 PR/IR/SEC/合同/并购/融资/财报/FDA 作为强催化时必出;普通单票分析可压缩到 1 行,最终 A/B 候选必须完整表格化。
11. `单票价格概率表`: 每只 A/B 主攻/备攻必须有;C 级观察/条件触发票若来自`高弹性翻倍 / 极端彩票 / 榜首审计 / 彩票仓`请求,也必须给**触发后情景概率表**;D 级不估概率。
12. `报价校验与放弃备注`: 每只最终候选写报价来源;热门但未推荐的票写降级原因。今日/盘前/妖股/彩票仓请求必须额外说明`盘后点火/次日延续池`和`跌幅错杀修复池`是否扫描;若发现 HCWB 型盘后低基数点火票或 Top Losers 错杀候选而未推荐,必须写一句降级原因。
13. `场景审计附录`: 普通推荐中放在最终候选和单票计划之后。高弹性/40%+ 请求的`40%+ 全景审计池`不属于后置附录,必须按第 3 项前置;其它日志和研究表仍可后置。

复盘类请求的强制输出顺序: `推荐票结果表` → `当日 Top5 命中表` → `漏网原因表` → `skill 修正项表`。当日 Top5 按推荐后完整交易日实际涨幅排序;每只必须标注是否进入过`预测评分总表 / 40%+ 审计池 / 最终候选`,没有进入时写明`Web movers 未扫 / MCP no data 被误删 / 低量低基数漏掉 / 软催化红旗降级但未展示 / 报价冲突`等原因。

若回答缺少 `一句话结论`、`当前时段与报价基准`、`预测评分总表`、`稳定三篮复核表`、`最终候选表`、`报价校验` 任一项,必须补齐后再输出。无明确点名的妖股/第一口肉/预测请求仍必须生成`预测候选日志 / 点火预判池 / 因果链`;其中`预测评分总表`必须放在最终候选前,`预测候选日志`可放入场景审计附录。若用户问`+40%以上 / 高涨幅 / 大涨 / 高回报 / 翻倍 / 最猛 / 彩票仓 / 极端`,还必须在首屏补齐 `当前盘前/盘中/AH 40%+ 全景审计池`;不得只给较稳的二突候选而不展示已涨 40%+、AH/overnight 跳涨票或 IPO/new listing 榜单。若用户要求结合`涨的 Top / 跌的 Top / 抄底 / 修复`,必须同时展示`跌幅错杀修复池`或在稳定三篮里写明 Top Losers 被排除原因。若数据不足无法补齐,该票只能标为`观察/数据不完整`,不得给主攻、入场、仓位、建议本金、最大亏损或正常概率。

### 预测评分总表

开放式预测请求必须先输出本表。它是**预测依据**,不是新闻摘要;新闻只能作为`催化分`或`红旗扣分`的一列。

```md
| 票 | 来源 | 纸面雷达/派系 | 综合分 | 放量$ | 量比 | ATR | 微盘/低流通 | 深跌/位置 | 翻倍基因 | 催化硬度 | 红旗扣分 | 当前位置 | 证据等级 | 排名理由 |
|---|---|---|---:|---:|---:|---:|---|---|---|---|---|---|---|---|
| TICKER | MCP雷达/Web升级/40+审计 | A/B/C/混合 | XX/19 | X | X | X | X | X | X | A/B/C/无 | -X | 0-15/15-40/40+ | A/B/C/D | 为什么排在这里 |
```

硬要求:
- 表中至少包含 3 只 MCP/纸面雷达候选和 Web movers/40%+ 候选；如果某一侧没有数据,必须写明原因。
- 用户要求结合涨跌 Top 或抄底修复时,表中还必须至少包含 2-5 只 Web `top losers / premarket losers / after-hours losers` 或本地深跌候选；若没有合格错杀修复票,必须写明是`核心利空 / 无新利好 / 报价不足 / 红旗过重`哪一类排除。
- 若发现常规盘低量/收跌但盘后因 48h 内 PR/IR/SEC 公告跳涨 `>=50%` 的 AMST/HCWB 型票,必须列入本表的 `Web升级/盘后点火延续` 行；不能因 MCP 日线成交额低、`volume_ratio <1` 或本地 SQL 未入榜而省略。
- `综合分`是最终排序第一依据;`当前涨幅`只能影响`当前位置`分和审计展示,不能替代综合分。
- 盘前榜单 Top 1-3 若综合分低、红旗重或缺 bid/ask,必须排在观察/审计而非最终主攻。
- 最终候选若不是综合分最高者,必须解释为什么(例如证据等级、流动性、红旗或 R/R 让高分票降级)。
- 用户明确要求`十日内 / 10日内 / 很快看到结果 / 不想hold long`时,预测评分总表必须额外包含或紧跟一张`闭环周期分层表`,把候选分为 `T+0~10 快闭环 / T+11~40 中周期 / T+40+ 长周期`。最终候选只能来自 `T+0~10 快闭环`;中周期票只能列观察并说明下一验证节点。

```md
| 票 | 闭环层 | 下一验证节点 | 预计节点日期/窗口 | 为什么适合或不适合本轮 |
|---|---|---|---|---|
| TICKER | T+0~10快闭环/T+11~40中周期/T+40+长周期 | 订单/交付/财报/FDA接受/PDUFA/股东会/二突 | YYYY-MM-DD或区间 | 是否匹配用户持有周期 |
```

## 稳定三篮复核表

开放式`今日 / 妖股 / 翻倍 / 高弹性 / 彩票仓`请求必须输出本表。若用户质疑推荐来回变,本表必须前置到最终候选总表之前,用它解释为何保留、降级或换票。

```md
| 篮子 | 票 | 来源 | 最新价/时间/来源 | 48h 催化 | 结构状态 | 保留/换票理由 | 当前结论 | 下一次换票硬门 |
|---|---|---|---|---|---|---|---|---|
| 主升/点火候选 | TICKER | Gainers/点火预判/挑战者 | $X / HH:MM ET / 来源 | PR/SEC/财报/无 | 二突/回踩/VWAP上/破位 | 为什么继续排这里 | 条件可做/观察/降级 | 跌破 $X / 新红旗 / 新票多赢 2 项 |
| 抄底修复候选 | TICKER | Losers/深跌修复 | $X / HH:MM ET / 来源 | 正面但被砸/无 | 等 reclaim/站回/失败 | 为什么是错杀或为什么不是 | 条件观察/放弃 | 站回 $Y / 跌破 $Z |
| 只审计不交易 | TICKER | 40%+审计/无源/纯利空 | $X / HH:MM ET / 来源 | 无/利空/旧闻 | 过热/报价不足/核心利空 | 为什么不推荐 | 只审计不追/放弃 | A/B 报价 + 硬催化 + reclaim |
```

硬要求:
- `主升/点火候选`与`抄底修复候选`必须分开,不得把深跌票直接与涨幅榜票按涨跌幅排序混在一起。
- 抄底修复候选必须引用 [../modes/reclaim-bounce-hunter.md](../modes/reclaim-bounce-hunter.md) 的二次修复逻辑: 只允许`回踩做 / 二突做 / 等 reclaim / 放弃`,不得用"跌多了"作为买入理由。
- 若本轮最终候选和上一轮不同,必须在`保留/换票理由`写清旧票降级原因和新票胜出的至少 2 个维度。
- 若没有可用抄底票,也必须保留该篮子并写`无合格错杀修复`,不能省略 Top Losers 审计。
- 单纯榜单名次变化不是换票理由。

## 催化时间链表

凡是输出中使用`强 Web 消息 / 硬催化 / 48h 内催化 / 公告点火 / 并购 / 合同 / 融资 / 财报 / FDA`作为推荐或观察依据,必须输出本表或等价字段。该表的判定口径见 [web-cross-check.md §催化时间戳拆解硬门](web-cross-check.md)。

```md
| 票 | 事件/签署时间 | 原始 PR 时间 | SEC/交易所披露时间 | 媒体转发时间 | 行情反应时间 | 最早可交易公开时间 | 时间链结论 |
|---|---|---|---|---|---|---|---|
| TICKER | YYYY-MM-DD / 未找到 | HH:MM ET / 来源 | HH:MM ET / 8-K/6-K/无 | HH:MM ET / 媒体 | HH:MM ET / movers或quote | PR或SEC时间 | 新鲜硬催化/先涨后公告/媒体转发非首发/时间链不完整 |
```

硬要求:
- `原始 PR 时间` 与 `SEC/交易所披露时间` 至少尝试一个官方/准官方来源;两个都拿不到时,该催化最高 C 级。
- `媒体转发时间` 不得替代 `原始 PR 时间`;若只能拿到媒体时间,结论写`媒体转发非首发,待 IR/SEC 确认`。
- `最早可交易公开时间` 用于判断是否已经过热;若行情反应早于 PR/SEC,写`先涨后公告`,不得把该票升为强确认。
- 用户追问"几点发的 / 是不是刚发 / 为什么现在涨"时,优先输出本表,再给交易结论。

## 报价基准页眉(强制)

每次今日/盘前/盘中/妖股/彩票仓/翻倍请求,输出顶部必须先写:

```md
当前阶段: 官方盘前前 / 盘前 / 盘中 / 盘后 / overnight
本地日线日期: YYYY-MM-DD
最新报价基准: $X.XX / HH:MM ET / 来源 / 是否用户券商
本次结论等级: 实时可交易 / 条件可交易 / 观察 / 报价冲突
```

拿不到当前/延长交易报价时,`本次结论等级`只能是`观察`或`报价冲突`;不得把本地日线候选写成当前可交易推荐。
若价格来自 movers / top gainers / scanner 榜单,必须在`最新报价基准`写明`仅候选发现价,非当前 quote`;只有 quote 页面、两个公开 quote 源互证或用户券商价,才允许写成当前交易基准。
夜盘/盘前/盘后请求不得写`无法获取盘前价`作为默认结论;必须先按 [web-cross-check.md §延长时段价格获取强化](web-cross-check.md) 尝试用户券商/quote 页/第二公开源/movers 的取价链。若最终仍拿不到可交易 quote,页眉必须写清尝试过的来源和缺失字段。

## 证据等级与概率

### 推荐前置: 明确催化

`主攻 / 备攻 / 推荐 / 可买 / 建议本金 / 最大亏损 / 正常概率表` 的前置条件是: 已找到 48 小时内可追溯硬催化,且通过报价证据等级与当前流动性硬门。硬催化包括公司 PR/IR、SEC 8-K/6-K、财报/指引、FDA/临床结果、合同/订单、并购/控制权、监管/指数/政策等。

若只存在`当前涨幅 / 盘前成交量 / 低流通 / ATR / short float / 历史翻倍基因`,但没有明确利好或硬催化:

- 该票只能写`无源异动审计 / 观察 / 只审计不追 / 不推荐`。
- 不得写`主攻/备攻/推荐/当前可买`。
- 不得给`建议本金 / 最大亏损 / 正常概率表`。
- 高弹性请求中仍可给`条件触发后粗略涨幅概率`,但表头必须明确`无明确催化,仅审计情景`,且结论不能是推荐。

| 等级 | 定义 | 允许输出 |
|---|---|---|
| A 实时/夜盘可交易 | 用户券商实时报价,或两个公开源给出当前/延长交易报价,且有价格、时间戳、当前成交量/成交额和 bid/ask;同时通过 [web-cross-check.md §当前流动性硬门](web-cross-check.md) | 主攻/备攻、入场、仓位、正常概率表 |
| B 单源延长交易 | 一个公开源给出 after-hours / overnight / premarket 报价,且有明确时间戳、当前成交量/成交额和 bid/ask;同时通过 [web-cross-check.md §当前流动性硬门](web-cross-check.md) 的主攻/备攻最低门槛 | 可给条件主攻/备攻;必须标注单源报价,开盘前复核 |
| C 最近完整日线 + 新闻 / movers 发现价 | 只有本地最近完整日线和新闻催化,或只有 movers/榜单发现价但缺 quote 页、当前成交量/成交额或 bid-ask | 只能观察;可给条件触发后的粗略概率,不得写当前可买 |
| D 报价冲突/过期 | 公开源日期过旧、源之间冲突、只有昨收、无成交、无时间戳或价差无法解释 | 不给入场、仓位或概率 |

本地 `latest_prices`、`symbol_snapshot`、SQL scripts 的价格只能作为背景和候选初筛。A/B 之前不得输出`主攻/备攻`、`建议本金`、`最大亏损`或正常概率表;C/D 票即使出现在极端彩票榜首池,也只能是`观察/待盘前复核/报价冲突`。若只有价格与涨跌幅,但没有当前成交量/成交额或 bid-ask,不得升到 A/B;若相对量很高但当前绝对成交量/成交额不过硬门,必须降为`观察/极小仓触发`。

概率是交易路径校准,不是收益保证。必须用区间表达,例如 `25%-35%`。A/B 级必须给概率区间;C 级只能给`若站上/回踩确认后的粗略情景概率`;D 级写`报价不足,不估概率`。

**高弹性/彩票仓概率硬门**: 用户明确说`翻倍 / 本金翻倍 / 极端 / 彩票仓 / 最猛 / +50% / +100%`时,不能只给`先到第一止盈概率`。每只最终候选和每只 C 级条件观察票都必须额外给一张`涨幅阶梯概率表`,以`触发入场价`为起点,至少包含 `+10% / +20% / +50% / +100% / 先到硬止损` 五列。C 级表头必须写`条件触发后粗略概率`,不得写成当前可买或正常仓位概率;D 级仍写`报价不足,不估概率`。若没有 A 级候选,也必须输出 A/B/C 数量与最高等级候选的涨幅阶梯概率,不能用"观察"省略。

概率必须以`计划入场价`为起点,比较价格先触及`第一止盈`还是先触及`硬止损`,不要用"当天可能摸到目标"替代触发后的路径概率。估算依据至少引用 3 项: `证据等级 / 当前或延长报价来源 / ATR14% / 相对量 / 日内振幅 / 是否在 VWAP 上 / 催化硬度 / 红旗分 / 是否第 2-3 天`。

- `条件未触发`: 预测票后续上涨,但没有触发原计划触发价/报价硬门;不得算命中。
- `漏网`: 实际日涨 `>=40%`,且前一日/盘前已有可观察预测信号,但预测候选池没有列出。
- `误报`: 进入预测候选池且触发,但先到失效价或最大顺向收益 `<10%`。
- `不可提前发现`: 直到盘中/盘后突发公告才出现,或只有实时券商/延迟榜单可见,本地前一日数据预测分 `<4`。

### 预测验证摘要表

S8 输出不能只列个案,必须先给摘要,让用户看到规则到底有没有预测力。摘要必须把`纸面雷达`和`可交易确认`分开统计:

```md
| 窗口 | >=40%妖股样本数 | 纸面雷达命中数 | Web升级命中数 | 第一口肉命中数 | 接棒/审计命中数 | 漏网数 | 误报数 | 主要误差来源 | 规则动作 |
|---|---:|---:|---:|---:|---:|---:|---:|---|---|
| YYYY-MM-DD 至 YYYY-MM-DD | N | N | N | N | N | N | N | 盘前已飞/无催化/静态壳误报/报价不足 | 强化Web升级/不改/新增过滤 |
```

口径:
- `纸面雷达命中数`: 历史回放里进入预测候选日志,且目标日最大顺向收益 `>=20%`。
- `Web升级命中数`: 纸面候选在当时可通过盘前/盘中 Web gate 升级,且目标日最大顺向收益 `>=20%`。
- `第一口肉命中数`: 发现时涨幅 `0%~+15%`,触发后最大顺向收益 `>=20%`;已经 `>+15%` 不算第一口肉。
- `接棒/审计命中数`: 发现时已 `+15%` 以上或已过热,只能证明接棒/防漏能力。
- `漏网数`: 周内 `>=40%` 样本中,按当时可见数据应进入纸面雷达或 Web升级,但没有进入。
- `误报数`: 纸面雷达 `>=6` 且触发,但目标日最大顺向收益 `<10%` 或先到失效价。

## 推荐质量复盘表

用户要求`昨天 / 上次 / 最近一周 / 复盘 / 是否准 / 偏差原因 / skill 是否可信`时,必须输出本表,并按 [../workflow/causal-first.md §复盘闭环](../workflow/causal-first.md) 解释命中、失败、遗漏和可复用规则。

```md
| 票 | 推荐触发价 | 硬止损 | 第一/第二目标 | 实际开盘 | 实际最高 | 实际最低 | 实际收盘 | 最大顺向收益 | 最大逆向回撤 | 是否触发止损 | 是否达目标 | 命中/偏差原因 | skill 是否需修正 |
|---|---:|---:|---|---:|---:|---:|---:|---:|---:|---|---|---|---|
| TICKER | $X.XX | $Y.YY | $A/$B | $O | $H | $L | $C | +X.X% | -Y.Y% | 是/否 | 目标1/目标2/未达 | 量价/报价/退潮/红旗 | 是/否 + 规则名 |
```

计算口径:
- `最大顺向收益` = `(实际最高 / 推荐触发价 - 1)`.
- `最大逆向回撤` = `(实际最低 / 推荐触发价 - 1)`.
- 若实际开盘价已高于触发价,仍以推荐触发价计算策略质量,并补充开盘是否可成交。
- 若推荐条件未触发,不得把后续上涨算作命中;标为`未触发/遗漏`。
- 若先到第一目标后再跌破止损,结论写`快打命中,持有失败`,并检查是否需要强化快打退出纪律。
- 复盘只能评价规则质量,不得用事后最高价声称用户一定能吃满。

## 账户收益现实校准

用户要求`本金翻倍 / 本金 40%+ / 账户收益 40% / 回本翻本`时,必须同时把**单票涨幅**换算成**账户收益**:

```md
| 单票仓位占账户 | 单票 +20% | 单票 +40% | 单票 +50% | 单票 +100% |
|---:|---:|---:|---:|---:|
| 5% | +1.0% | +2.0% | +2.5% | +5.0% |
| 10% | +2.0% | +4.0% | +5.0% | +10.0% |
| 20% | +4.0% | +8.0% | +10.0% | +20.0% |
```

结论必须说明:单票 `+40%` 不等于账户 `+40%`。账户 `+40%` 需要高仓位或连续多笔命中,与妖股彩票低胜率冲突;不得为了迎合目标暗示满仓。

## 报价校验

最终表后必须加报价校验:

```md
| 票 | 昨收 | 最新价 | 报价时间 | 来源 | 成交量/Bid-Ask | 相对昨收 | 可交易性 |
|---|---:|---:|---|---|---|---:|---|

```

若公开源之间价差或涨跌幅冲突超过 `3%`,或用户报价与网页报价冲突但无法解释时间戳,该票只能标`报价冲突/只观察`。

用户提供券商报价后,该报价优先级最高。若用户报价或实时/延长报价相对本地日线收盘价偏离 `>=5%`,原日线入场、止损、止盈、建议本金、最大亏损、概率和 R/R 全部作废,必须基于最新报价重算;不能只在备注中说明价格变化。

若用户报价或实时/延长报价已经触及或跌破上一版计划的硬止损 / 降仓触发价 / 结构失效价,原计划必须标`作废`;不得把旧止损价改口成"低吸入场价"。只有价格重新站回 VWAP、开盘价、盘前承接区或前高 reclaim,并且基于新报价重算 R/R 后,才允许输出新的触发计划。

每次给出当前操作建议前,必须在报价校验后追加一行`最新价状态判定`,按 [web-cross-check.md §最新报价驱动决策状态机](web-cross-check.md) 把最新价映射为: `旧计划有效 / 旧计划作废 / 等回踩 / 等二突 / 只观察 / 持仓处置`。不得只展示最新价而不改变策略。

- 原型 4(微盘公告窗口)候选是否跑完 [pre-recommend-triplet.md §微盘 biotech 资本事件扩展硬门](pre-recommend-triplet.md) 7 天/30 天检查;命中一票否决项必须放弃,不得给主攻/备攻。
- 每只主攻/备攻是否用计划入场价 → 第一止盈 → 硬止损 计算 R/R,且 `R/R >= 2:1`(主攻)或 `>= 1.5:1`(极端彩票/妖股,显式标注);`R/R < 1.5` 不得给主攻/备攻。
- 每只主攻/备攻是否检查止损距离 `>= 0.4 ATR14`;不达标时是否已放宽止损到 `0.5 ATR14` 后重算 R/R 与建议本金。
- **(全表格化)** 每只 A/B 主攻/备攻是否使用了 §单票详情表 的表格格式;**不允许出现散文段落短评**;字段拿不到时是否写明来源失败并降级,而不是编造。
- **(MCP 调用记录)** 每只 A/B 主攻/备攻的单票详情表里 `MCP 调用记录` 一行必须列出已完成的 MCP 工具;原型 2/4 候选额外列 `earnings_dates`;高弹性 mode 候选额外列 `hyper_elastic_audit.sql`。MCP 调用失败时按 [mcp-fallback.md](mcp-fallback.md) 三层降级标注,不允许假装已调用。
- **(确定性 tie-breaker)** 候选排序是否按 [../workflow/ignition-forecast.md §候选排序确定性 tie-breaker](../workflow/ignition-forecast.md) 7 级硬规则逐级应用;输出是否显式写出哪一级 tie-breaker 决定了排序;**不允许**用"我感觉" / "看起来更干净" / "用户点名"作为排序理由。
- **(关键字段完整性)** 每只 A/B 候选的流通盘 / short float / ATR14% / 量比 / 当前 quote + 时间戳 / 当前成交量或成交额 / bid-ask 或 spread / 7 天红旗状态 / 历史财报反应(原型 2/4)是否全部有具体值;关键字段拿不到时,该票必须降为 C/D 观察。
- 若用户提供的最新价已触及旧硬止损/失效价,是否已撤销旧计划,并避免把旧止损改成新入场。
- 每只主攻/备攻是否有价格概率表。
- 若用户要求`翻倍/极端/彩票仓/最猛`,是否输出了`+10% / +20% / +50% / +100% / 先到止损`涨幅阶梯概率;C 级条件观察票是否也给了`条件触发后粗略概率`,且没有写成当前可买。
- 是否在最终候选表之前输出了`实时价存活/失败复核表`;偏离 `<= -8%` 是否降为`等reclaim`,偏离 `<= -15%` 或低于硬止损是否标`作废`;作废票是否已移除主攻/备攻/建议本金。
- 复盘类请求是否输出了`推荐质量复盘表`,并区分`快打命中,持有失败`与`完全失败`。
- 用户要求账户 40%+/翻倍时,是否输出了账户收益换算,并说明单票涨幅不等于账户收益。
- A/B/C/D 证据等级是否决定了能否给概率和仓位。
- C/D 票是否被阻断为观察/数据不完整,且没有主攻/备攻、建议本金、最大亏损或正常概率表。
- 是否同时写了建议本金和最大亏损。
- 是否有报价校验表。
- 是否解释了热门但放弃/降级的票。
- 若是今日/盘前/妖股/彩票仓请求,是否明确扫描并展示或说明了`盘后点火/次日延续池`与 `after-hours/overnight gainers` 涨幅榜;若未完成,是否在顶部降级声明可能漏掉 RGNT/HCWB 型票。
- 仓位段是否按 ignition-forecast §三段加仓状态机 顺序升级(预判 1/3 → 确认 2/3 → 主攻满仓),无跳级。
