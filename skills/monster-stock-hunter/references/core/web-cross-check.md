# Web 交叉认证与逐步交易决策

本文件用于所有推荐、加仓、减仓、补仓、止损和复盘场景。目标是避免只听用户描述、只看本地日线、或只看单一网页源就给交易动作。

## Web + MCP 分工矩阵 (canonical, 2026-05-15 加固)

Web 与 MCP **不是替代关系,是交叉验证关系**。Web 负责"实时变化",MCP 负责"量化结构 + 历史 + 基本盘"。任何最终候选必须**双源都跑过**,只跑一边都是执行不完整。

### 分工矩阵(每个场景必跑哪个,不允许交换)

| 场景 | 负责数据源 | 必调工具 | 不允许只用对面替代 |
|---|---|---|---|
| 当前 movers 榜 / 盘前/盘后涨幅榜 | **Web** | WebSearch + WebFetch | MCP 没有实时榜单 |
| 候选 ticker 当前 quote / 时间戳 / Bid-Ask | **Web** | WebSearch + WebFetch | MCP 只有上一交易日收盘 |
| 48h 内催化新闻 / 8-K / PR / 临床 / 合同 | **Web** | WebSearch + WebFetch (IR/SEC/财经媒体) | MCP 不索引新闻 |
| 7 天红旗(ATM / 反拆 / S-1 / going concern) | **Web** | WebSearch + WebFetch (SEC EDGAR 优先) | MCP 不直接索引 SEC 文件 |
| **基本盘(流通 / short float / 市值 / 52 周高低)** | **MCP** | `company_fundamentals` 或 `query_market_data` 查 `us_equity_company_profile` | Web 数据可能滞后或来源不一致 |
| **量化结构(ATR14 / 量比 / 20 日位置 / RSI)** | **MCP** | `symbol_snapshot` 或 `latest_prices` | Web 数据通常没有标准化量化指标 |
| **历史翻倍基因(180 天单日 +50%/+100% 次数)** | **MCP** | `query_market_data` 跑 SQL 或 `hyper_elastic_audit.sql` | Web 不提供结构化历史 |
| **历史财报反应(过去 4 次次日反应)** | **MCP** | `earnings_dates` + `query_market_data` 跨表 join 财报日 ±1 日涨跌 | Web 数据散且不标准 |
| **未来 0-3 天事件日历(财报/FDA)** | **MCP** | `earnings_dates(within_days=3)` | Web 检索遗漏严重 |
| **本地防漏 SQL** | **MCP** | `leaderboard_audit.sql` / `hyper_elastic_audit.sql` / `reclaim_bounce_audit.sql` | Web 无法做结构化扫描 |
| 用户已亏损/被套的成本和盘口处置 | **用户输入 + Web 复核** | 用户截图 + WebFetch 当前 quote | MCP 只能当背景 |

### 当前流动性硬门 (canonical)

盘前/夜盘/盘后推荐必须先过**当前可交易流动性**硬门。`volume_ratio`、20 日均成交额、昨日爆量和本地 SQL 成交额只能证明历史活跃,不能替代当前盘前/盘中能否成交。

### 明确催化硬门 (canonical, 2026-05-21)

今日/盘前/盘中/妖股/彩票仓请求中,**明确催化是推荐前置条件**。涨幅榜、低流通、盘前放量、ATR、高 short float、历史翻倍基因只能生成`候选/审计池`,不能单独生成`推荐/主攻/备攻`。

可升级为推荐候选的催化必须满足:

- 48 小时内可追溯到公司 PR/IR、SEC 8-K/6-K、财报/指引、FDA/临床结果、合同/订单、并购/控制权、监管/指数/政策等来源之一。
- 能说明"为什么今天有资金反应",而不是旧闻、泛题材、论坛传闻或只有榜单涨幅。
- 催化源与 ticker 匹配,且发布时间/文件日期明确。

处理规则:

- 找到硬催化 + 报价 A/B + 流动性硬门通过 → 才允许写`主攻/备攻/推荐/可买`。
- 找到硬催化但报价或流动性不足 → `条件观察`,只给触发条件。
- 没找到硬催化 → `无源异动审计/只观察/不推荐`;不得给建议本金、最大亏损、正常概率或"推荐"措辞。
- 用户问 `40%+ / 翻倍 / 最猛 / 高回报` 时,无源异动票仍必须进入`40%+ 全景审计池`,但分类必须写`只审计不追`或`报价冲突`,并在备注中写明`未找到明确利好`。

最低 WebSearch 调用中第 3 类`48h 内催化新闻`如果失败,不得用第 1 类 movers 结果替代;最终结论必须从`推荐`降为`观察/审计`。

### 催化时间戳拆解硬门 (canonical, 2026-05-22)

美股公告常见流程不是单一时间点,必须拆开记录。凡是把新闻/公告作为`硬催化`、`强 Web 消息`、`48h 内催化`、`盘后点火`、`合同/并购/融资/财报/FDA`依据时,必须至少尝试拆解下面 5 类时间,不得只写"今天发布"或把媒体转发时间当作原始公告时间:

| 时间层 | 定义 | 优先来源 | 用途 |
|---|---|---|---|
| 事件/签署/董事会时间 | 协议签署、董事会批准、临床/监管实际事件发生日期 | SEC 8-K/6-K exhibits、公司 IR 原文、合同摘要 | 判断消息是否新鲜、是否已提前泄露 |
| 原始 PR 发布时间 | PRNewswire/GlobeNewswire/BusinessWire/公司 IR 首发时间,统一换算 ET | 公司 IR、PR wire 原文、Nasdaq press release | 判断资金第一反应窗口 |
| SEC/交易所披露时间 | 8-K/6-K/424B/S-1 等 filing acceptance time | SEC EDGAR、StockTitan SEC filings、Nasdaq filings | 判断文件硬度和红旗 |
| 媒体转发时间 | Investing.com、MarketWatch、Benzinga、Yahoo 等二次转发时间 | 媒体页面 | 只能做传播热度,不能当原始催化时间 |
| 行情反应时间 | 盘前/AH/盘中 movers 首次出现或 quote 时间戳 | quote 页面、movers 页面、用户券商盘口 | 判断是否已过热、是否还有未定价空间 |

强制处理:

- 若能找到 SEC/IR/PR 原始来源,必须以其时间作为`催化发布时间`;媒体时间只写`转发时间`。
- 若协议签署日早于 PR 日,写成`签署于 YYYY-MM-DD, 公告于 YYYY-MM-DD HH:MM ET`,不要把签署日误当成市场知道的时间。
- 若只有媒体转发、没有 IR/PR/SEC 原文,证据等级最高 C,不得写`强硬催化`或`A/B 主攻`。
- 若 PR 发布时间晚于盘前/盘中异动开始,必须标`先涨后公告/疑似预期或泄露`,从确认池降一级,除非能找到更早的 SEC/IR 来源。
- 若 SEC filing 时间晚于 PR 时间,PR 可作为催化起点,SEC 作为硬度确认;若 SEC filing 早于 PR 时间,SEC acceptance time 是最早可追溯公开时间。

建议 WebSearch 模板:

```text
"{ticker}" "{event keyword}" "PRNewswire" OR "GlobeNewswire" OR "BusinessWire"
"{ticker}" "8-K" OR "6-K" "{date}"
site:sec.gov/Archives "{ticker}" "{date}"
"{company name}" "investor relations" "{event keyword}" "{date}"
"{ticker}" "stock" "premarket" "{date}" "volume"
```

每只最终候选或用户点名催化票,内部必须形成:

```text
catalyst_timeline = event/signed? -> original PR? -> SEC/filing? -> media repost? -> quote/movers reaction?
```

缺失项可以写`未找到`,但不能省略该时间层的尝试。该 timeline 优先级高于新闻摘要;时间链不清时,结论降为`观察/待确认`。

### 延长时段价格获取强化 (canonical, 2026-05-20)

夜盘、盘前、盘后价格不是"不可获取"。对今日可交易结论,默认必须主动尝试获取当前可见的 extended-hours quote;只有尝试失败、源冲突或缺成交字段后才允许降级。不得因为 MCP 只有上一完整日线,就写"盘前价格未取/无法获取/只能等开盘"。

按优先级取价:

1. 用户券商盘口或截图:最高优先级;必须记录价格、时间戳、成交量或 bid/ask。
2. 公开 quote 页:Nasdaq、MarketWatch、Yahoo、StockAnalysis、Stocknear、ADVFN 等,优先选择能显示 premarket / after-hours / overnight 价格、更新时间、成交量或 bid/ask 的页面。
3. 实时/准实时 movers 页:StockAnalysis / Nasdaq / MarketWatch / StockTitan / Stocknear / ADVFN / 券商榜单,用于发现候选和估计涨幅;若无成交量/bid-ask,只能作为`候选发现价`。
4. 新闻/IR/SEC 中的盘后反应描述:只能证明催化和方向,不能当交易基准。
5. 本地日线/MCP:只作为前收、ATR、量比、流通和历史背景。

每只进入最终候选或 40%+ 审计池的票,必须输出或内部记录一次`取价尝试链`:

```text
quote_attempts = 用户券商? → quote页? → 第二公开源? → movers? → 新闻? → MCP背景
```

升级规则:

- `quote 页 + 第二公开源` 在同一交易阶段价格差 `<3%`,且至少一个源给出成交量/成交额或 bid/ask → 可升 A/B,再看流动性硬门。
- 单一 quote 页有明确 extended-hours 价格、时间戳、当前成交量/成交额和 bid/ask → 可升 B,再看流动性硬门。
- 只有 movers 榜单价格或涨幅,但没有 quote 页/成交量/bid-ask → 最高 C;可以列入审计池,不得给主攻/备攻、建议本金、最大亏损或正常概率。
- 搜到 quote 页但显示 regular close 与 movers 盘前价冲突 `>=3%` → D 报价冲突;必须写明哪个源是 regular close、哪个源是 premarket/AH。
- 如果公开源拿不到 bid/ask,但用户券商截图有 bid/ask,以用户券商为 A 级候选报价,同时标注公开源无法互证。

失败时必须具体写失败点,例如`已查 StockAnalysis/Nasdaq/Yahoo,只有 movers 发现价,没有当前成交量/bid-ask`,而不是笼统写`未取`。

### 盘前榜单快照多时间戳规则 (canonical, 2026-05-20)

盘前/夜盘 movers 是高速变化的发现源,同一票在 `04:17 ET`、`05:30 ET`、`09:05 ET` 的价格可能完全不同。输出必须记录每个榜单或 quote 的时间戳;若同一票不同快照价格偏离 `>=8%`,必须触发 §实时价存活闸门,用最新可复核 quote 重算`可执行 / 等reclaim / 作废 / 报价冲突`。

处理:
- 旧榜单价只写作`候选发现价`,不得继续当入场价。
- 最新价较发现价回撤 `>=8%` → 至少降为`等reclaim`;回撤 `>=15%` 且无 reclaim → `作废/退潮减仓`。
- 若一个来源显示 regular close、另一个来源显示 premarket/AH,必须标清交易阶段;无法解释时按 D 报价冲突。
- 对用户已持仓票,用户券商截图价优先于公开榜单;若券商价已破位,立即切换到持仓/破位处置,不能继续用旧推荐价安慰持仓。

### Movers 必展示但不替代 Quote

Web movers / top gainers 不能替代可交易 quote,但高弹性请求也不能省略 movers。用户问`+40% / 翻倍 / 最猛 / 高回报 / 彩票仓`时,即使公开源缺 bid/ask、成交额或双源互证,也必须把 movers Top 10 中所有 `>=40%` 或 Top 3 极端票放进`40%+ 全景审计池`,证据等级标 C/D 并降级。禁止因为无法升 A/B 就不展示 MWC/SLXN 式榜首票。

### 盘中突发 Web 升级硬门 (canonical, 2026-06-09)

对`今天 / 今日 / 翻倍 / 最猛 / 高回报 / 彩票仓 / 极端`请求,Web 第 1 类 movers 发现必须覆盖三张榜: `当前 movers Top10`、`premarket Top10`、`intraday Top10`。任一来源显示候选满足 `当前涨幅 >=40%`、`当前成交额 >=$10M`、`成交量 >=10M 股` 或进入 Top 3,必须先进入`40%+ 全景审计池`,再做报价、催化、红旗和结构降级。

本硬门高于 MCP 本地过滤:

- `symbol_snapshot` 返回 `no data`、fundamentals 缺失、最近完整日线低量/收跌、或 SQL 未入榜,都不能作为删除理由。
- 缺 MCP 结构时,证据等级最高 C,分类写 `Web-only C/D 审计` 或 `Web升级观察`;不得给正常主攻/备攻、建议本金、最大亏损或正常概率表。
- 若 Web 只给涨幅/榜单价,缺当前成交额、bid-ask 或时间戳,仍必须展示,但操作结论写`只审计不追 / 报价冲突 / 等 A/B quote`。
- 典型校准样本: `INHD 2026-06-08` 本地日线低量且 `symbol_snapshot` 无数据,盘中因 AI development deal 爆发;`SUNE 2026-06-08` 盘前 reverse merger 催化;`NPT/BYAH/TDIC 2026-06-08` 极端低流通/动量票。它们即使不适合主推,也必须入审计池。

### After-hours / overnight 涨幅榜独立硬门 (canonical, 2026-06-09)

`after-hours gainers / post-market gainers / overnight movers` 是独立发现源,不能被 `most traded / most active / unusual volume` 或本地常规盘涨幅榜替代。若用户问`今天 / 明天 / 盘前 / 夜盘 / 妖股 / 翻倍 / 最猛 / 高回报 / 彩票仓`,Web 第 1 类 movers 必须额外覆盖最近一次 AH/overnight 涨幅榜 Top10 或 Top3 + 4-15 名接棒池。

硬规则:

- 常规盘收跌、低量、低成交额、`symbol_snapshot` 低分或 SQL 未入榜,都不能删除 AH 点火票。
- AH/overnight 相对常规盘收盘 `>=40%`,或进入 AH/overnight Top3,或 AH/overnight 成交额 `>= $1M`,必须进入`盘后点火/次日延续池`和`40%+ 全景审计池`。
- 只打开 `most traded / most active / volume leaders` 不算完成涨幅榜扫描;这些页面只能补充流动性,不能证明没有漏掉涨幅 Top。
- 若只能拿到社媒/第三方 watchlist 的 AH 发现价,该票仍要展示为 `Web-only C/D 审计`,再用 quote 页、SEC/PR、成交额、bid/ask 和红旗决定是否升级。
- 典型校准样本: `RGNT 2026-06-08` 常规盘 `-3.03%` 且 MCP 成交额很低,但盘后因 6-K/PR 宣布 GelrinC 欧洲 surgeon training 跳涨;正确处理是入`盘后点火/次日延续池`,不是被常规盘 top gainers 或 most-traded 扫描漏掉。

### IPO / New Listing Fallback

若 `company_fundamentals` 返回 `no fundamentals data`、本地日线历史不足 `10` 个交易日、或 ATR/流通字段为空,先判断是否是 IPO / new listing / ADS listing / recently listed:

- 若 Web IPO/IR/Nasdaq/StockAnalysis 证明上市 `<=10` 个交易日,且盘前/盘中涨幅 `>=40%`、成交额 `>= $5M`、或进入 movers Top 10,必须进入`40%+ 全景审计池`或`极端彩票榜首池`。
- 本地数据缺失只能让证据等级最高到 C,不能作为删除理由。
- 只有用户券商盘口或两个公开 quote 源在同一交易阶段互证,且当前流动性硬门通过,才允许从 C 升为 A/B。
- 分类默认写`IPO squeeze / new listing squeeze`;操作结论默认`已飞等回踩`或`只审计不追`,除非出现回踩承接、二突或 LULD 恢复后站稳。

按交易阶段判定:

| 交易阶段 | 主攻/备攻最低门槛 | 只允许极小仓触发 | 观察/拒绝 |
|---|---|---|---|
| 盘前/夜盘/盘后 | 当前延长交易成交量 `>=300k` 股 **或** 成交额 `>= $1M`; 最近成交 `<=5` 分钟; bid-ask spread `<=2%`; 成交不是单笔孤岛 | 当前成交量 `100k-300k` 股 **或** 成交额 `$300k-$1M`; spread `<=3%`; 只能账户风险 `0.25%-0.5%` | 低于 `100k` 股且低于 `$300k`,或几分钟无成交,或 spread `>3%` |
| 09:30-09:45 ET | 默认观察;除非成交额已 `>= $3M` 且站稳开盘价/VWAP | 可极小仓二突,但必须成交连续 | 开盘 5 分钟内市价追单一律拒绝 |
| 09:45 ET 后 | 当前日成交额 `>= $5M` 且最近 5 分钟有连续成交; spread `<=2%` | 成交额 `$1M-$5M` 只能小仓战术 | 低于 `$1M` 或成交断续只能观察 |

关键规则:

- **相对量不能替代绝对量**: 例如 20 日均量很低导致 `volume_ratio >100x`,但盘前实际成交只有几万股,不得升主攻/备攻。
- **单源报价不能自动升 B**: 若 quote 页面只有价格和涨跌幅,没有当前成交量/成交额或 bid-ask,最多 C 级观察。
- **低流通不是流动性**: float 很小只说明弹性大,也说明滑点和停牌风险更高;必须用当前成交额和点差证明能进能出。
- **当前流动性不达标时**: 即使催化硬、R/R 足、A 级翻倍基因,也必须降为`观察/极小仓触发`,不得写`主攻/备攻`、正常仓位或正常概率。

### 调用顺序(固定不可调换)

每次妖股/彩票仓/点火预判请求,按下面顺序逐项跑完;**任一项跳过 = 执行失败**,必须重发:

1. **Web 第 1 类**:当前段 movers 榜(详见 §最低 WebSearch 调用清单)→ 拿到候选 ticker 池。高弹性/翻倍/最猛请求必须额外覆盖`当前 movers Top10 + premarket Top10 + intraday Top10 + after-hours/overnight Top10`;所有 `>=40%`、进入任一时段 Top3、或成交额 `>=$10M` 的票先入审计池,不得等 MCP 通过后才展示
2. **MCP 量化筛**:对每只 Web 候选并行调 `symbol_snapshot` 拿 ATR/量比/位置/RSI;调 `company_fundamentals` 拿流通/short/市值/52周高
3. **MCP 历史**:对原型 2/4 候选并行调 `earnings_dates(within_days=3)` 拿未来事件;调 `query_market_data` 跑 [hyper_elastic_audit.sql](../../scripts/hyper_elastic_audit.sql) 验证翻倍基因
4. **Web 第 2 类**:对 Top 5 候选 WebFetch 当前 quote 拿时间戳和成交量
5. **Web 第 3 类**:对 Top 5 候选 WebSearch 48h 催化新闻
6. **Web 第 4 类**:对 Top 5 候选 WebSearch 7 天红旗(SEC EDGAR + 公告)

**不允许只跑 1+4+5+6**(只 Web 不 MCP) — 那样基本盘和量化结构是猜的;**也不允许只跑 2+3**(只 MCP 不 Web) — 那样错过当前催化和实时报价。

### MCP 数据**不**能做的事

- 不能给"当前可买"入场/止损 — 那是 Web 实时报价的职责
- 不能确认"今天有什么催化" — 那是 Web 新闻和 SEC 实时的职责
- 不能识别"刚刚出现的 ATM 或 offering" — 那是 Web 7 天红旗扫描的职责

### Web 数据**不**能做的事

- 不能替代流通盘和 short float 数字(Web 数据滞后或不一致,MCP 是单一来源)
- 不能替代 ATR14% 和 20 日量价结构(Web 没有标准化字段)
- 不能替代历史财报反应统计(Web 散且需要人工聚合)
- 不能替代未来 N 天财报日历(Web 检索严重遗漏)

### 拿不到数据时的硬规则

- **不允许用空占位符代替数据来源**;拿不到字段时写明已尝试来源、失败原因,并按证据等级降级
- **MCP 调用失败 / db_health 报错 / 用户说"数据库挂了"时,加载 [mcp-fallback.md](mcp-fallback.md) 走 L0/L1/L2 三层降级**;默认请求**不**加载该文件以节省 token

## 最低 WebSearch 调用清单 (硬门)

每次妖股 / 彩票仓 / 点火预判请求,**至少完成下面 4 类调用**才算扫描完整;少一类就不能写主攻/备攻/可买结论:

| 类别 | 关键词模板(替换 `{date}` 为当日,`{ticker}` 为候选) | 用途 |
|---|---|---|
| 1. 当前段 movers 榜 | `"premarket gainers {date}"` / `"intraday top gainers {date}"` / `"top gainers stocks today {date}"` / `"after hours gainers {date}"` / `"post market gainers {date}"` / `"overnight movers stocks {date}"` / `"StockTitan top gainers {date}"` / `"StockAnalysis after hours gainers {date}"` / `"Reddit StockTitan {date} top gainers"` | 拿当前/premarket/intraday/AH/overnight Top 10 + 4-15 名(原型 1 接棒池);高弹性请求中所有 `>=40%`、任一时段 Top3、或成交额 `>= $10M` 先入审计池。`most traded/most active/unusual volume` 只能辅助流动性,不得替代涨幅榜 |
| 2. 候选 ticker quote | `"{ticker} stock premarket"` / `"{ticker} stock price today"` / `"{ticker} after hours"` | 拿候选当前价 / 时间戳 / 成交量 / Bid-Ask;升级 C → B/A 证据等级 |
| 3. 48h 内催化新闻 | `"{ticker} stock news today"` / `"{ticker} 8-K {date}"` / `"{ticker} earnings results"` / `"{ticker} FDA"` / `"{ticker} contract award"` | 找硬催化(财报/合同/FDA/控制权交易/8-K);确认原型分类 |
| 4. 7 天红旗扫描 | `"{ticker} offering ATM dilution"` / `"{ticker} reverse split"` / `"{ticker} S-1 EFFECT"` / `"{ticker} SEC investigation halt"` | 防止把反拆/ATM/利好出尽包装成真催化 |

每类至少 1 次成功调用。**调用不发起 = 执行失败**,模型必须显式打"今天没扫,我现在去扫",而不是"未取/缺数据/只能 C 级"自我开脱。

### 盘后公告点火强制 Web 升级 (AMST 型)

若 Web movers / 新闻源显示某票在最近一次常规盘低量、收跌或未上榜,但收盘后 48 小时内出现可追溯公司 PR/IR/SEC 公告,且 after-hours / overnight / premarket 涨幅 `>=50%` 或相对常规盘收盘价偏离 `>=40%`,该票必须进入`盘后点火/次日延续池`和`预测评分总表`的 Web 升级侧。不得因为 MCP 日线 `volume_ratio <1`、成交额低、收盘价未动或本地 SQL 未入榜而静默删除。

处理顺序:

1. 先记录 Web 发现价、公告时间、延长交易涨幅和来源。
2. 再查当前 quote / 成交量 / 成交额 / spread,决定 A/B/C/D 证据等级。
3. 同步查 MCP `symbol_snapshot` 和 `company_fundamentals`,补 ATR、低流通、市值、现金和历史结构。
4. 若当前流动性不足或只能拿到单一榜单快照,结论写`条件观察/等开盘确认`,但仍必须展示触发价、失效价和未推荐原因。

典型样本: `AMST 2026-05-18` 常规盘低量收弱,盘后因 NurseMagic 企业客户 PR 跳涨约 `+90%`;正确处理是 Web 升级候选/盘后点火延续,而不是被本地日线低成交额过滤。

## 报价时间戳硬要求

输出顶部 `报价基准页眉`(详见 [output-contract.md §报价基准页眉](output-contract.md))中的"最新报价基准"字段,必须来自**当次 WebSearch** 调用拿到的数据(含具体源 + ET 时间戳)。

- 不允许写"未取" → 改成"WebSearch 已尝试 X 次,获取到的最近报价是 [价格] / [时间] / [源]"
- 不允许只填本地日线收盘价 → 那是背景,不是当前报价基准
- 若处于官方盘前前、夜盘、盘前或盘后,必须先说明是否查到 extended-hours quote;查到则写`夜盘/盘前/AH quote`,未查到则写`仅 movers 发现价/仅 regular close`并降级。
- 若 4 类调用全部失败,顶部必须打:"⚠️ Web 实时调用全部失败 — 本次只能给历史扫描背景,不能给当前可交易计划"

## 三方认证硬门

每次给具体操作前,必须整理三类证据:

| 类别 | 必填字段 | 用途 |
|---|---|---|
| 用户信息 | 成本、仓位占比、已成交价、券商截图/盘口时间 | 作为最高优先级的实盘输入 |
| Web/公开源 | 最新价、时间戳、当前成交量/成交额、bid-ask 或 spread、最近成交连续性、日内高低点、新闻/SEC/IR、红旗 | 防止用户信息过期或误读,并判断能否实际成交 |
| 本地数据 | 前收、ATR、量比、20 日位置、财报窗口、基本面/short float | 判断结构、波动和风险边界 |

## 当前可交易性硬门

本地 `latest_prices` / `symbol_snapshot` / SQL scripts 只代表最近一个完整日线或本地初筛结果,**不能当作盘中、盘前或夜盘可交易报价**。凡是用户问`今天 / 今日 / 盘前 / 夜盘 / 开盘 / 妖股 / 彩票仓 / 翻倍 / 现在买 / 止盈概率 / 入场止损`,每只最终候选必须先通过当前可交易性检查。

### 证据等级闸门

每只候选必须先标注报价证据等级 `A / B / C / D`,再决定能否推荐。等级定义、条件和允许输出按 [output-contract.md §证据等级与概率](output-contract.md) 执行,本文件不重复;只有 A/B 且通过 §当前流动性硬门 才允许 `主攻/备攻`、具体入场计划、建议本金、最大亏损和正常概率表;C 只能列为`观察/待盘前报价`;D 只能列入`数据不完整/报价冲突`。

延迟源识别(本节细化):

- 榜单或 quote 页面无明确时间戳: 默认按 C/D 处理,不能升 A
- 时间戳早于当前时间 30 分钟以上(在交易时段内): 只能给 B,且必须标注`延迟报价,开盘/盘中复核`
- 多个延迟源对最新价/涨跌幅互相冲突 `>=3%`: 强制 D,只观察

### 报价获取顺序

固定不可调换:

1. 先查当前/延长交易 movers 榜单与候选 ticker 的 quote 页面(Nasdaq / MarketWatch / Yahoo / ADVFN / StockAnalysis / Stocknear / 券商可见报价)
2. 再查新闻催化、公司 IR、SEC/8-K 和红旗
3. 最后才用本地日线、ATR、量比、20 日位置做技术背景

不能先基于日线价给入场、止损、止盈、仓位或概率表,再补一句"需实时复核"。
拿到 movers / top gainers 榜单后,必须立刻对最终候选逐票打开 quote 页面或使用用户券商盘口,获取**当前时间点最新可交易价**;榜单中的价格只允许写作`候选发现价`,不得写作`最新报价基准`。

延长时段取价补充:如果某个网页同时显示 regular close 和 premarket/AH 价,必须选用与当前交易阶段一致的 extended-hours 字段作为交易基准,并把 regular close 只写进`前收/背景`。如果页面只显示 regular close,但 movers 页面显示 premarket/AH 涨幅,必须继续找第二个 quote 源或用户券商价;不能把 regular close 当成当前价。

### Movers 快照不得当作最新成交价 (canonical)

对盘前/盘中涨幅榜 Top 1-10、极端彩票、低价股和用户已持仓票,`movers / top gainers / StockTitan scanner / Public榜单 / Reddit转贴`只能用于**发现候选和涨幅背景**,不得单独作为最终`最新报价基准`。最终交易基准必须来自以下之一:

1. 用户券商实时价或截图时间戳。
2. 可打开的 quote 页面,同时有价格、更新时间、成交量或 bid/ask。
3. 两个公开 quote 源在同一交易阶段内价格差 `<3%`。

若只能拿到 `15 min delay`、榜单快照、盘前旧时间点或无 bid/ask 的 quote,证据等级最多 C,且必须输出`报价可能滞后`。若用户券商价与榜单/网页价偏离 `>=3%`,用户券商价优先;原计划状态按 §最新报价驱动决策状态机立即重算。若用户价已经低于旧硬止损/失效价,旧计划必须写`作废`,不得把旧止损改成新低吸入场。

若已发现候选但未能打开当前 quote 页面、未拿到券商价、也没有两个公开 quote 源互证,必须在报价基准写`当前 quote 未确认;仅有 movers 发现价`,并把该票降为`观察/待实时复核`;不得给`主攻/备攻`、建议本金、最大亏损或正常概率表。

对全景优先请求,上一条只限制`交易动作`,不允许限制`展示`: movers 中的 40%+ / Top 3 / IPO squeeze 票必须进入全景审计池,再因缺 quote 降级。

### 实时价存活闸门 (canonical)

每次输出`妖股 / 彩票仓 / 翻倍 / 极端 / 今日可交易`候选前,必须在最终候选表之前执行一次**实时价存活闸门**。该闸门用最终报价基准和候选发现价/推荐基准比较,决定候选是否仍可执行:

| 状态 | 判定 | 允许输出 |
|---|---|---|
| 可执行 | 最新价仍在触发区/承接区上方,且报价证据 A/B,未跌破计划止损或盘前承接区 | 可写主攻/备攻、入场、仓位、正常概率 |
| 等 reclaim | 最新价低于触发区但高于硬止损,或较候选发现价/推荐基准回撤 `>=8%` 但未结构破位 | 只给重新站回哪个价才触发;不得写当前可买 |
| 作废 | 最新价 `<=` 计划硬止损/结构失效价,或低于盘前承接区且 5-15 分钟收不回,或较候选发现价/推荐基准回撤 `>=15%` 且无 reclaim | 旧计划作废;只能观察或重新定义新触发价 |
| 报价冲突 | 用户券商价与公开源偏离 `>=3%` 且无法解释时间段,或公开源互相冲突 `>=3%` | 以用户券商价为最高优先级;无用户价则只观察 |

候选发现价包括 movers 榜单价、scanner 价、盘前旧价、上一轮推荐基准价或本地日线价。最终报价基准必须按 §Movers 快照不得当作最新成交价 获取。若没有最终报价基准,候选不能通过存活闸门,只能写`观察/报价不可复核`。

存活闸门优先级高于因果分、妖性分和榜单排名。任何票一旦被判定`等 reclaim`或`作废`,不得继续保留`主攻/备攻/当前可买/建议本金/最大亏损`措辞;只能输出条件触发后的粗略概率。

### 开盘后 30 分钟二次确认

对盘前暴拉、低价财报、榜首、极端彩票候选,若回答发生在 `09:45 ET` 之后,必须做开盘后二次确认。没有分钟级 VWAP 时,用 `open/high/low/current` 或用户券商分时替代:

| 状态 | 判定 | 动作 |
|---|---|---|
| 二次确认通过 | 开盘后站稳开盘价/VWAP/盘前承接区之一,且高点回撤 `<50%` 的 open-to-high 涨幅 | 可继续按原触发计划 |
| 快打已结束 | 从触发价或开盘价先冲 `+10%-15%` 后回撤,且回撤超过已涨幅一半 | 若未入场,不追;若已入场,默认第一目标兑现/抬保本 |
| 退潮观察 | open-to-high 后回撤超过一半涨幅,且收不回开盘价/VWAP | 降为`只做反抽观察`;不得给新彩票仓入场 |
| 结构作废 | 跌破开盘价/盘前承接区/硬止损后 5-15 分钟收不回 | 旧计划作废;只等全新 reclaim |

该确认解决 NXXT 型问题:开盘后确实曾给 `+10%` 快打窗口,但若随后跌破承接并收不回,不得继续等待 `+50%/+100%` 目标。

### 当前夜盘价优先规则

凡是用户问`今天 / 今日 / 盘前 / 夜盘 / 开盘 / 妖股 / 彩票仓 / 止盈概率 / 入场止损`这类需要当天执行的结论,必须先尝试获取**当前 overnight / premarket / after-hours 可见报价**。最新可交易报价是入场、止损、止盈、仓位和概率的基准;本地日线收盘价只能做背景。若只能拿到上一交易日 after-hours 或旧缓存,必须标注`不是当前夜盘/盘前`,并把结论降为`待盘前复核/观察`;不得用昨收直接输出`可买/主攻`。

### 最新报价驱动决策状态机

拿到 Web/用户券商最新价后,必须先用最新价重算交易状态,再给任何"入不入 / 加不加 / 追不追"结论。**获取报价不是完成任务;把报价转成新策略才是完成任务。**

对每只候选按以下顺序判定:

1. `最新价 <= 旧硬止损 / 结构失效价`:旧计划作废;未持仓写`不入,等 reclaim`;已持仓切换止损/被套处置。
2. `最新价在旧入场区内,且未跌破 VWAP/盘前承接区`:允许保留或微调计划,但必须用最新价重算止损、止盈、R/R 和建议本金。
3. `最新价低于旧入场区但高于旧止损`:不得机械说"更便宜";必须判断是健康回踩还是承接失败。只有守住 VWAP/盘前低点/关键平台并出现成交回流,才可改为`回踩做`;否则`观察`。
4. `最新价高于旧入场上限`:不得保留旧入场上限;改为`等待回踩`或`二次突破`,重新给触发价和失效价。
5. `最新价触及或越过旧第一止盈`:旧入场机会已错过;只能给回踩/二突二轮计划,并降低仓位。

输出必须显式写一行`最新价状态判定`:例如`ONDS 最新 $10.85 = 旧止损位 → 旧计划作废,不入;新计划只等 $11.25 reclaim 或 $11.75 二突`。

如果三方冲突:

- 用户券商盘口优先用于交易价格,但必须标注`用户券商信息,公开源不一致`
- Web/公开源优先用于新闻、SEC、融资/ATM/反拆红旗
- 本地日线只能做背景,不能替代盘中报价或新闻
- 无法解释价差或时间冲突时,只能输出`报价冲突/待复核`,不得给加仓或主攻
- 若实时/延长报价或用户券商报价与本地日线收盘价偏离 `>=5%`,旧的日线入场/止损/止盈/仓位/R:R 全部作废,必须基于最新报价重算;不能只在备注中说明偏离。
- 若公开源之间价差或涨跌幅冲突 `>=3%`,或用户报价与网页报价冲突但无法解释时间戳,该票只能标`报价冲突/只观察`,不得给入场、加仓或主攻。

## 用户报价纠错触发规则

用户说`我看到价格是 X`、`你价格不对`、`现在不是这个价`、`券商显示 X`、`已经跌到/涨到 X`时,必须立即停止沿用原推荐:

1. 承认原报价基准失效,把用户券商价作为最高优先级交易基准。
2. 用用户价重算入场、硬止损、第一/第二止盈、建议本金、最大亏损和 R/R。
3. 若缺少成交量、bid/ask、时间戳或 Web 复核,只能输出`修正版状态表`,不得继续保留原主攻/备攻结论。
4. 若用户价与 Web/本地偏离 `>=5%` 且无法解释为不同交易时段,原计划必须写`作废`,该票降为`报价冲突/只观察`。

### 当前价触发旧止损的硬门

若用户券商价或可复核实时/延长报价已经 `<=` 上一版计划的硬止损、降仓触发价或结构失效价,旧入场计划必须立即作废:

- 不得继续回答"入不入"为按旧计划入场;结论先写`不入/旧计划已失效`。
- 不得把旧止损价同时当作新入场价,除非重新完成 Web/券商报价复核,且价格重新站回 VWAP / 开盘价 / 盘前承接区 / 前高 reclaim 之一。
- 若要给新计划,必须以当前价重新定义`新触发价 / 新失效价 / 新止损 / 新止盈 / 新 R/R`;没有 reclaim 或二突结构时只能`观察`。
- 用户已经持仓时,切换到 [../workflow/exit-framework.md](../workflow/exit-framework.md) 的被套反抽处置;用户尚未持仓时,直接取消旧单,等待新结构。

## Web 验证最小清单

推荐、加仓、减仓、补仓、止损、复盘前至少尝试验证:

- 当前或最近延长交易价、时间戳、成交量/成交额
- 若处于官方盘前前或夜盘时段,优先查 overnight/Blue Ocean/券商夜盘可见价;无夜盘成交时再退回最近 after-hours,并写明时间段差异
- 当日/最近交易日开高低收、VWAP 或可替代的开盘价/日内均价
- 48 小时内新闻源:公司 IR/PR、SEC/8-K、Nasdaq/MarketWatch/StockAnalysis/主流财经媒体
- 过去 7 天红旗:`offering / ATM / dilution / reverse split / bankruptcy / SEC investigation / halt`
- 催化是否仍未证伪:财报、合同、MOU、控制权交易、FDA/临床、分析师、回购等

若不能拿到实时 VWAP,用以下替代顺序:

1. 用户券商 VWAP/均价/分时图
2. 开盘价 + 日内低点/高点 + 当前价
3. 前收 + 盘前/盘后承接区

缺少 VWAP 时必须降低置信度,但不能因此忽略明显的二次修复结构。

## 每步交易决策模板

所有持仓处置和补仓建议必须按这个顺序输出:

```md
用户输入:
- 成本 / 仓位 / 已卖出比例 / 用户盘口时间

Web 验证:
- 最新价 / 时间 / 成交量或成交额 / 新闻 / 红旗

本地数据:
- 前收 / ATR / 量比 / 20 日位置 / short float 或基本面重点

状态判断:
- 推荐 / 计划内持仓 / 深套 / 二次修复 / 失败退潮

操作:
- 建仓 / 补仓 / 持有 / 减仓 / 清仓

失效条件:
- 跌破哪个价位、多久收不回则取消原计划

账户影响:
- 最多还能亏多少;反抽到目标能修复多少
```

## 防用户信息误导

- 用户说"跌很多/涨很多/能到某价"时,不要直接采用结论;先核对价格、时间段和成交量
- 用户截图必须读出截图时间和交易阶段:盘中、盘后、夜盘、昨日收盘
- 用户成本、仓位、卖出价不完整时,先复述假设并给可替换公式
- 如果用户信息与公开源冲突,给两套情景,并明确哪套以用户券商盘口为准

## 补仓认证条件

亏损后`分批补仓`的具体认证条件、必须包含的输出字段(补仓触发价、金额上限、补仓后均价、失效价、最大账户亏损、反抽减仓阶梯)统一按 [../workflow/exit-framework.md §补仓修复输出](../workflow/exit-framework.md) 执行;不满足条件时只能减风险或保留观察仓,不得建议补仓。
