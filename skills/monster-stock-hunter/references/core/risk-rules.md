# 妖股仓位与纪律

本 skill 默认服务的是高赔率、低胜率、极高波动的日内交易。

## 三段加仓状态机引用

妖股做 T 的本金**不是一次性给满**;按 [../workflow/ignition-forecast.md §三段加仓状态机](../workflow/ignition-forecast.md) 的 `预判仓 1/3 → 确认仓 2/3 → 主攻仓满仓` 顺序渐进升级。本文件只锁定每段的账户级风险上限和反推公式;具体段位转换条件 (触发价 / 早信号阈值 / 降级条件) **不在本文件**,以 ignition-forecast 为 canonical。

每段对应的"建议本金"按下面 §仓位纪律 的反推公式计算,但**账户风险额输入要先除以段位系数**:

- 预判仓: 账户风险额 = `0.5%-2%` 的 `1/3`,即 `0.15%-0.7%`
- 确认仓: 账户风险额 = `0.5%-2%` 的 `2/3`,即 `0.35%-1.35%`
- 主攻仓: 账户风险额 = `0.5%-2%` 全额

三段累计本金不得突破"单票本金 `30%` 默认上限"。

## 仓位纪律

- 默认单笔**账户风险额**: 总资金的 `0.5% - 2%`
- 这里的百分比指"最多愿意亏多少钱",不是买入本金占比
- 只有最干净的主攻票才允许账户风险额到 `3%`
- 仓位本金用公式反推: `建议本金 = 账户风险额 / 止损距离`
- 例: `$5,000` 账户,本单愿亏 `$50`,默认止损距离 `5%`,建议本金约 `$1,000`
- 止损距离 `6%-8%` 仅适用于 §高波动例外;不是默认值,且使用时必须按例外条款同步降低账户风险额
- 同时最多持有 `2` 只妖股仓位
- 单只票如果需要解释超过一句"为什么还能抗",就说明仓位已经太大
- **禁止满仓**: 本 skill 不允许给出、暗示或认可满仓/重仓妖股操作。单票本金超过账户 `30%` 默认超限;超过 `50%` 必须先降仓,不得继续按正常交易计划分析。
- **主攻不等于重仓**: `主攻`只表示候选优先级,不是本金仓位。输出必须同时写`建议本金`和`最大亏损`,且建议本金必须由止损距离反推。
- **三段加仓不允许跳级**: 仅命中原型识别 + 早信号 ≥3 → 预判仓 1/3;须等 reclaim VWAP + 量能加速 + 早信号 ≥5 → 确认仓 2/3;须等进 Top 5 + 量能持续 + 早信号 ≥7 → 主攻仓全额。详见 [../workflow/ignition-forecast.md §三段加仓状态机](../workflow/ignition-forecast.md)。
- **亏损后降档**: 若用户当日已出现大额亏损、心态失衡、急于回本,单笔账户风险额上限降到 `0.25%-0.75%`,建议本金通常不超过账户 `10%-20%`;不得推荐纯 pump 或极端彩票作为回本主路径。**此时三段加仓状态机禁用**,只允许单笔小仓试错。

### 小账户做 T 常用区间

对 `$5,000` 附近账户,按三段加仓状态机分档:

- **预判仓试错**: 风险 `$10-$25`,本金通常 `$150-$350`(账户风险 0.15%-0.5%)
- **确认仓**: 风险 `$25-$45`,本金通常 `$300-$700`(账户风险 0.5%-0.9%)
- **主攻仓**: 风险 `$40-$75`,本金通常 `$700-$1,200`(账户风险 0.8%-1.5%)
- 极端停牌/反拆/无源 pump: 风险 `$12-$30`,本金通常 `$150-$500`,**只允许预判仓段**,不得升级
- 单票本金超过 `$1,500` 时,必须解释为什么这不是重仓赌方向
- 单票本金超过账户 `30%` 时,即使用户愿意冒险,也必须建议拆小或放弃;不得为了追求单次回本而放大本金。
- 若用户上一轮实际用了 `$2,000-$3,000+` 大本金且已盈利,后续同票默认不再允许同等本金;除非出现全新催化和全新结构,第二轮本金通常压回 `$500-$800` 以下

## 时间纪律

- 默认只做日内
- 默认不过夜
- `14:45 ET` 强制清仓
- 最佳建仓窗口: `09:45 ET - 11:00 ET`
- 尾盘点火默认不参与

## 止损纪律

- 默认硬止损: `-5%`
- 止损触发立即全出
- 不允许无条件摊平
- 亏损加仓只有在完成 Web 交叉认证、二次修复评分达标、并写清补仓后均价/失效价/最大账户亏损时才允许
- 同票止损后当日默认不再重买
- 若亏损已经超过原计划止损 `2x`,进入`风险处置`而不是继续交易分析: 先减仓/止损,再讨论是否保留极小观察仓。
- 若用户用满仓或 50%+ 本金持有亏损妖股,第一建议必须是降低本金暴露;不得把后续催化作为继续满仓的理由。

### 重仓亏损后的反抽权利

当用户已经在高波动票中重仓/满仓亏损时,风控目标不是机械清仓,而是把账户从"可能被单票毁掉"降到"还能等反抽"。完整的`账户风险表 / 反抽路径表 / 分层动作 / 失效条件 / 禁止项`按 [../workflow/exit-framework.md §被套反抽处置](../workflow/exit-framework.md) 执行;本文件只锁定账户级风险刹车上限:

- 本金仓位 `>=70%`: 第一动作是先降到 `40%-60%`,不得继续按正常交易计划分析
- 本金仓位 `40%-70%`: 先降到 `25%-40%`
- 不得把"满仓错误"和"标的已经失败"混为一谈;满仓错误必须先降风险再讨论标的

### 高波动例外

- 若标的是真正的 monster 龙头,且主导偏向非常强,允许把止损放宽到 `-6% ~ -8%`
- 但账户风险额必须同步降一档,通常从 `1.5%-2%` 降到 `0.5%-1%`
- 这类放宽只服务于**给高波动票留呼吸空间**,不是给错误交易找借口
- 如果你发现自己只是因为舍不得止损才用这个例外,说明不该用
- 高波动例外永远不能和重仓同时出现;`更宽止损`必须配`更小本金`。

## 止盈纪律

具体止盈百分比(第一止盈 / 第二止盈 / 延展目标 / 高波动例外)按 [../workflow/exit-framework.md §标准处理](../workflow/exit-framework.md) 执行,本节只锁定单轮利润保护与同票二轮的账户级纪律:

- 已经完成一轮 `8%-20%` 做 T 后,默认至少兑现一半以上
- 单轮盈利达到账户 `1.5%-2%` 后,后续同票只能用同等或更小风险继续,不能利润上头加码
- 单轮盈利达到账户 `5%+` 后,默认建议停止同票交易;若继续,只能把已实现利润的 `10%-20%` 作为第二轮最大亏损,不能把本金重新暴露给同一只票
- 若同票第二轮失败,当日停止继续做 T

## 适用心态

这类交易只适合接受以下事实的用户:

- 你可能连续错两三次
- 但一次抓对可能覆盖多次小亏
- 你的任务不是每次都赢,而是**只在对的时候拿到大肉**
- 对妖股来说,**波动越大,理论可操作空间往往越大**;关键不是怕波动,而是分清这波动还有没有第二段

## 情绪与恢复模式

### 设计原则 (canonical)

本节只服务**真实的情绪/财务风险处置**,不服务"用户想要的高收益叙事"。`想翻倍 / 想赚多 / 要极端 / 要最猛` 不是心态问题,而是合理的高赔率诉求 — 必须用 [../modes/high-elasticity-lottery.md](../modes/high-elasticity-lottery.md) 高弹性翻倍候选池能力满足,**不是**用本节降档处置回避。把高收益诉求贴上"心态有问题"标签自动降档,是错误的产品设计,不允许这样做。

### 触发条件 (canonical, 满足任一即触发)

**A. 亏损/被套已发生**:用户已实际亏损、被套、套牢、爆仓、要求"回本/解套/翻本"(账户已经在亏的事实);"翻本"特指账户已亏需要找回,而不是"想多赚"

**B. 心态预警(真情绪失控)**:心态崩、上头、不甘心、必须赢回来、不能再亏、亏麻了、停不下来、追单 — 这些词描述的是失控状态,不是高赔率诉求

**C. 财务风险**:用户表达正在用满仓/重仓/梭哈/借钱/挪用生活费/超出可承受亏损金额参与

**不触发本节的情况**(明确):

- "我想翻倍 / 想赚得多 / 要高收益 / 要极端 / 最猛 / 要妖股 / 抓到就大肉 / 高弹性"等高赔率诉求,**不触发**本节;应调用 [../modes/high-elasticity-lottery.md](../modes/high-elasticity-lottery.md) 给高弹性候选,而不是降档说教
- 用户给账户大小 + 想达到的目标(如 "$5,000 想做到 $10,000"),只是目标说明,**不触发**本节,除非同时出现 A/B/C 类信号

### 触发后的硬规则 (仅在 A/B/C 真触发时执行)

- 不再输出"今天必须翻本"式目标(注意:这条针对"已亏需要回本"的场景,不针对"想翻倍"的高赔率诉求)
- 先要求把单票仓位降到可承受水平
- **三段加仓状态机禁用**:只允许预判仓档(账户风险 0.15%-0.5%)
- 单笔账户风险额上限降到 `0.25%-0.75%`,建议本金通常不超过账户 `10%-20%`
- 恢复交易只允许 1 笔,最多 2 笔;第一笔再亏则停止
- 恢复单目标是重建执行纪律,不是一次赚回全部亏损
- 在回答顶部显式声明: "检测到 [亏损/被套/心态崩/财务风险] 信号,按 §情绪与恢复模式 处理:本笔只给小仓试错和纪律。" 必须基于真实信号,不允许把"想翻倍"曲解为心态问题
