# 因果优先扫描工作流

本文件是 `monster-stock-hunter` 默认妖股请求的工作流来源:**先以因断果,再用榜单验证**。涨幅榜、盘前榜、盘后榜是资金响应和防漏工具,不是第一因子。

妖股请求默认主线是 [ignition-forecast.md](ignition-forecast.md) 5 原型;本文件接收 ignition 输出的`原型编号 / 早信号命中数 / 当前仓位段`,在此基础上排序与降级。**不得绕过 ignition 直接从涨幅榜起手**。

本文件只负责**候选生成与排序**。其他职责由 sibling 默认必读文件承担,不得在本文件内重复或覆盖:

- 5 原型识别 / 早信号打分 / 三段加仓状态机 → [ignition-forecast.md](ignition-forecast.md)
- 输出格式与字段定义 → [../core/output-contract.md](../core/output-contract.md)
- 仓位/止损/账户风险刹车 → [../core/risk-rules.md](../core/risk-rules.md)
- 报价/新闻三方认证 → [../core/web-cross-check.md](../core/web-cross-check.md)
- 三连校验 / 微盘资本事件 → [../core/pre-recommend-triplet.md](../core/pre-recommend-triplet.md)
- 红旗清单与降级判定 → [red-flags.md](red-flags.md)
- 真催化 / squeeze / pump 三类分类 → [pump-vs-real.md](pump-vs-real.md)
- 做 T/被套反抽/补仓修复 → [exit-framework.md](exit-framework.md)
- 高位事件票/MOU/控制权交易判断 → [catalyst-stage.md](catalyst-stage.md) (按需)

## 核心原则

- 妖性先行: 本 skill 宁可空仓,也不把普通强势股包装成妖股。
- 先建立`因果假设池`,再看当前榜单。
- 开放式推荐必须先固定`稳定三篮`: `主升/点火候选`、`抄底修复候选`、`只审计不交易`。最终 Top 1-3 只能在三篮内排序,不得因为 Web movers 排名刷新或新媒体标题出现而整篮换票。
- 先解释为什么资金可能在今天/近期反应,再判断价格是否已经确认或透支。
- 明确催化先行: 没有 48 小时内可追溯硬催化的盘前/盘中异动,只能进入`无源异动审计/观察`,不得进入`确认池`或最终主攻/备攻。
- 当前涨幅只能证明"已经发生",不能单独证明"还会继续"。
- 没有确认信号的票可以进入`预判池`,但不能给正常仓位或"当前可买"结论。
- 榜首只作防漏与风险对照;除非同时通过因果链、未定价和结构确认,否则不得单独因为排名升为主攻。

## 数据源与 schema

默认顺序:

1. 当前报价源: 用户券商盘口、Nasdaq / MarketWatch / Yahoo / ADVFN / StockAnalysis / Stocknear 等当前或延长交易 quote/movers,用于决定是否可交易。
2. 因果源: 公司 PR/IR、SEC/8-K、财报、FDA/临床、合同、控制权交易、行业/同业传导、事件日历、低流通/高空头结构。
3. 红旗源: SEC、公司公告、主流财经媒体、融资/ATM/反拆/停牌/退市合规信息。
4. 本地结构源: TradingAgents MCP 的 `db_health`, `latest_prices`, `symbol_snapshot`, `query_market_data`, `company_fundamentals`, 只用于查候选、未定价、量能斜率、ATR 和红旗背景。
5. 榜单验证源: WebSearch / 公开源的 `after-hours gainers`, `overnight movers`, `premarket gainers`, `top gainers`, `intraday top gainers`, `top losers`, `premarket losers`, `after-hours losers`, `unusual volume movers`, `trading halt / LULD resume`, 只用于确认资金响应、防漏、过热降级和错杀修复候选发现。

禁止顺序倒置: 不得先用本地日线价格生成入场、止损、止盈、建议本金或概率,再补一句"需实时复核"。没有当前/延长交易报价时,本地结果只能进入`候选池/观察池`,不能进入最终`主攻/备攻`。

本地日线表固定为 `market_data.us_equity_daily_prices`;日期字段为 `price_date`;默认 `provider='yfinance'`。手写 SQL 或执行 scripts 时不得使用旧版日线表名或旧版日期字段名。

如果 MCP 不可用,必须说明本次走 WebSearch 回退路径,不得伪装成完整量化筛选。

## 妖性硬门

每只最终候选必须先过`妖性评分`,再进入因果评分。妖性分衡量的是"有没有暴拉空间",不是安全性。

| 因子 | 分数 | 口径 |
|---|---:|---|
| 弹性结构 | 0-2 | 流通 `<30M` 或市值 `<$300M` +1;short float `>=15%` 或 SPAC/壳股流通压缩 +1 |
| 波动空间 | 0-2 | ATR14 `>=8%` +1;过去 180 天有单日 `+50%` 或 20 日振幅 `>=40%` +1 |
| 点火强度 | 0-2 | 催化 `<48h` 且可追溯 +1;催化硬度 A/A-/B 或同业龙头刚爆 `+50%` +1 |
| 量能斜率 | 0-2 | 当前或最近完整日线相对量 `>=3x` +1;当前成交额/盘后成交额通过可交易硬门 +1 |
| 未定价空间 | 0-2 | 未进 Top 3 或刚从低基数启动 +1;距关键前高/52周高仍有 `15%+` 空间或低于前高 `5%-20%` 可二突 +1 |

结论:

- `>=7`: 妖性足,可进入主攻/备攻竞争,再看因果分、报价和红旗。
- `5-6`: 有妖性但不够纯,只能条件触发或高弹性备选。
- `<5`: 不够妖;即使走势稳、基本面好,也不得作为 monster 主攻/备攻,只能转普通日内观察或放弃。

例外: 已经进入极端彩票榜首池的票可展示,但若妖性来自纯 pump 而非催化/结构,必须按 [pump-vs-real.md](pump-vs-real.md) 降级为观察或极小仓触发。

## 默认三段决策流程

### 1. 因果假设池

先从事件和结构出发建立候选,每只必须写出:

- `因`: 财报、FDA/临床、硬合同、控制权交易、救命融资、行业传导、低流通挤压、高空头、政策/主题窗口。
- `传导`: 为什么这个因会吸引今天或近期资金;例如同业刚爆、事件临近、题材热、成交额开始转移、空头拥挤。
- `未定价证据`: 为什么不是已经涨完;例如未上主流榜、5 日涨幅未透支、仍低于前高、有明确触发价、成交额尚未极端拥挤。
- `确认信号`: 盘前/盘中量价如何证明假设开始兑现。
- `证伪信号`: 哪个价位、消息或红旗说明假设失败。

硬门: `因` 必须至少有一个 48 小时内可追溯硬催化来源。低流通挤压、高空头、盘前放量、历史翻倍基因只算结构,不算硬催化;若没有 PR/SEC/财报/FDA/合同/并购/监管等明确事件,该票只能放入`无源异动审计池`,不得进入确认池。

### 2. 榜单验证池

再查看涨幅榜、盘前榜、盘后榜和 unusual volume:

- 用于确认资金是否开始响应因果假设。
- 用于发现遗漏的极端票并放入`极端彩票榜首池`。
- 用于识别已过热、报价冲突、点差宽、停牌或红旗票。
- 不得把榜单名次当作主攻理由;若理由写不出"因 -> 传导 -> 未定价 -> 确认",只能观察或放弃。

### 3. 三类输出

- `预判池`: 因果链强、尚未完全定价,但量价未确认;只能给条件触发和小仓计划。
- `确认池`: 因果链强,且已经出现可复核量价确认;可给主攻/备攻。
- `已过热放弃池`: 因果存在但价格已经透支,或榜首远离 VWAP/回撤失败/红旗过重;只写放弃原因。

确认池额外硬门: `量价确认` 必须同时包含当前绝对成交量/成交额与成交连续性,口径见 [../core/web-cross-check.md §当前流动性硬门](../core/web-cross-check.md)。只有本地 `volume_ratio` 很高、昨日爆量、或榜单显示涨幅,但盘前/盘中当前成交量、成交额、bid-ask 不达标时,不得进入确认池;只能放入`预判池/观察`。

## 默认池子扫描顺序 (canonical)

默认妖股请求必须按以下顺序扫描并合并评分。**点火预判池排第一,现任榜首池后移到挑战者池之后**,纠正"已涨 = 可信"的锚定偏差:

```text
点火预判池 (5 原型, ignition-forecast)
  → 因果假设池
  → 盘后点火/次日延续池
  → 挑战者池
  → 跌幅错杀修复池 (Top Losers + reclaim-bounce)
  → 二次点火池
  → 现任榜首池 (只作防漏 / 已过热对照, 不作主攻入口)
  → 已过热放弃池
```

**全景优先例外**: 当用户问`+40% / 高涨幅 / 高回报 / 翻倍 / 最猛 / 彩票仓 / 极端`时,用户首先需要看到完整强势票全景。此时用户可见输出必须先展示`当前盘前/盘中 40%+ 全景审计池`(含现任榜首池、盘前/盘后 Top 10、IPO/new listing 异动票),然后再按下方默认顺序做因果评分和交易分层。展示前置不等于推荐前置;榜首仍需按报价、承接、红旗和 R/R 降级。

各池职责:

- `点火预判池 (5 原型)`: 由 [ignition-forecast.md](ignition-forecast.md) 输出 5 原型识别 + 早信号 + 仓位段。涨幅 `0%~+15%`、催化 `<4h`、未上 Top 3 的候选先汇入本池。本池是默认主线;无明确点名的妖股请求必须生成本池扫描结果,但用户可见位置统一服从 [../core/output-contract.md](../core/output-contract.md) 的输出顺序,普通推荐请求放入`场景审计附录`或压缩摘要。
- `因果假设池`: 先按催化硬度、事件临近度、未定价程度、主题传导、可执行触发器和红旗控制筛选。该池决定预判方向,与点火预判池可共享候选。
- `盘后点火/次日延续池`: 对"今日/明日/盘前/妖股/彩票仓"请求,必须在本地日线榜之前先抓最近 after-hours / overnight / premarket 异动。凡收盘后因财报、8-K、合同、FDA/临床、授权交易、指引等跳涨 `>=20%`,或盘后成交额 `>= $5M`,或进入公开 after-hours / premarket movers Top 3 的票,即使最近完整日线收跌、量比很低、收在低点或不在本地涨幅榜,也必须进入本池并按最新延长交易价复核。
- `HCWB / AMST 型盘后低基数点火`: 这是盘后点火池的强制防漏子类。若常规盘 `ret_1d <= 0`、`volume_ratio < 1` 或成交额很低,但 after-hours / overnight / premarket 相对常规盘收盘跳涨 `>=40%` 且 48 小时内有财报、8-K、PR、临床/监管、合同、企业客户、控制权/融资公告,必须入池展示。该类票常被本地日线涨幅榜漏掉;默认结论是`盘后点火审计/观察`,只有报价 A/B、连续成交、点差可控并出现 reclaim/二突后,才允许极小仓触发。HCWB 型若涨幅 `<+30%` 且未上 AH Top 3,可同时归入**点火预判池原型 1**。
- `本地低分不能覆盖 Web 新鲜催化`: 对盘后公告点火票,本地日线低量/低量比只表示"本地 SQL 会漏",不是放弃理由。排序时先给`Web升级观察`身份,再用当前 quote、流动性硬门、红旗和 R/R 决定是否进入最终候选;若最终未进 Top 3,必须在放弃备注里写明是因为`流动性/报价/红旗/已过热`,不得写成"未扫描到"。
- `挑战者池`: 当前涨幅约 `+8%~+45%`,48 小时内有新鲜催化,量能开始加速,仍有可执行入场结构。是点火预判池的下一阶段(涨幅边界 +15% 切换)。
- `跌幅错杀修复池`: 同步审计 Web `top losers / premarket losers / after-hours losers` 与本地最近完整日线跌幅榜。只接收`跌得狠但利好未证伪`的票: 48 小时内有正面临床/财报/合同/并购/战略评估/监管进展,但当前因安全疑虑、融资担忧、获利了结、读数细节争议或市场误读大跌；必须由 [../modes/reclaim-bounce-hunter.md](../modes/reclaim-bounce-hunter.md) 给出二次修复评分和 reclaim 触发位。若下跌原因是核心临床失败、主资产终止、财报/指引实质恶化、融资稀释、退市/反拆或无新利好,只能列`跌幅审计/放弃`,不得包装成抄底。
- `二次点火池`: 强票低开/急跌后重新站回 VWAP、开盘价、昨收或前一日承接区。可同时归入**点火预判池原型 5**。
- `现任榜首池`: Web 盘前/盘中/收盘涨幅榜 Top 1-3,**只作防漏和已过热对照,不作主攻入口**。榜首只在用户明确问"榜首能不能追"时才单独展开;用户问`+40%以上 / 高涨幅 / 高回报 / 翻倍 / 最猛 / 彩票仓`时,必须把榜首、movers Top 10 中所有 40%+ 票、以及 IPO/new listing squeeze 票前置到全景审计池展示,再标`可交易候选 / 已飞等回踩 / 只审计不追 / 报价冲突 / 退潮减仓`。
- `无源异动审计池`: Web movers/盘前榜显示大涨,但 48 小时内未找到明确硬催化。必须展示涨幅、成交、结构和红旗,但结论只能是`只审计不追/观察/不推荐`;不得给主攻/备攻、建议本金或正常概率。
- `AI 微盘点火池`: 当日/48h 内出现 `AI agent / AI data center / AI infrastructure / AI sales platform / AI development agreement / AI pivot` 新闻、PR 或 SEC 披露,且候选是市值 `<$300M` 或流通 `<30M` 的低基数票,必须单独汇入本池。该池先判断 AI 催化质量: `有金额/客户/合同/部署节点/收入路径` 才能进预判或确认;只有 AI 关键词、旧闻重炒、顾问任命、MOU/LOI 或反拆后低流通,只能转`AI 主题审计/观察`。INHD 型可解释为`AI软催化 + 低流通 + 成交额跃迁`,CHAI 型若无 48h 官方硬催化则归`AI贴标签/低流通盘后审计`。
- `已过热放弃池`: 盘后/盘前已大涨、1 日/5 日透支、榜首远离 VWAP、从高点回撤失败、报价冲突或红旗过重的票。

合并评分因子固定为: `妖性评分 / 催化硬度 / 事件临近度 / 未定价程度 / 主题传导 / 可执行触发器 / 红旗控制 / 报价证据等级 / 原型加分`。`当前涨幅榜排名`只能作为资金响应验证或过热降级因子。AI 题材只增强`主题传导`和`催化硬度`的解释,不得单独覆盖红旗、报价或位置折扣。最终 Top 1-3 必须写清来源池: `点火预判池(原型 X) / 因果预判池 / AI微盘点火池 / 现任榜首延续 / 挑战者接棒预判 / 二次点火接棒 / 极端彩票审计 / 盘后点火延续`。

## 稳定三篮与换票硬门

开放式`今天 / 妖股 / 翻倍 / 高弹性 / 有没有机会`请求,在第一次完整扫描后必须冻结一个`稳定三篮`,并在同一轮回答或连续追问中沿用,除非触发下方换票硬门:

| 篮子 | 角色 | 入篮条件 | 出篮/降级条件 |
|---|---|---|---|
| 主升/点火候选 | 从涨幅 Top、点火预判、挑战者中选 1-2 只 | 48 小时内硬催化 + 妖性分达标 + 未过热或有二突/reclaim | 跌破承接/硬止损、催化证伪、新融资/停牌/报价冲突、R/R 失效 |
| 抄底修复候选 | 从 Top Losers / 深跌强催化票中选 0-1 只 | 跌幅大但利好未证伪 + 二次修复评分 `>=3` + 有明确 reclaim 价 | 下跌原因确认为核心利空、reclaim 失败、成交萎缩、红旗新增 |
| 只审计不交易 | 极端涨幅、无源异动、纯利空暴跌、报价不足票 | 需要防漏展示但不满足推荐硬门 | 出现 A/B 报价 + 硬催化 + reclaim/二突后,可重新进入前两篮 |

换票硬门:

- 新票必须比当前同篮候选同时多满足至少 `2` 个关键维度: 催化更硬、报价证据更高、流动性更好、R/R 更好、红旗更少、闭环周期更匹配。
- 单纯`涨幅榜排名上升 / 媒体标题更新 / 盘前价格刷新`不得触发换票。
- 当前候选若只是 C 级报价不足,先写`待 A/B 报价升级`,不得直接用另一个同样 C 级票替换。
- 若推荐发生变化,必须输出`换票原因`: 旧票为何降级、新票在哪些维度更强、是否只是从`主攻`变为`观察`。
- 若用户质疑推荐来回变,下一次输出必须先展示`稳定三篮复核表`,再给新的候选计划。

## 已涨太高过滤

任一命中时,默认从主攻降级为`已过热放弃池`或`只观察/极小仓触发`,除非有全新硬催化和重新 reclaim:

- 盘后/盘前已涨 `>=30%` 且没有回踩承接或二突结构。
- 过去 1 日已涨 `>20%` 且 5 日涨幅 `>60%`。
- 前一日或盘前已涨 `>=120%`,开盘后 `30-60` 分钟无法守住 VWAP、开盘价、盘前承接区或二突位。
- 榜首离 VWAP 太远,或从高点回撤 `>=12%` 仍不能收回 VWAP。
- 价格已经按 MOU/LOI/框架合作当成确定订单重估,但没有合同金额、交割条件或收入路径。

这类票仍可展示在极端彩票池或放弃备注中,但不得包装成因果预判主攻。

### CODX 型盘前破位接棒降级

昨日强势、成交额大、量比高的接棒票,若次日盘前/盘中最新价已经跌破昨收、开盘价、盘前承接区或原计划硬止损,必须从`挑战者接棒预判/最终候选`移入`持仓/破位处置表`:

- 未持仓: 旧计划作废,只写`等 reclaim`;不得继续按旧触发价推荐。
- 已持仓: 标为`退潮减仓/反抽减仓`,先降风险;不得建议补仓。
- 重新评估条件: 重新站回 VWAP/开盘价/昨收之一,成交额回流,且 5-15 分钟不跌回承接区。
- 若最新价较候选发现价或旧推荐基准回撤 `>=15%`,且无 reclaim,默认`作废`;只能定义全新 reclaim 计划。

优先级声明: 若该票同时命中 [../modes/extreme-lottery-leaderboard.md §入池条件](../modes/extreme-lottery-leaderboard.md) 的极端彩票阈值,**仍按本节降级处理**。彩票池只用作防漏展示,不得给主攻/备攻或正常仓位。仅当点火距今 ≤1 个完整交易日、且当前价已重新站回 reclaim 位(VWAP / 开盘价 / 盘前承接区之一),才允许标 `极小仓触发`,否则只能 `观察`。

例外: 极端超卖 AH 反抽。若标的当日跌幅 `>=80%` + 收近全天最低点 + AH 出现强反抽(`+30%` 以上且伴随成交),允许进入 [../modes/extended-hours-scalp.md §极端超卖 AH 反抽 子模式](../modes/extended-hours-scalp.md) 执行 AH 限价埋伏 + 不过夜的小仓做 T;此时本节"已涨太高"指的是从 AH 反抽起点继续追高,而不是反抽本身。该例外不适用于资本结构红旗(反拆 30 天内 + F-1/S-1 EFFECT、ELOC、ATM)成立的票,该类票仍只能观察。

## 因果评分

默认排序先看妖性分,再看因果分和榜单验证。妖性分 `<5` 的票不进入 monster 主攻/备攻竞争。

| 因子 | 分数 | 说明 |
|---|---:|---|
| 催化硬度 | 0-2 | 硬合同/FDA/财报/控制权优先;MOU/传闻降级 |
| 事件临近度 | 0-2 | 0-10 日内明确事件或刚公告优先 |
| 未定价程度 | 0-2 | 未上榜、未透支、仍有触发空间 |
| 主题传导 | 0-2 | 同业刚爆、板块资金迁移、热门主题直接相关 |
| 可执行触发器 | 0-2 | 回踩/二突/reclaim/量能阈值清楚,且当前绝对成交量/成交额通过流动性硬门 |
| 红旗控制 | 0-2 | 无近期融资、反拆、停牌、报价冲突 |
| 原型加分 | 0-1 | 命中 [ignition-forecast.md](ignition-forecast.md) 5 原型其一 +1;0 表示不在点火预判窗口 |

满分 `13`。

`>=10`: 可进确认池或强预判池,但确认池仍必须通过当前流动性硬门。
`8-9`: 条件触发/观察。
`<=7`: 只观察或放弃。

榜单验证只能作为加注或降级理由,不能替代因果评分。报价证据和当前流动性共同决定能否从预判池升级为确认池,不得替代因果分。原型加分不替代催化硬度,只是确认该候选处于点火预判窗口。

最终排序优先级: `妖性分` → `因果分` → 当前流动性/报价证据 → R/R → 红旗控制。不要因为某票"更稳"而把妖性更强且风控可控的票排到后面;也不要因为妖性强就跳过报价、红旗和止损。

## 本地防漏 scripts

当 MCP 可用时,在 Web 榜单之外执行只读 SQL 防漏:

- [../../scripts/leaderboard_audit.sql](../../scripts/leaderboard_audit.sql): 最近完整日线的涨幅榜、低价高成交妖股、高成交额强动量。
- [../../scripts/reclaim_bounce_audit.sql](../../scripts/reclaim_bounce_audit.sql): 强票急跌后修复/反包初筛。

本地 SQL 只产生`候选池`,不能产生`最终推荐`。SQL 结果里的 close/high/low/volume 只代表最近完整日线,不得直接作为今日入场、止损、止盈、建议本金或概率基准。

极端彩票审计不再使用独立 SQL;用 [../../scripts/leaderboard_audit.sql](../../scripts/leaderboard_audit.sql) 的涨幅榜/低价高成交池叠加 Web 当前 movers、停牌恢复、盘后公告和 [../modes/extreme-lottery-leaderboard.md](../modes/extreme-lottery-leaderboard.md) 的入池规则防漏。

因果假设池、三张验证池与 WebSearch 候选合并去重后,必须先做当前/延长交易报价复核,再做新闻、SEC、红旗和结构复核。任一 script Top 10 候选未进入最终推荐时,在放弃/降级备注中说明原因。

## Web 防漏硬门

用户问`今天 / 今日 / 盘前 / 夜盘 / 开盘 / 妖股 / 彩票仓 / 高收益当天机会`时,必须主动抓当前盘前/盘中或最近 after-hours / overnight 强势票 Top 1-3,并额外用 `after-hours gainers + earnings/8-K/PR/customer win/contract` 关键词扫一轮低价盘后点火票。若 Web/券商榜显示某票盘后涨幅 Top 3,即使本地日线收跌,也必须进入审计池并按最新延长交易价复核。但榜单进入只代表防漏,不代表推荐;必须回到因果链和已涨太高过滤。

**AI 微盘 Web 防漏 (canonical, 2026-06-09)**: 开放式妖股/高弹性请求必须额外扫一轮 AI 微盘主题:

```text
"AI agent" OR "AI sales platform" OR "AI data center" OR "AI infrastructure" OR "AI development agreement" OR "AI pivot"
+ microcap / low float / Nasdaq / press release / 8-K / 6-K
+ after-hours / premarket / top gainers / unusual volume
```

入池条件: 48h 内可追溯官方/主流来源 + 当前/延长交易涨幅 `>=10%` 或当前成交额跃迁;或同主题已有榜首 `>=100%` 且候选涨幅仍 `<40%`。输出必须给`AI催化质量`: `硬催化 / 软催化 / 贴标签`。`硬催化`可参与主攻竞争;`软催化`只能条件触发;`贴标签`只能审计不追。

**盘中/AH 突发 Top10 防漏 (canonical, 2026-06-09)**: 用户问`翻倍 / 最猛 / 今日 / 高回报 / 彩票仓 / 极端`时,Web 防漏从 Top 1-3 扩展到 `当前 movers Top10 + premarket Top10 + intraday Top10 + after-hours/overnight Top10`。凡 Web 显示 `>=40%`、当前成交额 `>= $10M`、或进入任一交易段 Top 3 的票,都必须进入`40%+ 全景审计池`;`symbol_snapshot: no data`、本地日线低量/收跌、fundamentals 缺失或 SQL 未入榜只能影响证据等级,不能删除。缺本地结构时标`Web-only C/D 审计`;若无 A/B quote 或硬催化,只能`只审计不追 / 已飞等回踩 / 报价冲突`。`most traded/most active` 只能补充流动性,不能替代 `gainers` 榜。

低基数盘后点火硬门: 若出现类似 HCWB/RGNT 的形态(常规盘收跌/低量/低成交额,盘后因财报、6-K/8-K、PR、临床/监管、商业化进展或合同公告跳涨 `>=40%`,但伴随报价冲突、成交不足、going concern、Nasdaq 合规、权证/融资稀释等红旗),不得因为本地日线没入榜而静默删除。它至少要出现在`盘后点火/次日延续池`或`放弃/降级备注`中,并写明"未推荐是因为报价/流动性/红旗/结构",不是因为没有扫描到。

硬门: 在输出最终候选前,必须逐项确认`after-hours/overnight gainers`与`盘后点火/次日延续池`已被扫描。若只能完成本地日线扫描、盘前榜或 most-traded/most-active 页面,而没有 AH/夜盘涨幅榜 Web 异动扫描,必须在回答顶部标注`AH/盘后涨幅池未完成,可能漏掉 RGNT/HCWB 型票`,且不得把结果称为完整今日妖股推荐。

处于官方盘前前或 overnight 时,优先抓当前 overnight 可见价;没有当前成交时,再退回最近 after-hours,并标注不是当前盘前。若 Web 市场时钟与本地环境时钟冲突,以交易所/报价源交易状态为准。

## 用户点名防锚定

用户点名 ticker 只能进入待验证池,不能自动排在推荐前面。必须与全市场候选使用同一把尺子比较:

- 若点名票在涨幅、成交额、相对量、催化新鲜度、红旗控制和报价证据上领先,才可升为主攻或备攻。
- 若只是用户关注,但量能不足、新催化不清或已进入尾声,必须降级或排除。
- 输出时说明这是独立筛选结果,不是因为用户点名而优先。

每次用户新增 ticker 后,至少重新扫描一轮 Web/本地候选池,避免锚定偏差。

## 主导偏向与阶段

每只候选先定义一个主导偏向,再看辅助因子。可用主导偏向:

`低流通挤压 / 真催化点火 / 控制权重估 / 救命融资重估 / 榜首惯性 / 准榜首挑战 / 低位催化点火 / 量能斜率启动 / 次日延续 / 二次点火接棒 / SPAC 流通压缩 / 财报彩票 / 高 beta 预期差 / squeeze 财报窗口 / 主题财报共振`

若理由需要写成 4-5 条平均发力,通常说明它不是妖股龙头。遇到 MOU/LOI、控制权交易、救命融资或高位事件票,必须读取 [catalyst-stage.md](catalyst-stage.md)。

## 参与结构

默认只允许三种参与方式:

- 回踩做
- 二次突破做
- 放弃或只观察

对真正的高波动龙头,只要结构仍在,不必因为离均线远就自动拒绝;重点看是否还有二段空间、是否可快速退出。禁止盘前榜首市价硬追、开盘 5 分钟内情绪追单、止损后同票立刻重买、夜盘/盘前用市价单追垂直拉升。

## 时间上下文

- 盘前 `< 09:30 ET`: 只写条件单和开盘确认计划。
- 夜盘/盘前极早段: 只允许限价单和小仓做 T,必须说明点差/流动性不足风险。
- `09:30-09:45 ET`: 默认观察,不催追。
- `09:45-11:00 ET`: 妖股主要决策窗口。
- `11:00-13:30 ET`: 若未形成第二波,优先减仓。
- `13:30-14:45 ET`: 只管理仓位,少开新仓。
- `14:45 ET` 后: 不再给新开仓建议。

## 回答硬要求

- 最终候选必须先写因果链,再写价格计划。
- 放弃热门票时优先说明是`已过热 / 结果追认 / 结构失败 / 红旗过重 / 报价不可复核`哪一种。
- 若用户要求"未来股价预测",必须明确这是条件路径预测:先满足确认信号,才有目标价概率;不得用已发生涨幅外推目标。

## 复盘闭环

用户要求复盘昨日/上次推荐时,用 `latest_prices` 或 `symbol_snapshot` 拉推荐后最新完整交易日的 `open/high/low/close/volume/volume_ratio/dollar_volume`,并与原入场、止损、第一止盈、第二止盈对照。

复盘输出必须先使用 [../core/output-contract.md §推荐质量复盘表](../core/output-contract.md),不得只用散文总结。每只推荐至少计算:

- `最大顺向收益`:实际最高价相对推荐触发价的涨幅。
- `最大逆向回撤`:实际最低价相对推荐触发价的跌幅。
- `目标命中`:是否触达第一/第二目标。
- `止损命中`:是否触发硬止损或结构失效。
- `复盘分类`: `命中 / 快打命中,持有失败 / 未触发 / 完全失败 / 方向对但仓位应降级 / 遗漏更强票`。

复盘必须记录:

- `命中`: 触发入场后是否达到第一/第二止盈,是否符合原主导偏向。
- `失败`: 是否触发止损、跳空低开、跌破 VWAP/开盘价/承接区,以及原推荐哪里过度乐观。
- `遗漏`: 当日实际涨幅榜/成交额榜、盘后公告榜、盘前/夜盘榜是否有更强票未进入审计池。
- `当日 Top5 命中`: 复盘昨日/上次推荐时,必须列出推荐后完整交易日真实涨幅 Top5,并标注每只是否进入过预测评分总表、40%+ 审计池或最终候选;未进入的写`漏网原因`。
- `skill 修正项`: 对每个完全失败或漏网样本,必须给出一条可执行规则修正,例如`实时价存活闸门应降级`、`Web-only C/D 审计应展示`、`路径型催化应降级`。
- `量价解读`: 爆量必须结合收盘位置;`>=20x` 量比但收近全天低点是次日延续负面信号(换手兑现/出货),不能解读为趋势确认。
- `复用`: 成功或失败是否值得写入 [../case-studies.md](../case-studies.md),并沉淀成可执行规则。

判断规则:

- 触发后先到 `+10%~+15%` 再回落到触发价/止损附近,写`快打命中,持有失败`;修正点是退出纪律,不是选股完全失败。
- 触发后直接跌破硬止损且未到第一目标,写`完全失败`;修正点优先查报价基准、存活闸门和红旗。
- 推荐条件未触发但股票后来上涨,写`未触发/遗漏`;不得算作命中。
- 若推荐票有效但当天另有 `+80%` 以上更强票未进入审计池,写`遗漏更强票`,并检查盘前/盘中 movers 扫描是否失效。
- 全景优先请求额外自检: 如果最终推荐没有包含 Web movers Top 3 或任何 `+100%` 票,必须在`40%+ 全景审计池`逐票说明为什么没进主推;若未展示,直接判定为`遗漏更强票`。
- 复盘表最少包含四块: `推荐票结果表`、`当日 Top5 命中表`、`漏网原因表`、`skill 修正项表`;不得只用散文解释。

复盘只能强化规则和案例库,不能声称未来收益有保证。
