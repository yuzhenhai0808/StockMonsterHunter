本文件只负责**入场后的参与方式、止盈/止损阶梯、做 T、被套反抽、补仓修复**。其他职责不在本文件:

## 允许的参与方式

### 1. 回踩做

适用于:

- 盘前高开
- 开盘后第一次回踩
- 不破关键区间或 VWAP
- 高波动龙头首波冲太快,但第一次真正像样的回踩出现

要求:

- 回踩后要有承接
- 不能一路掉回涨停前起点
- 对 monster 票,回踩幅度可以更深,关键是**回踩后能不能迅速被接住**

### 2. 二次突破做

适用于:

- 开盘第一波冲高后整理
- 再次放量突破小平台
- 高波动票首波过猛,无法舒服低吸时

要求:

- 整理时间不能太久
- 量能要跟得上
- 若波动极大,允许平台更宽,不要用过窄的稳健票标准去要求它

### 3. 放弃

出现以下任一情况直接放弃:

- 开盘直接一字冲太高
- 当前价贴着日内高点,没有结构
- 只剩追高这一个动作
- 波动很大但完全没有承接,只有上蹿下跳

### 4. 做 T

适用于:

- 反拆首日 / 低流通 squeeze / 榜首惯性票
- 当日振幅极大,但价格能反复回到 VWAP 或关键平台上方
- 量能爆炸,买卖盘足够承接小仓快速进出
- 用户目标是吃 `8%-20%` 波段,不是持有到收盘赌第二天
- 用户已经用同类票做出过盈利,需要把利润保护放在继续扩张之前

要求:

- 只在确认承接后做低吸或二突,不在垂直拉升中追
- 每轮先设硬止损,第一目标及时减仓
- 一旦跌破 VWAP 且反抽不过,停止同票继续做 T
- 做 T 成功后不要把利润重新暴露给尾盘回撤,尾盘前降低仓位
- 每次输出必须写清这是第几轮 T: `首轮试错 / 盈利后第二轮 / 只允许利润仓`
- 若上一轮已盈利到账户 `1.5%-2%`,下一轮默认只允许用原风险的 `50%-70%`
- 若上一轮已盈利到账户 `5%+`,下一轮默认应建议停止同票继续 T;若用户坚持继续,只能用已实现利润的 `10%-20%` 做最大亏损预算
- 若已经完成一轮 `10%+` 收益,第二轮只在重新站回 VWAP / 关键平台 / 前高 reclaim 后做,不能因为"刚赚过"而追
- 若首轮使用了超过账户 `30%-40%` 的本金,即使盈利,第二轮也必须降本金;不要用"已经证明能涨"作为继续重仓的理由

### 5. 被套反抽处置

适用于:

- 用户已经在高波动票中持有 `30%+` 本金仓位
- 当前浮亏明显,但标的仍有真实催化、爆量、short squeeze 或二次修复结构
- 价格没有持续跌破催化前平台,并且出现 VWAP / 开盘价 / 昨收 / 用户成本附近的 reclaim
- 已按 [../core/web-cross-check.md](../core/web-cross-check.md) 复核当前价、成交额、新闻和红旗

目标:

- 先解除账户级灾难风险
- 再保留足够仓位参与二次修复
- 不把风险控制做成低位清仓、反抽卖飞

前置表(必出):

- `账户风险表`: 成本、当前价、仓位本金占比、浮亏金额、若再跌 5%/10% 的账户损失
- `反抽路径表`: 最近开盘价、VWAP/盘中均价、昨收、日内高低点、用户成本价、可减仓目标位

分层动作:

- 第一层: 若仍是满仓或 70%+ 本金,先降到 `40%-60%`;这是风险刹车,不是看空标的
- 第二层: 若本金 `40%-70%`,先降到 `25%-40%`,不要默认直接降到 `10%-20%`
- 第三层: 若价格站回 VWAP/开盘价/昨收/用户主要减仓位,停止继续砍仓,把剩余仓位改成`反抽仓`
- 第四层: 若重新站回用户成本下方 `5%-10%` 区间,优先给`分批逃生计划`,不要只给"继续减仓"
- 第五层: 若二次修复评分达标且 Web 认证无红旗,可在关键支撑或 reclaim 后小幅补`修复仓`(按 §补仓修复输出)
- 第六层: 只有跌破 reclaim 位后 5-15 分钟不能收回,或跌破催化前关键平台,才继续降仓到 `10%-20%` 或清仓
- 第七层: 若反抽越过用户成本或前高,剩余仓位按止盈处理,不要再用亏损心态急卖

剩余仓位卖出阶梯必须写清: `先降多少 / 什么价停止降 / 什么价继续降 / 什么价保留到反抽目标`。

失效:

- reclaim 后再次跌破 VWAP 且 5-15 分钟收不回
- 跌破当日低点或催化前关键平台
- 反抽到逃生区但量能明显衰减、连续冲不过
- 出现 offering / ATM / 反拆 / 交易所风险等新红旗

禁止:

- 不得把"满仓错误"和"标的已经失败"混为一谈。满仓错误需要降风险;标的是否失败必须看催化、VWAP/reclaim、量能和盘口路径
- 不得在股价已经重新站回 VWAP/开盘价后继续要求用户一路减到极小仓,除非有新的红旗或量价再次转弱
- 不得只给一个卖出比例
- 不得根据用户一句"应该补仓/会涨回去"直接补仓;必须先用 Web/公开源复核价位、新闻、成交和红旗

## 失效信号

- 跌破 VWAP 且不回
- 跌破爆量起涨区
- 二次突破失败并快速回落
- 量能明显衰减
- 高波动票若连续两次尝试上攻都被砸回,说明空间可能已经用完

## 标准处理

止盈百分比 (canonical, 任何止盈数字以本节为准):

- 第一止盈: `+10%`
- 第二止盈: `+20%`
- 高波动例外: 极端强票可上调到第一止盈 `+12%~+15%`、第二止盈 `+25%~+30%`;前提是第一止盈后把剩余仓位止损抬到保本
- 二突目标距离过近(第一目标距入场不到 `10%-12%`)时,触达必须立刻减半,不再期待自然延展
- 夜盘/盘前第一目标可降到 `+6%-12%`(详见 §夜盘/盘前做 T 补充)

### R/R 最低硬门 (canonical)

每只候选必须用**计划入场价 → 第一止盈 → 硬止损**计算 R/R = `(第一止盈 - 入场价) / (入场价 - 硬止损)`。任何输出主攻/备攻前都必须先过这道门:

- **默认主攻**: R/R `>= 2:1`(允许低胜率换高赔率)
- **极端彩票/妖股例外**: R/R 可放宽到 `>= 1.5:1`,但同时必须满足 (a) 单票本金降到预判仓档 (b) 在结论或全局`单票详情表`中显式标注"放宽到 1.5 因为高波动彩票仓"
- **R/R `< 1.5`**: 一律不得给主攻/备攻/正常仓位,只能写`观察/不追`,即使催化再硬、用户再点名
- **止损距离 `< 0.4 ATR14`**: 即使账面 R/R 达标,实际"先到止损"概率会被日内噪音压成多数,必须把止损放到至少 `0.5 ATR14` 后重算 R/R;放宽止损同时按 §高波动例外 降一档账户风险

R/R 计算必须用计划入场价,不是"入场上限"。"入场上限"是不允许超过的边界,不是真实进场基准。

入场后流程:

- 入场后立刻带止损
- 第一止盈后减仓 50%
- 剩余仓位止损抬到保本
- 14:45 ET 前清仓
- 做 T 票如果已经完成 `10%+` 单轮收益,优先兑现一半以上,不要等"完美顶部"
- 做 T 票如果已经完成账户 `$100` 左右利润,对 `$5,000` 账户约等于 `2%`,应优先保护成果: 可以停止交易,或只用利润仓做极小第二轮
- 做 T 票如果已经完成账户 `$250-$300` 利润,对 `$5,000` 账户约等于 `5%-6%`,默认当天目标已经达成;只有在 VWAP/关键平台/前高 reclaim 三者至少满足其一时,才允许极小第二轮

## 重仓亏损标准处理

- 先确认用户真实成本、股数、当前仓位占账户比例,不得用猜测成本做处置
- 如果用户成本价与系统假设相差 `>=5%`,必须重算全部减仓/反抽价位
- 用户截图或口述价格必须标注时间段,并用 Web/公开源或本地最近完整日线解释差异;无法解释时只给情景,不给确定操作
- 若标的已经从低点重新站回 VWAP/开盘价,不得继续使用"跌破止损立即全出"的普通入场规则;应切换为`被套反抽处置`
- 输出必须包含: `立即风险刹车仓位 / 停止继续减仓价 / 反抽减仓价 / 失效价 / 最多还能亏多少`

## 补仓修复输出

亏损后允许`分批补仓`,但它不是无条件摊平。必须同时满足以下硬门:

### 跑路 / 减仓 / 补仓 决策表硬门

用户给出持仓截图、成本、股数、浮亏,或直接问`是不是跑路 / 能不能补仓 / 要不要减仓`时,任何补仓讨论前必须先输出一张决策表:

```md
| 票 | 成本/股数 | 最新价/时间 | 当前状态 | 跑路条件 | 减仓条件 | 补仓条件 | 当前动作 |
|---|---|---|---|---|---|---|---|
| TICKER | $X / N股 | $Y / HH:MM | 破位/等reclaim/反抽中 | 跌破 $A | 反抽 $B-$C | 站回 $D 且成交回流 | 不补仓/先减风险 |
```

硬规则:
- 最新价低于原硬止损、盘前承接区,或较候选发现价/旧推荐基准回撤 `>=15%` 且没有 reclaim → 当前动作只能是`先减风险 / 反抽减仓 / 作废`,不得补仓。
- 最新价低于用户成本 `>=8%`,且未重新站回 VWAP、开盘价、昨收之一 → 默认禁止补仓;若用户已经重仓,先降本金暴露。
- 盘前破位票的固定动作是: `先减风险 → 等反抽减仓 → reclaim 后重新评估`;不得用"昨日量大/催化还在"倒推补仓。
- 补仓只能是`修复仓`,不是摊低成本;补仓后总本金占账户比例仍受 [../core/risk-rules.md](../core/risk-rules.md) 限制。

- 真实催化未证伪,且 48 小时内仍有可验证新闻/公告支撑
- 无新增 offering / ATM / reverse split / halt 等红旗
- 价格守住关键低点或催化前平台
- 重新站回 VWAP、开盘价、昨收之一;若无 VWAP,至少站回开盘价或用户截图中的分时均线
- 成交量/成交额回流,不是无量反抽
- 二次修复评分按 [../modes/reclaim-bounce-hunter.md §二次修复评分](../modes/reclaim-bounce-hunter.md) 达标

满足后输出必须包含:

- `补仓触发`: 价格站回哪个关键位才补
- `补仓金额`: 补仓后总本金占账户比例
- `补仓后均价`: 用用户成本和补仓价重算
- `失效价`: 跌破后取消修复计划
- `最大账户亏损`: 失效价触发时账户还会亏多少
- `反抽阶梯`: 成本下方、成本附近、前高/目标价分别卖多少

不满足任一项时,只能减风险或保留观察仓,不得建议补仓。

## 夜盘 / 盘前做 T 补充

- 只用限价单,不使用市价单
- 必须检查点差是否过宽;若买卖价差超过 `2%-3%`,默认降低仓位或放弃
- 必须检查盘口成交是否连续;若几分钟无成交,不要用日线成交额假装流动性充足
- 夜盘/盘前第一目标可以低于常规盘,通常 `+6%-12%` 就应兑现一半以上
- 若价格跌破盘前 VWAP 或关键整数位后反抽不过,停止同票继续做 T
- 夜盘盈利不能自动推导常规盘还能继续做;开盘后必须重新判断量能、VWAP 和前高 reclaim

## 波动使用原则

- 在本 skill 里,大波动不是扣分项,而是潜在加分项
- 真正的过滤条件不是"波动太大",而是"大波动里有没有结构和承接"
- 如果一只票波动巨大、主导偏向集中、且还有二段空间,应优先于低波动票研究
