# 点火预判主线 (Ignition Forecast)

本文件是 `monster-stock-hunter` 的**默认主线**。任何"今天有啥妖股 / 今天最猛 / 抓翻倍 / 彩票仓"请求,必须先按本文件 5 原型扫描,**再**进入 [causal-first.md](causal-first.md) 排序。不允许绕过本文件直接从涨幅榜起手。

本文件只负责**点火前夜识别 + 早信号打分 + 三段加仓状态机**。其他职责不在本文件:

- 候选池排序 / 已涨太高过滤 → [causal-first.md](causal-first.md)
- 仓位/账户风险刹车硬上限 → [../core/risk-rules.md](../core/risk-rules.md)
- 红旗清单与降级 → [red-flags.md](red-flags.md)
- 真催化 / squeeze / pump 分类 → [pump-vs-real.md](pump-vs-real.md)
- 催化硬度分层 A/A-/B/C/D/E → [catalyst-stage.md](catalyst-stage.md)
- 入场后做 T / 止盈止损 / 被套反抽 → [exit-framework.md](exit-framework.md)
- 输出格式 / 表格字段 / 自检 → [../core/output-contract.md](../core/output-contract.md)
- 三连校验 / 微盘资本事件 → [../core/pre-recommend-triplet.md](../core/pre-recommend-triplet.md)

## 目的与适用窗口

用户要的是**第一口肉**: 在票还没起爆、没被主流媒体推送、没进涨幅榜 Top 3 时识别出来。本文件适用窗口固定为:

- 涨幅区间: `0% ~ +15%`(超出转其他文件,详见 §明确放弃)
- 催化新鲜度: `<4 小时`(超出转 case-studies 或 earnings-lottery)
- 榜单位置: 未进入主流 movers Top 3,或 AH/PM 榜 4-15 名
- 量能: 相对量正在 `1x → 3x → 5x` 上升,而非已达 `50x+`

本文件**不是事后追榜**:`top-runner.md`(榜首事后)和 `extreme-lottery-leaderboard.md`(+80% 事后审计)只在用户**明确点名**时才加载,不参与默认主线。

## 5 类点火前夜原型

每只候选必须先归类到某一原型,并标注`原型编号 1-5`。若同时命中多原型,按命中数最高的原型为主,其它列为辅助。

### 原型 1 — 盘后低基数初燃 (Post-Hours Early Ignition)

| 字段 | 内容 |
|---|---|
| 识别特征 | 公告时间戳 `<4h`;AH/PM 涨幅 `+10%~+30%`;AH 榜单第 `4-10` 名而非 Top 3;常规盘量比 `<1` 或收跌 |
| 涨幅区间 | `+10% ~ +30%`(AH/PM 段) |
| 候选来源 | AH/PM movers 第 4-15 名 + SEC EDGAR 8-K 最近 4 小时 + 财经 PR Wire 时间戳 |
| 失效信号 | 进 AH/PM Top 3 即转 [top-runner.md](../modes/top-runner.md);AH 涨 `>30%` 转 [extreme-lottery-leaderboard.md](../modes/extreme-lottery-leaderboard.md) §HCWB 参考审计;点差 `>2%` 或几分钟无成交 → 只观察 |
| 关联 mode | [extended-hours-scalp.md](../modes/extended-hours-scalp.md)(夜盘/盘前做 T 战术) + [extreme-lottery-leaderboard.md §HCWB 参考](../modes/extreme-lottery-leaderboard.md)(若有资本红旗) |

## 早信号清单 (Early Ignition Signals)

10 项打分卡,每项命中 +1。**满分 10 分,分数同时决定能否升仓**:

| # | 信号 | 判定口径 |
|---|---|---|
| 1 | 公告时间戳 `<4h` | 以 IR/SEC/8-K 公布时间为准,不以 Web 推送时间为准 |
| 2 | AH/PM 榜单 4-10 名(非 Top 3) | 防止已被主流盯上 |
| 3 | 相对量 `>5x` 但涨幅 `<20%` | 量已抢跑、价未跟上 |
| 4 | 板块龙头资金外溢 | 同业 ETF 资金流入加速 + 同业相对量倾斜 |
| 5 | 历史财报反应至少 1 次 `>=+30%` | 引用 [../core/pre-recommend-triplet.md §校验1](../core/pre-recommend-triplet.md) |
| 6 | 52 周低位区 `+ 突发硬催化` | 距 52 周低 `<15%` 且公告硬度 `>=B` |
| 7 | 公告硬度 `A/A-/B` | 引用 [catalyst-stage.md §催化硬度分层](catalyst-stage.md) |
| 8 | 流通盘 `<30M shares` | 低流通可放大波动 |
| 9 | short float `>=15%` | 拥挤空头叙事 |

### AI 微盘点火主题加分 (INHD/CHAI 型)

当 48 小时内市场反复出现 `AI agent / AI data center / AI infrastructure / AI sales platform / AI development agreement / AI pivot` 题材的微盘暴拉时,在 10 项早信号之外额外打一个**主题热度标签**,但不直接替代硬门:

| 标签 | 升级条件 | 处理 |
|---|---|---|
| `AI硬催化` | PR/SEC/8-K/6-K 可追溯;有金额、客户、合同、部署/交付节点或收入路径;市值 `<$300M` 或流通 `<30M`;当前成交额跃迁 | 可进入原型 4 微盘公告窗口或原型 3 同主题传导;仍需 A/B 报价和红旗通过 |
| `AI软催化` | AI development agreement、MOU/LOI、顾问任命、AI pivot、数据中心意向,但缺少确定收入或商业部署 | 只能观察/极小仓触发;若已涨 `>=40%` 转极端彩票审计 |
| `AI贴标签` | 旧闻重炒、新闻标题带 AI 但公司主营无清晰变更、无当天官方来源 | `无源异动审计`;不得写主攻/备攻 |

AI 主题加分只在同时命中至少 2 个妖性结构信号时生效: 低流通/小市值、前一完整日低基数、当前成交额跃迁、高 ATR/翻倍基因、同主题龙头当日已爆。典型校准: `INHD 2026-06-08` 属于`AI软催化 + 低流通 + 当前成交额跃迁`,可解释点火但不适合建仓;`CHAI` 若找不到 48 小时官方硬催化,只能归入`AI贴标签/低流通盘后审计`。

**门槛**:

- `<3`: 不入池,只观察
- `>=3`: 进**预判仓**(账户风险 1/3)
- `>=5`: 升**确认仓**(账户风险 2/3)需同时满足 §三段加仓状态机 第 2 段触发条件
- `>=7` 且涨幅 `>+5%`: 升**主攻仓**(账户风险全额)需同时满足 §三段加仓状态机 第 3 段触发条件

## 点火前预测评分卡 (Prediction Radar)

本节用于 S8 预测/验证场景,也用于默认妖股请求里的`预测候选池`。目标是把"最近一周实际打出 `>=40%` 的妖股"拆成点火前可观察信号,形成可复盘的概率雷达。预测不是收益承诺;它只决定候选是否进入观察/预判/确认流程,仓位仍受报价证据等级、R/R、红旗和三段加仓状态机约束。

### 三段分层

每只候选必须按发现时涨幅分层,不得混在同一张表里:

| 分层 | 发现时涨幅 | 处理 |
|---|---:|---|
| 点火前 | `0%~+15%` | 核心预测池;必须给原型、预测分、触发价、失效价 |
| 接棒 | `+15%~+40%` | 挑战者/二次点火;只做回踩承接或二突,不得写"第一口肉" |
| 事后审计 | `>+40%` | 榜首/极端彩票审计;**默认不主攻**, 但**必须展示**, 用于复盘漏网/下次提前 + 看是否有第二天延续机会;按 [modes/extreme-lottery-leaderboard.md](../modes/extreme-lottery-leaderboard.md) 处理 |

#### 扫描 SQL 反模式 (canonical, 2026-05 加固)

**禁止的 SQL 过滤写法 (会让用户看不到完整图景)**:

```sql
-- ❌ 错误: 把 +30%+ 票全部排除, 用户看不到极端彩票池
WHERE (ret_1d) BETWEEN -10 AND 30

-- ❌ 错误: 把 -10% 以下票排除, 会漏掉深跌反弹候选 (派系 C)
WHERE ret_1d >= -10

-- ❌ 错误: 直接卡当日涨跌幅做主筛
WHERE ret_1d <= 50
```

**正确写法**: `ret_1d` 只作为**分层标签**, 不作为硬过滤; 用 dollar_vol / vol_ratio / 派系信号 做主筛, ret_1d 用 CASE 表达式标层:

```sql
-- ✓ 正确: ret_1d 只用作分层标签
SELECT ...,
  CASE
    WHEN ret_1d >= 100 THEN 'EXTREME 100%+'
    WHEN ret_1d >= 40  THEN '审计 40-80%'
    WHEN ret_1d >= 15  THEN '接棒 15-40%'
    WHEN ret_1d >= 0   THEN '第一口肉 0-15%'
    ELSE '下跌反转候选'
  END AS layer
FROM ...
WHERE close >= 0.20
  AND avg_vol_20p >= 500000
  AND close*volume >= 5000000  -- 主攻硬门 $5M+
  AND volume >= avg_vol_20p * 3  -- r2 量能跃迁
  -- 注意: 没有 ret_1d 过滤上下限
```

这条规则是 2026-05 实操发现的 bug 后加固: 之前 LLM 给推荐时写过 `BETWEEN -10 AND 30`, 把 GOVX (+79%) / HCAI (+98%) / VRAX (+52%) / AIM (+63%) / NXXT (+46%) / GCTS (+41%) 等当日大涨候选全部静默过滤, 让用户看不到完整图景。**任何 ret_1d 上限过滤都是反模式**, 即使最终决策档是观察/不推荐, 也必须先扫描出来再降级。

### 预测状态

| 预测分 | 状态 | 输出动作 |
|---:|---|---|
| `8-10` | A 级预测 | 写入预测候选日志;若报价 A/B 且 R/R 达标,可进确认候选 |
| `6-7` | B 级预测 | 写入预测候选日志;只能条件触发/预判仓 |
| `5` | Web升级观察 | 先写入预测候选日志的观察层;必须用 Web/quote 查当前催化、gap、成交额跃迁;无 A/B 报价不得主攻 |
| `4` | C 级观察 | 可列观察,不写主攻/备攻;若次日大涨,复盘标`弱信号漏网` |
| `<4` | 不入池 | 若后续大涨,优先标`不可提前发现`或`盘中突发` |

MCP/SQL 回放分是 OHLCV-only 分数,刻意不使用未来新闻和实时盘前成交。回放中 `5` 分且次日打出 `>=20%` 的票,优先判为`需要 Web 升级`而不是立即追认规则失败;只有在当时公开 Web/quote 已有催化或成交额跃迁、但候选池仍未列出时,才标`漏网`。

### 本期验证后操作总览 (2026-05, 单期)

**这是当前判定 monster 时应该用的有效信号集**。研究证据见 [validation/walkforward-202605.md](../validation/walkforward-202605.md) (规则验证) + [validation/supervised-discovery-202605.md](../validation/supervised-discovery-202605.md) (数据驱动发现)。两份报告基于同一 33 天 (2026-03-30 ~ 2026-05-14) 数据, ~98k 候选, 876 个 ≥40% monster, 基线 p0(40%)=0.90%。

#### 三派系框架 (主攻判定主线, 互不相交, 并行扫描)

本期数据揭示 monster 来自三个**互不相交**的派系, 应作为**并行硬门**而非堆叠分数 (相互堆叠会漏掉单派系强信号):

**派系 A — 微盘低基数 (现有 scorecard 派系, 全部 ✓)**
- 入场条件: 流通 `<30M` 或市值 `<$300M` (规则 5, lift_40=5.08x)
- 量能触发: 当日成交额相对 20 日均 `>=3x` (规则 2, lift_40=6.04x, **本期最强单条规则**)

### 双分数机制: 纸面雷达分 vs 可交易确认分

最近一周回放显示,仅用前一日 OHLCV 会产生大量 `6` 分误报。以后所有预测必须拆成两步,防止把"像妖股"误写成"能买":

| 分数 | 数据来源 | 用途 | 升级条件 |
|---|---|---|---|
| 纸面雷达分 | MCP/SQL 前一完整日、profile、历史涨幅 | 生成预测候选日志,发现可能点火的低基数票 | `>=6` 入 B 级纸面预测;`5` 入 Web升级观察 |
| 可交易确认分 | Web/quote 当前 movers、AH/PM gap、48h 催化、当前成交额、bid/ask | 决定能否从日志升级为预判仓/确认仓 | 必须补足`当前/盘前成交额跃迁`或`48h 新鲜催化`之一,且报价证据 A/B |

硬规则:

- `纸面雷达分 >=6` 但没有 Web 当前量价或 48h 催化,只能写`纸面预测/待触发`,不得给主攻/备攻。
- `纸面雷达分 =5` 且命中`低流通/低基数/弱势反转/翻倍基因`任意 2 项,必须进入`Web升级观察`;HCWB/RETO/FCUV 型往往就在这里,不能直接丢弃。
- `纸面雷达分 >=6` 但前一日成交额过低、当日没有盘前/AH 成交额跃迁,优先标为`静态壳风险`,不得直接主攻;滚动验证显示它不能作为硬剔除,只能作为 Web gate 前的风险标签。
- 已经 `>+40%` 的票只能算`审计命中/接棒命中`,不能倒算成`第一口肉命中`。

### 滚动 Walk-forward 采纳规则

S8 预测/验证复盘优先使用 MCP `rolling_ignition_walkforward`。规则只有通过滚动验证后才能写成 skill 结论;单窗口复盘只能生成假设。

默认参数:

| 参数 | 默认 | 说明 |
|---|---:|---|
| `lookback_days` | `12` | 最近 12 个完整交易日 |
| `train_days` | `5` | 每折只用前 5 个交易日观察妖股环境 |
| `validate_days` | `2` | 用后 2 个交易日验证,不回看未来 |
| `monster_threshold` | `40` | 验证日收盘涨幅 `>=40%` 定义为妖股;盘中最高 `>=40%` 只作辅助 |

分层规则:

- `主攻层`: 纸面雷达分 `>=6`,且不是`静态壳风险`;目标是少而准,实盘仍需 Web/quote gate 才能升级。
- `观察层`: 纸面雷达分 `=5`,且命中低基数/低流通/弱势反转/翻倍基因任意 2 项;目标是防漏,尤其 HCWB/RETO/FCUV 型低基数票。
- `静态壳风险`: 纸面雷达分 `>=6`,但前一日成交额低于 20 日均成交额 `50%`;不能直接剔除,必须等待盘前/盘中成交额跃迁或 48h 催化确认。

采纳标准:

- 主攻层若在多数折中有 `>=20%` 最大顺向收益且误报不失控,保留为`主攻层可保留`。
- 观察层若捕获主攻层漏掉的 `>=40%` 妖股或 `>=20%` 强命中,保留为`Web升级观察`,不得直接主攻。
- 某规则只在单折有效时,只能写`待观察假设`,不得沉淀为稳定结论。
- 若主攻层误报数超过命中数 `3x`,不能写成稳定主攻结论,必须强化 Web gate 或降级为条件规则。
- 若`静态壳风险`在滚动验证中多次出现后续强命中,不得作为硬过滤写入 skill;只保留为`无当前量能不得主攻`。

### 盘前升级硬门 (Web Upgrade Gate)

任何 S8 或默认妖股预测输出,都必须对 `纸面雷达分 >=5` 的候选做盘前/盘中升级检查:

1. 查当前 movers 或 AH/PM 榜:是否从低基数变成 `+3%~+30%` 初始 gap;`>+40%` 转接棒/审计。
2. 查当前成交额:是否已达到 20 日均成交额 `>=5x`,或 30 分钟成交额超过前一日全天 `50%`。
3. 查 48h 催化:是否有财报、8-K、PR、临床/FDA、合同、控制权/债务重组等可追溯来源。
4. 查可成交性:连续成交、spread `<=2%`,拿不到 bid/ask 或时间戳则保持 C/D。
5. 查红旗:7 天内 ATM/offering/反拆/合规/停牌/无源拉升,命中硬拒绝则只审计。

升级结论只能是:

- `升级为预判`: 纸面分 `>=6` + Web gate 至少 2 项 + 报价 B 以上。
- `升级为确认`: 纸面分 `>=6` + Web gate 至少 3 项 + 报价 A + 站稳 VWAP/开盘价/盘前承接区。
- `保留观察`: 纸面分 `5-7`,但 Web gate 只有 0-1 项。
- `剔除误报`: 纸面分高但无催化、无量能跃迁、无连续成交,或触发红旗。

### 可提前/不可提前判定

复盘 `>=40%` 妖股时,必须逐只写:

- `可提前发现理由`: 点火前已经可见的预测信号编号,例如低基数成交额、盘前 gap、48h 财报/8-K、低流通、同主题接棒。
- `不能提前发现理由`: 直到盘中/盘后才公告、只有实时券商榜单可见、公开源无报价时间戳、或前一日/盘前预测分 `<4`。
- `规则回写`: 若漏网票预测分本应 `>=6`,必须说明要强化哪条候选生成规则;若预测分为 `5`,先检查 Web 升级流程是否漏跑;若预测分 `<4`,不得硬改规则去追认。

### 最近一周校准结论 (2026-05-12 至 2026-05-18)

最近一周 `>=40%` 妖股显示三类最常见点火前痕迹:

- `低基数 + 成交额跃迁 + 盘前 gap`: HCWB/PIII/GOVX/SBFM/LESL/SNAL 类,日线前一日常不强,必须靠盘前/盘后 Web + 当前成交额发现。
- `前日已强 + 次日接棒`: TDIC/QUCY/AIIO/WOK/ERNA 类,不是第一口肉,但可在接棒池用回踩/二突做快打。
- `低位弱势反转 + 爆量`: CREG/AIM/VRAX/ATYR 类,多数红旗重,只能预测观察或极小仓触发,不能因涨幅大自动升级。

因此,默认妖股请求必须先输出预测候选日志,再输出确认/榜首审计;不能只从已经 `>=40%` 的榜单开始。

## 三段加仓状态机 (Tranche State Machine)

本节是本文件 canonical 定义。`risk-rules.md` 只引用本节,不重定义。**每段必须先满足触发条件再加仓,不允许跳级。**

### 段 1 — 预判仓 (Forecast tranche)

- **本金**: 账户风险额的 `1/3`,即账户风险 `0.15% ~ 0.7%`
- **触发条件(全部满足)**:
  - 命中 5 原型其一
  - 早信号 `>=3`
  - 报价证据等级 B 及以上(引用 [../core/web-cross-check.md](../core/web-cross-check.md))
  - pre-recommend-triplet 强制项已跑完(原型 2/4 必跑)
- **降级条件(任一触发)**:
  - 早信号掉到 `<3`
  - 命中红旗(引用 [red-flags.md](red-flags.md) §直接拒绝)
  - 5 原型对应失效信号触发
  → **直接全出,不允许扛**

### 段 2 — 确认仓 (Confirmation tranche)

- **本金**: 升到账户风险额的 `2/3`
- **触发条件(全部满足)**:
  - 涨幅突破 `+5%~+10%` 且站稳 VWAP
  - 相对量加速(从 `1-3x` 升到 `3-10x`)
  - reclaim VWAP/开盘价/盘前承接区其一
  - 报价证据等级 A
  - 早信号补强到 `>=5`
- **降级条件**:
  - 跌破 VWAP 5-15 分钟收不回 → 降回预判仓或全出
  - 量能未跟 → 不加仓,只持有

### 段 3 — 主攻仓 (Lead tranche)

- **本金**: 升满账户风险额(详细数字按 [../core/risk-rules.md §仓位纪律](../core/risk-rules.md))
- **触发条件(全部满足)**:
  - 进入 movers Top 5
  - 量能持续(20 日相对量 `>=10x`)
  - 第二段空间确认(突破前一日高点或盘前高点)
  - 早信号 `>=7`
  - 报价证据 A 且无新增红旗
- **降级条件**:
  - 从日内高点回撤 `>=12%` 且不能 reclaim VWAP → 按 [exit-framework.md](exit-framework.md) 减仓
  - 出现红旗(ATM/offering/反拆/halt)→ 立即全出

### 状态机硬规则

- **不允许跳级**: 必须 1 → 2 → 3 顺序加,不得直接从预判仓跳主攻仓
- **不允许加仓在追高**: 升仓只能在回踩、reclaim 或二突结构中加,不得在垂直拉升中加
- **单票总仓位上限不变**: 三段累计不得突破 [../core/risk-rules.md §仓位纪律](../core/risk-rules.md) 的"单票本金 `30%` 默认上限"
- **亏损/翻本心态下禁用**: 用户处于亏损、翻本、心态失衡状态时,本状态机直接禁用,只允许观察或单笔小仓试错(按 [../core/risk-rules.md §情绪与恢复模式](../core/risk-rules.md))

## 候选排序确定性 tie-breaker (canonical)

为了让相同输入在不同模型 / 不同会话 / 不同平台输出一致,候选排序**不允许**用主观偏好。同一池里候选合并后,按以下硬规则**逐级**排序:

### 第 1 级 - 原型优先级

不同原型的票排序权重(数字越小越优先):

| 原型 | 优先级权重 |
|---|---:|
| 原型 1 盘后低基数初燃 | 1 |
| 原型 2 催化日历埋伏 | 2 |
| 原型 3 同业传导接棒 | 3 |
| 原型 4 微盘公告窗口 | 4 |
| 原型 5 二次点火重启 | 5 |
| 多原型同时命中 | 取命中原型里最小权重 |

理由: 1 是最早期(盘后跳涨初起),5 是已涨过又重启。早识别 = 更接近"第一口肉",权重越高。

### 第 2 级 - 早信号命中数(降序)

早信号 X/10 命中数高的排前面。同样原型时,信号多 = 更可能点火。

### 第 3 级 - 量比(降序)

`symbol_snapshot.volume_ratio` 高的排前面;早信号同分时,量比已经放大的资金更可能继续。

### 第 4 级 - 流通盘(升序)

`company_fundamentals.float_shares` 小的排前面。流通越小,弹性越大。

### 第 5 级 - short float(降序)

`short_percent_of_float` 高的排前面。空头拥挤 + 任何利好 = 挤压可能性更大。

### 第 6 级 - 市值(升序)

`market_cap` 小的排前面。同样基本面,小市值波动倍数更大。

### 第 7 级 - ticker 字母序(升序)

最后兜底,确保完全确定性。

### 强制使用规则

- **不允许**用"我感觉这只更干净 / 这只看起来更稳"为理由跳过 tie-breaker
- **不允许**因为"用户点名了某只"而把它排前面(用户点名只决定是否进候选池,不决定排序)
- 输出必须显式写出候选排序应用了哪一级 tie-breaker 决定的(如"SNAL > SKK 因为 SNAL 命中原型 5 + 2 双原型,SKK 单原型 4")

### 关键字段完整性

每只 A/B 候选的下列字段必须有具体值;拿不到关键字段时必须写明来源失败并降为 C/D 观察,不能编造:

- 流通盘(从 `company_fundamentals` 拿)
- short float(从 `company_fundamentals` 拿)
- ATR14%(从 `symbol_snapshot` 拿)
- 量比(从 `symbol_snapshot` 拿)
- 当前 quote + 时间戳(从 Web 拿)
- 7 天红旗状态(从 Web SEC 拿)
- 历史财报反应(原型 2/4 必填,从 MCP 历史 SQL 拿)

任一关键字段无法取得时,该票不得给主攻/备攻。

## 用户点名票全部已超窗口反推流程 (canonical)

**触发条件**:用户在一次请求里点名 1 只或多只具体 ticker,且**全部已涨 `>+15%`**,即全部超出本框架适用窗口。常见场景:用户拿涨幅榜 +120% 票来问"还能不能上";用户问"LESL/QUCY/CREG 怎么样";用户带翻倍/翻本心态,自带一组已飞票来寻找入场点。

**默认行为**(必须执行,不允许跳过):

1. **不顺着点名走**:不得直接给点名票的入场/止损/止盈/概率表,即使用户问得很急。先按 §明确放弃 转入对应事后审计 mode,显式标出"本框架不适用",并按 [../core/output-contract.md §证据等级与概率](../core/output-contract.md) 把这些票降为 `观察/不追`。

2. **主动反推同主题点火预判候选**(走 5 原型):基于用户点名票的主题/板块/催化类型,反推 0%-15% 涨幅区间内的同主题候选。反推路径:
   - 提取用户点名票的**主导叙事**(板块、题材、催化类型)。例: LESL = pool 零售财报 squeeze;QUCY = 军工无人机题材;CREG = 微盘纯 pump 无主题
   - 在该主题板块内扫 5 原型:
     - 原型 1: 同板块 AH/PM 榜 4-15 名,公告 <4h
     - 原型 2: 同板块 0-3 天财报临近 + 历史反应 ≥+30%
     - 原型 3: 用户点名票就是龙头 → 扫**二三梯度同业**(相对量 >3x、涨幅 <15%)
     - 原型 4: 同板块微盘 <$2 亿、突发 8-K/PR <4h、未上榜
     - 原型 5: 同板块 5-20 日内曾强势、当日 reclaim VWAP 的票
   - **若主题为"纯 pump 无主题"(如 CREG)**:5 原型无可传导逻辑,但仍要走第 3 步高弹性扫描

3. **同时调用高弹性翻倍候选池扫描**(必跑):用户既然带着 +120% 票来问,**几乎一定带着高赔率/翻倍诉求**;5 原型只覆盖"安全的第一口肉",不覆盖"我就要赌翻倍",必须同时调用 [../modes/high-elasticity-lottery.md](../modes/high-elasticity-lottery.md):
   - 执行 [../../scripts/hyper_elastic_audit.sql](../../scripts/hyper_elastic_audit.sql) 全市场扫翻倍基因票
   - 优先返回与用户点名票**同主题板块**的高弹性候选(若同主题没有则给跨主题 A 级)
   - 按 high-elasticity-lottery.md A/B/C 评级输出
   - 若 A 级 0 只,诚实说明,不允许把 C 级冒充 A 给用户

4. **强制三栏对比展示**:把"用户点名票(已飞,只观察)"、"同主题点火预判候选(0%-15%,第一口肉入口)"、"高弹性翻倍候选(可能翻倍的进攻型彩票)"并列展示。格式:

   ```md
   ## A. 用户点名票 (已超窗口,只观察)
   | 票 | 当前涨幅 | 转入 | 结论 |
   |---|---:|---|---|
   | LESL | +144% | top-runner 事后审计 | 只观察/末段追风险高 |

   ## B. 同主题点火预判候选 (0%-15%, 第一口肉入口)
   | 票 | 当前涨幅 | 原型 | 早信号 | 仓位段 | 结论 |
   |---|---:|---:|---:|---|---|
   | XYZ | +6% | 原型 3 同业接棒 | 5/10 | 预判仓 | 可触发 |

   ## C. 高弹性翻倍候选 (赌翻倍进攻型彩票)
   | 票 | 等级 | 翻倍基因 | ATR14% | 流通(M) | 临近催化 | R/R | 仓位 | 结论 |
   |---|---|---|---:|---:|---|---:|---:|---|
   | ABC | A | 180天内+120% | 14 | 18 | 财报3天后 | 2.1 | $XXX | 主攻彩票 |
   ```

5. **若反推后 B 池 + C 池 都为 0**: 诚实说明"今天该主题既没有 0-15% 第一口肉入口,也没有高弹性翻倍候选;唯一选项是观察用户点名票末段(极小仓)或放弃今天该主题"。**不得**为了凑数硬上。

6. **结合 [../core/risk-rules.md §情绪与恢复模式](../core/risk-rules.md) 真实信号检查**:仅在用户消息出现真实亏损/被套/心态崩/财务风险信号时(不是单纯"想翻倍"),才切换降档处置。`想翻倍 / 想赚多 / 要极端` 等高赔率诉求**不触发降档**,而是触发本流程 + 高弹性候选池调用。

**禁止行为**:

- 不得只给"用户点名票观察"就结束,等同于让用户继续盯着已飞票
- 不得在反推时绕过 R/R 最低硬门或 5 原型识别条件
- 不得用"用户已经看好这只所以我跟着分析"作为放弃反推的理由
- **不得把"想翻倍"贴上心态问题标签自动降档**:这是错误的产品设计;skill 的责任是用 high-elasticity-lottery.md 真实地找高弹性候选,不是降档说教

## 回放校准样本(指引,不强制完整列表)

- 原型 1 典型: 财报后 AH 跳涨 +18% 但 AH 榜第 6 名,常规盘量比 0.6,4h 内 8-K 公告硬度 B 级
- 原型 2 典型: 财报前 2 天,历史 4 次有 1 次 +35%,盘前 +4%~+7% 初起,板块主题正在被资金追
