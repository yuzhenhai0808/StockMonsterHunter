# 夜盘 / 盘前做 T

**场景**: 用户在夜盘或盘前交易强波动票,目标是快进快出吃 `6%-20%` 波段。  
**目标**: 不是预测全天最终收盘,而是在流动性较差的延长交易时段里判断是否有可控的小仓做 T 结构。

## 触发条件

- 夜盘
- 盘前做T
- after-hours
- premarket scalp
- 快进快出
- 已经做了一轮,问还能不能继续 T
- 跌得太狠 / 当天跌 80%+ / 收最低 / 捡尸 / 反抽 / 抄底 AH(走 §极端超卖 AH 反抽 子模式)

## 核心原则

夜盘/盘前的 edge 来自短时间流动性错配,不是来自更高胜率。

夜盘/盘前价格默认可通过用户券商、公开 quote 页和 movers 页面获取;不得直接退回昨收。若用户没有给盘口,先按 [../core/web-cross-check.md §延长时段价格获取强化](../core/web-cross-check.md) 跑取价链:

`用户券商 → quote页 → 第二公开源 → movers发现价 → 新闻描述 → MCP背景`

只有拿到 A/B 级 extended-hours quote 并通过当前流动性硬门,才允许给主攻/备攻或正常仓位;只有 movers 发现价时,本模式只能输出`条件观察/待实时复核`。

必须先回答:

- 现在是首轮试错,还是盈利后的第二轮?
- 用户提供的实时价格是否站在昨收 / 盘前 VWAP / 关键整数位上方?
- 买卖价差是否足够窄,限价单能否成交?
- 若错了,是否能在 `3%-8%` 内退出?

## 先看用户已实现利润

如果用户已经在同一只票上盈利:

- 盈利达到本金 `8%-20%`: 默认先保护利润
- 盈利达到账户 `1.5%-2%`: 对 `$5,000` 账户约 `$75-$100`,默认进入"不把利润吐回去"模式
- 后续同票只能用原风险的 `50%-70%`,或只用利润仓
- 若第二轮失败,当日停止继续做 T

## 必查项

### A. 延长时段流动性

- 买卖价差是否超过 `2%-3%`
- 近几分钟成交是否连续
- 当前价是否远离盘前 VWAP
- 是否只能用市价单才能成交

若点差过宽或成交断层,即使日线成交额很大,也不能按流动性充足处理。

### B. 结构

允许:

- VWAP 回踩后重新站上
- 关键整数位 reclaim
- 盘前小平台二次突破
- 冲高回落后没有跌破爆量起涨区

禁止:

- 垂直拉升中追
- 贴着盘前最高点买
- 跌破 VWAP 后反抽不过还继续做 T
- 已完成一轮大肉后立刻加大本金再追

### C. 催化与红旗

必须查:

- 当日上涨是否有具体催化
- 最近 7 天 offering / ATM / reverse split / toxic financing
- 若是反拆/低流通/壳股,只能按战术票,不能按基本面主攻

## 极端超卖 AH 反抽 子模式

适用场景: 当日跌幅 `>=80%`,收近全天最低点,但 AH/夜盘出现强反抽。

这是 §核心原则 的一个**例外子模式**: 普通 AH 做 T 要求"站在 VWAP 上方",但本子模式承认**极端超卖后的空头回补 + 抢反抽资金**是一种独立机制,与"趋势延续"不同。仅在以下条件全部满足时启用,否则按 §核心原则 处理。

### 入池条件 (全部满足)

- 当日收盘跌幅 `>=80%` 且收盘位置在全天 0%-5% 区间(`close_position_pct <= 5%`)
- AH 第一小时成交量 `>=` 当日 RTH 成交量的 `5%`,且 AH 价格已脱离收盘价至少 `+30%`
- 当日 RTH 成交量 `>=` 20 日均量 `3x`(说明卖盘已耗尽)
- 报价证据等级 `A/B`(用户券商或公开 AH 报价含时间戳和成交量)

### 入场结构 (只允许限价单)

- 限价单埋伏区间: `当日收盘价 ×1.3` 到 `当日收盘价 ×2.0`
- 不在 AH 首笔跳涨中追(空头回补抢跑通常已透支 `+50%~+80%`)
- 当前价已在 `×2.0` 之上时,本子模式不再适用 → 转为观察
- 若 AH 出现两次冲高回落不过同一价位,放弃

### 退出 (硬规则)

- 第一止盈: `+15%~+25%`,触达**必须减半**
- 第二止盈: `+30%~+50%`,触达**必须清仓**或只留利润仓
- 硬止损: 跌破入场价 `-5%~-8%`,或跌破 AH 反抽起点
- **绝不持有过夜到次日开盘**: 9:30 真量进入后,无新催化的 AH 反抽通常回吐 `30%-50%`;次日要做必须重新按 [reclaim-bounce-hunter.md](reclaim-bounce-hunter.md) 评分

### 资本结构硬拒绝 (任一命中即排除)

- 反拆生效 30 天内 + 当前 F-1 / 424B5 / S-1 已 EFFECT 或 pending
- ELOC / equity line / ATM 仍在执行窗口
- 过去 7 天有 toxic financing / 转股票据 / shelf takedown

满足上述任一红旗,即使 AH 反抽再强,也只能 `观察`。原因: 公司可在反抽中以市价增发,你的本金对手盘是公司自己。

### 仓位

按 [../core/risk-rules.md](../core/risk-rules.md) 执行,但本子模式额外约束:

- 单笔账户风险额上限 `0.5%`(显著低于普通主攻),因为开盘跳水风险无法用 AH 止损覆盖
- 单票本金不超过账户 `5%-10%`
- 用户处于亏损/翻本/满仓状态时,**直接禁止**参与本子模式

### 输出要求

主攻/备攻表 §来源池 列填 `极端超卖 AH 反抽`;短评必须额外回答:

- 当日 RTH 跌幅 / 收盘位置 / RTH 成交量比
- AH 第一小时成交量与价格区间
- 资本结构红旗是否触发硬拒绝
- 限价埋伏区间(`收盘价 ×1.3` 到 `×2.0`)
- 不过夜的退出价

## 仓位计算

仓位和盈利后降风险按 [../core/risk-rules.md](../core/risk-rules.md) 执行。本模式只额外判断当前是首轮试错、盈利后的第二轮,还是只能利润仓参与。

## 输出要求

最终候选总表、单票详情表、单票概率表、报价校验和用户可见输出顺序统一按 [../core/output-contract.md](../core/output-contract.md) 执行。本模式只额外要求写清:

- 当前轮次: 首轮试错 / 盈利后第二轮 / 只允许利润仓
- 用户已实现利润: $X 或未知
- 第一止盈后动作: 减半 / 全出 / 剩余保本
- 停止同票继续做 T 的条件
