# 高弹性财报彩票

**场景**: 用户想在财报前埋伏近期可能大涨、翻倍或出现 +30% 到 +100% 跳空的股票。  
**目标**: 不是找最稳的财报股,而是用小仓位买高赔率事件票。

## 触发条件

- 财报埋伏
- 赌财报
- 财报彩票
- 高收益财报
- 近期翻倍
- 想要大肉
- 稳健票收益太小

## 核心原则

高弹性财报彩票和普通财报预判不同:

- 普通财报预判偏胜率
- 本模式偏赔率
- 允许低胜率,但要求仓位小、失效快、组合化
- 不能把 NU 这类低波动优质票当主攻彩票
- 若用户问"今天出财报/今晚开奖",必须先从数据库取未来一周财报池,再用官方 IR/公司公告确认日期,最后只在**今天确认池**里选彩票;不要把未来几天票混入当天推荐

必须先回答一句:

```text
这只票不是因为稳,而是因为 ______ 有可能在财报窗口被重新定价。
```

## 当天确认池流程(硬要求)

用于用户问"今天出财报有哪些 / 今天可以赌哪个财报 / 今晚开奖"。

1. 调本地 `earnings_dates(within_days=7)` 或 SQL 取未来 `0-7` 天财报池
2. 用日线库补成交额、量比、5/20 日涨幅、20 日振幅、距 20 日高点
3. 对高弹性/高成交候选逐只 WebSearch:
   - `"{ticker} earnings date investor relations"`
   - `"{ticker} q1 q2 results date {current_year} site:globenewswire.com OR site:businesswire.com OR site:prnewswire.com"`
   - `"{ticker} reports results {current_month} {current_year}"`
4. 只保留官方确认今天 AMC/BMO 且尚未公布的票;BMO 已公布的票只做结果后反应交易
5. 对保留票计算过去 4 次财报次日上涨次数、最大正反应、最大负反应、最近一次反应
6. 再查融资/ATM/反拆/SEC/做空报告红旗
7. 输出推荐时必须写清`数据库日期 / 官方确认 / 是否 estimated / 历史反应`

官方优先级:
`公司 IR/SEC/PRNewswire/GlobeNewswire/BusinessWire > Nasdaq press release > MarketBeat/StockAnalysis/TipRanks estimated 页面`。

若只能找到 estimated 页面,不得给 A 级;最多 C 级观察。

## 候选优先级

优先:

1. 高 beta 主题票: 量子、AI 算力、核能、加密矿股、生科监管事件
2. 财报前 0-10 天,但市场还没完全定价;若是今天开奖,必须官方确认且仓位更小
3. 过去 4 次财报至少出现过一次 `+20%` 级别次日反应
4. 当前 20 日振幅大,但不是已经连续第三天尾声
5. 空头比例高或板块正在被资金追
6. 财报之外还有板块催化,例如 BTC、量子、AI、FDA、能源订单

降级:

- 低 beta 稳健成长股,即使基本面好,也只能列为防守备选
- 历史财报反应总是小于 `8%-10%`,不适合翻倍目标
- 财报只剩 1-2 天且已经涨完,除非有盘前/盘中结构,否则不埋伏
- 今天开奖但已经在盘前/盘中大幅冲高且接近 20 日高,只能小仓或放弃;不得因为"今晚开奖"而追高
- 财报结果已经公布且盘前涨幅 `>=120%` 时,只能按`结果后快打`处理: 回踩承接或二突触发后快速减仓;若爆量后收近全天低点,次日延续默认降级,不得再用财报利好解释继续持有

## 硬门

### A. 历史反应不是一票否决,但要分级

本模式追求赔率,所以不要求 3/4 次上涨才保留。

评级:

- A: 过去 4 次里至少 1 次 `+30%` 以上,且最近一次没有严重砸盘
- B: 过去 4 次里至少 1 次 `+15%` 以上,或 2 次以上正反应
- C: 历史反应差,但当前主题催化极强;只能极小彩票仓或观察,不得主推
- 放弃: 过去 4 次里多次 EPS beat 仍暴跌,且当前没有新主题催化

**历史反应硬降级**:

- 过去 4 次中 `<=1` 次次日上涨,且最大正反应 `< +8%`: 不适合财报彩票,只能`观察/等结果后反应`,不得列为主推或备攻
- 过去 4 次中 `<=1` 次次日上涨,且最大负反应 `<= -15%`: 即使有热门主题,也只能`观察/极小仓`,不得给正常财报埋伏仓位
- 出现 `EPS beat 仍次日下跌 >=2` 次: 标注`利好出尽/文字利好不被市场买账`,默认放弃;除非当前有全新硬催化,也只能 C 级观察
- 样例规则: ENVX 这类历史财报后多数下跌、最大正反应很小、且曾出现大幅负反应的票,不能因为 edge-AI/电池叙事被升为财报彩票主推

### B. 红旗

必须查:

- offering / ATM / toxic financing
- reverse split
- class action / SEC / going concern
- 财报日期是否已公布或是否只是 estimated

命中融资/ATM:

- 若是微盘/生科: 默认放弃或只允许极小仓
- 若是高成交额主题票: 降一级,且财报前必须减仓

### C. 仓位

仓位按 [../core/risk-rules.md](../core/risk-rules.md) 反推。本模式只负责标注财报彩票等级;不得因为"翻倍"目标放宽账户级风险纪律。
## 典型候选解释方式

### QUBT / QBTS / RGTI 型

主导偏向:

- `量子主题 + 财报窗口 + 高 beta squeeze`

可做原因:

- 股价本身有大波动基因
- 财报日可能触发订单、现金、技术里程碑或指引叙事
- 一旦板块资金回流,弹性远大于稳健财报股

风险:

- 收入基础弱
- 估值靠叙事
- 财报后若没有订单/现金/指引,容易直接跳水

### CLSK / MARA 型

主导偏向:

- `BTC beta + 空头挤压 + 财报窗口`

可做原因:

- 财报本身可能不重要,关键是 BTC 价格、算力、成本和持币叙事
- 历史财报后有过大幅正反应

风险:

- BTC 盘前/周末波动会直接覆盖财报逻辑
- 负现金流或高债务会压制估值

## 输出要求

最终候选总表、单票详情表、单票概率表、报价校验和用户可见输出顺序统一按 [../core/output-contract.md](../core/output-contract.md) 执行。本模式只额外要求在财报彩票池或全局`单票详情表`中写清:

- 数据库日期与官方确认日期是否一致,是否只是 estimated
- 彩票等级与历史反应: 过去 4 次上涨次数、最大正反应、最大负反应
- 为什么它可能大涨甚至翻倍
- 为什么它可能直接亏 `10%-30%`
- 财报前是否需要减仓
- 若财报前提前涨到第一目标,是否还持有过财报

## 明确放弃

- 低波动稳健财报票却被包装成翻倍票
- 用户想单票重仓赌财报
- 财报日期不确定且无法用 IR / SEC / 公司公告确认
- 最近 7 天融资或 ATM 且没有极强盘面结构
- 已经连续 3 天大涨,财报前只剩最后一棒
