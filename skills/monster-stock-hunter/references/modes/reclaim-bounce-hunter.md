# 二次修复 / 反包猎手

**场景**: 用户想找类似 RXT 的机会:前期因真实催化暴拉,随后低开/急跌/洗盘,但盘中重新站回关键位,有机会走二次修复、反抽回成本区或前高。

**目标**: 把 RXT 式盈利方式沉淀为可主动扫描的能力,不是只在用户被套后被动救火。

## 触发条件

- 类似 RXT 的股票
- 二次修复
- 反抽机会
- 下跌回头
- 低开修复
- 跌下来能不能接
- 被套后能不能补仓
- 有没有强票回拉
- 昨天暴拉今天还能不能接
- 第二天还能不能做
- 次日延续 / 延续

## 主导偏向

只能选一个:

- `真催化二次修复`
- `强票低开修复`
- `急跌回拉`
- `红转绿反包`
- `被套反抽修复`
- `次日延续修复`

不要把无源 pump 的随机反弹包装成二次修复。二次修复必须有前置催化或资金逻辑。

## 额外参考

`web-cross-check` 在 S1/S2/S3/S4/S6 场景已加载, `exit-framework` 在 S2 持仓处置场景已加载,本模式不重复列出。仅在用户问 RXT 复盘或类似失败案例时,加读 [../case-studies.md](../case-studies.md) 中的 `RXT 2026-05-11/12`。

## 候选池

### Web 池

WebSearch:

```text
"stocks down then up today unusual volume {date}"
"intraday reversal stocks today {date}"
"stocks reclaim VWAP today unusual volume"
"top gainers yesterday down today rebound"
"{ticker} stock news today earnings partnership MOU"
"{ticker} offering ATM reverse split halt last 7 days"
```

记录:

- ticker
- 当前价/时间戳/成交量或成交额
- 当日开高低/前收
- 是否站回 VWAP;若无 VWAP,用开盘价/昨收/盘前承接区替代
- 是否仍在昨收、盘前 VWAP、昨日尾盘承接平台或前一日关键区上方
- 48 小时内催化
- 过去 7 天红旗

### 本地池

若 MCP 可用,并行筛:

- 近 5 日涨幅 `>=20%`
- 最近完整交易日或当前 Web 显示低开/急跌 `>=5%`
- 最近完整交易日成交额 `>= $20M`,或当前 Web 成交额 `>= $10M`
- 20 日 ATR `>=5%`
- 20 日内曾接近高点,但当前不在退潮低点

本地池只负责防漏,最终能否参与必须由 Web/盘口确认。

### 次日延续过滤

前一日暴拉票只有满足以下多数条件,才可按二次修复/延续候选处理:

- 第一天是真催化或强主题传导,不是无源 pump
- 第二天仍有新增催化、盘前强承接,或至少当前成交额继续接近前一日成交额 `50%`
- 夜盘/盘前没有明显跌破昨收、昨日尾盘承接区或盘前关键平台
- 价格仍存在回踩承接或整理后二突结构,不是只剩贴高追价
- 若已是第三天以上,必须有新催化或相对量重新放大到 `3x+`

若前日极端暴涨后次日从高位崩盘并收近全天低点,盘后或次日盘前反抽只能进退潮反抽审计,不得包装成正常次日延续。

## 二次修复评分

| 条件 | 分数 |
|---|---:|
| 48 小时内真实催化仍在:财报扭亏、硬合同、AI/大客户合作、控制权交易、明确订单 | +2 |
| 催化是 MOU/LOI/框架合作但具名大客户/大厂,且市场已放量认可 | +1 |
| short float `>=15%` 或明确 squeeze/拥挤空头叙事 | +1 |
| 当日或当前成交额显著放大,至少接近前一日成交额 `50%` 或超过 `$20M` | +1 |
| 低点后重新站回 VWAP/开盘价/昨收之一 | +2 |
| 收复用户成本下方 `5%-10%` 逃生区,或收复前一日关键承接区 | +1 |
| 价格仍低于前高 `5%-20%`,有反抽空间 | +1 |
| 第二天仍有新催化、盘前承接,或当前成交额接近前一日 `50%` | +1 |
| 跌破催化前平台、VWAP reclaim 失败、或出现 financing/offering/ATM/reverse split/halt 红旗 | -3 |

结论:

- `>=5`: 可作为二次修复主候选;允许回踩做/二突做,已有仓位可保留反抽仓或小幅修复仓
- `3-4`: 观察/小仓试错;只能等再次站回关键位
- `<=2`: 退潮或无源反弹;不做

## 参与方式

### 空仓

只允许两种:

- `回踩做`: 站回 VWAP/开盘价后回踩不破
- `二突做`: 重新突破反抽平台或前一日关键承接区

不得在第一根急拉中追。

### 已持仓/被套

完整动作链(账户风险表 / 反抽路径表 / 七层降仓 / 补仓硬门)按 [../workflow/exit-framework.md §被套反抽处置](../workflow/exit-framework.md) 执行。本模式额外约束:

- 二次修复评分 `>=5` 且 Web 认证无红旗时,exit-framework §补仓修复输出 中的"二次修复评分达标"条件视为满足
- 评分 `3-4` 时,即使站回 reclaim 位也只能保留反抽仓,不得进入补仓修复仓

## 输出要求

最终候选总表、单票详情表、概率表、报价校验和用户可见输出顺序统一按 [../core/output-contract.md](../core/output-contract.md) 执行。本模式只额外要求在二次点火池或全局`单票详情表`中写清:

- 修复类型
- 二次修复评分
- 触发价、失效价、第一反抽、第二反抽
- 操作: 回踩做 / 二突做 / 保留反抽仓 / 观察 / 放弃

单票展开还必须包含:

- Web 认证: 当前价/时间/成交量/新闻/红旗
- 本地数据: 前收/ATR/量比/20 日位置/short float
- 为什么像或不像 RXT
- 空仓计划
- 已持仓计划
- 补仓是否允许;若允许,写补仓后均价/失效价/最大亏损
- 何时停止继续减仓
- 是否属于第二天机会,还是第三天尾声;若是尾声,直接写放弃
- 今日相对量和当前成交额是否足够支撑二段扩张

## RXT 复用边界

RXT 可复制的是:

- 真催化 + 高空头 + 爆量
- 低开/急跌后守住低点
- 重新站回开盘价/VWAP/成本逃生区
- 成交继续放大并接近前高

RXT 不可复制的是:

- 无新闻、无成交的随机反弹
- 已出现融资/ATM/反拆红旗的反抽
- 跌破催化前平台后无力修复
- 只因为用户亏损就补仓
